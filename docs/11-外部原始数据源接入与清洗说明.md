# 外部原始数据源接入与清洗说明

## 1. 目标

本文件定义外部原始资料如何接入 ACDP，覆盖：

- 原始资料的存放与登记
- 数据清洗与标准化流程
- 中间结果与最终词典的边界
- 当前已知上海路名与政府部门数据源的接入方式

## 2. 原则

原始数据源不直接进入发布词典。

必须经过：

1. 原始接入
2. 结构化清洗
3. 标准化映射
4. 候选生成
5. 人工审核
6. 版本发布

## 3. 数据分层

建议平台内按三层管理：

### 3.1 Raw 层

只保存原始资料，不做人工改写。

### 3.2 Clean 层

保存结构化清洗结果。

### 3.3 Dict 层

保存进入平台审核与发布流程后的词条。

## 4. 目录建议

建议后续在项目内形成如下目录：

```text
ACDP/
  data_sources/
    raw/
    cleaned/
    manifests/
```

其中：

- `raw/` 保存外部原始文件的链接信息或同步副本
- `cleaned/` 保存清洗后的 CSV/JSONL
- `manifests/` 保存数据源登记说明

## 6. 清洗流程

### 6.1 路名数据清洗

建议步骤：

1. 识别文本编码与分隔规律
2. 抽取路名主字段
3. 去除编号、页码、分段标记
4. 标准化路名后缀，如 `路/街/弄/支路`
5. 生成规范名和别名候选
6. 产出 `cleaned/shanghai_roads.csv`

### 6.2 政府部门数据清洗

建议步骤：

1. 校验 CSV 编码与表头
2. 去除空行与重复项
3. 统一部门名称、简称和别名字段
4. 生成机构别名候选
5. 产出 `cleaned/shanghai_gov_departments.csv`

## 7. 从清洗结果到词典候选

清洗结果不等于直接发布词典。

建议转换规则：

- 路名规范名进入 `poi` 类别候选
- 部门名称和简称进入 `gov_term` 类别候选
- 自动生成的别名或拼音候选默认进入待审，不直接发布

## 8. 审核重点

### 8.1 路名类

重点审：

- 是否为真实在用路名
- 是否有高频 ASR 混淆变体
- 是否存在和通用词冲突的短词

### 8.2 部门类

重点审：

- 全称与简称是否匹配
- 是否存在多个部门简称冲突
- 是否需要录入常见口语别称

## 9. 与拼音增强的关系

这两类数据都适合进入拼音画像流程：

- 路名通常实体性强，适合启用拼音候选增强
- 政府部门名称通常较长，也适合启用拼音候选增强

但要求：

- 自动生成的拼音近音词默认只进入候选池
- 必须经人工审核后才能进入自动替换链路
