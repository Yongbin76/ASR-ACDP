本文档用于把 [73-v1.0发布导向重构joblist草案](./73-v1.0发布导向重构joblist草案.md) 进一步细化到 checklist 粒度，作为后续：

## 2. 使用规则

当前 checklist 分为 3 层：

- `A` 方案与 contract
- `B` 后端实现与数据模型
- `C` 前端页面与帮助体系

每个工作包的完成标准以：

- checklist 全部完成
- 对应回归完成
- 文档同步完成

为准。

## 3. R1 运行治理域重构

目标：

- 以备案节点为主对象重建运行治理域

### R1-A 方案与状态模型

- [ ] 明确运行治理域主对象为“已备案节点”
- [ ] 明确未备案 runtime 作为异常接入事件处理，不进入主列表
- [ ] 明确运行节点至少拆成：
  - 注册状态
  - 在线状态
  - 版本对齐状态
  - 最近动作结果
  - 历史异常
- [ ] 明确运行节点中心与节点备案页的职责边界
- [ ] 明确运行验证工具与运行节点中心、发布中心的关系

### R1-B 后端 contract

- [ ] 重构运行节点主 read model，确保主列表以备案台账为主
- [ ] 重构运行节点详情 read model，确保实时状态和历史状态分层
- [ ] 明确“未注册 / 已注册未心跳 / 在线 / 离线 / 未备案接入”口径
- [ ] 收口节点部署提示 contract
- [ ] 收口节点注册失败的错误分类与错误码

### R1-C 前端与帮助

- [ ] 重构节点备案页，使其明确展示部署、密钥、接入说明
- [ ] 重构运行节点中心列表，使其不再混读状态
- [ ] 重构运行节点详情页，使其分层展示实时 / 历史 / 目标状态
- [ ] 为节点备案页补页面帮助
- [ ] 为运行节点页补页面帮助
- [ ] 为节点接入流程补流程手册

### R1-D 回归

- [ ] 补运行节点中心相关单测
- [ ] 补节点备案 / 运行节点 console read 回归
- [ ] 补节点接入工作流回归

## 4. R2 版本发布域重构
