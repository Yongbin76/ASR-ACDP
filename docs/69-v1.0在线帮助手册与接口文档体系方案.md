## 1. 文档目的

本文档用于把 `v1.0` 阶段“在线文档必须细化”的要求正式落地为一套可执行的帮助体系方案。

目标不是补几篇说明，而是建立一套可直接支撑：

- 日常使用
- 联调接入
- 运维部署
- 故障排查

的在线帮助手册体系。

## 2. 为什么这是 v1.0 硬门槛

当前系统已经不是一个单一页面工具，而是一个多域后台：

- 主数据管理
- 内容审核
- 版本发布
- 运行治理
- 系统管理

如果没有完整在线文档，`v1.0` 会出现这些问题：

- 用户不知道每个页面的职责边界
- 不知道按钮的前置条件和后果
- 运维不知道 admin/runtime 怎么部署
- 联调方不知道 runtime/admin 接口怎么调用
- 故障发生时只能靠猜

所以在线文档不应再视为“可选补充”，而应作为正式发布的硬门槛之一。

## 3. 在线帮助手册体系建议

建议拆成 4 大类。

### 3.1 页面级帮助

每个主要页面都必须有独立帮助说明。

至少覆盖：

- 页面作用
- 页面目标
- 适用角色
- 页面主对象
- 页面主要区块说明
- 每个按钮的作用
- 当前页与下一步的关系
- 常见问题

建议覆盖页面：

- 总览
- 词条中心
- 导入中心
- 内容审核
- 发布中心
- 样本与回流中心
- 节点备案
- 运行节点中心
- 运行验证工具
- 用户管理
- 角色管理
- 权限分配
- 治理策略

### 3.2 流程级帮助

按业务流程组织，而不是按页面组织。

至少覆盖：

- 词条新增到发布流程
- 批量导入到发布流程
- 内容审核流程
- release 构建 / 审核 / 灰度 / 正式发布流程
- 回滚流程
- 节点备案到 runtime 接入流程
- 运行验证流程

### 3.3 运维与部署帮助

这部分必须单独成体系。

至少覆盖：

- admin 启动命令
- runtime 启动命令
- `start:runtime:instance` 启动命令
- `service:start:runtime` 守护启动命令
- 多 runtime 部署示例
- `registration-secret` 与密钥指纹的区别
- `ACDP_RUNTIME_TOKEN` 的作用与是否启用的判断
- 节点注册失败的排查步骤

### 3.4 故障排查帮助

至少覆盖：

- 节点已启动但未注册
- 节点已备案但运行节点中心看不到
- 发布审核通过但无法正式发布
- build 后仍看到阻断信息如何理解
- 制品下载失败
- 运行节点离线
- 业务属性配置改后不生效
- 权限不足与治理策略阻断如何区分

## 4. 接口文档必须单独成体系

### 4.1 runtime 对外接口文档

必须单独成体系，不应只零散写在实现说明里。

至少覆盖：

- `POST /api/runtime/correct`
- `POST /api/runtime/correct_cand`
- `GET /ws/runtime/correct`
- `GET /ws/runtime/correct_cand`
- 认证方式
- 请求示例
- 响应示例
- 错误码
- 适用场景

### 4.2 admin / console 接口帮助

这里不是要求把所有内部接口全部公开成外部 API 文档，而是要求形成一套“管理接口帮助”。

建议覆盖：

- 主数据相关接口
- 审核相关接口
- 发布相关接口
- 运行治理相关接口
- 系统管理相关接口

目的：

- 让后续联调和自动化测试不再只能看代码猜接口

## 5. 页面级帮助模板建议

每个页面帮助建议统一模板。

### 5.1 基础信息

- 页面名称
- 页面目标
- 适用角色
- 进入入口

### 5.2 页面主体说明

- 页面主对象
- 主要区块说明
- 状态说明

### 5.3 动作说明

每个关键按钮都要写：

- 作用
- 前置条件
- 点击后会发生什么
- 成功后下一步去哪
- 常见失败原因

### 5.4 常见问题

- 这一页最容易引起误解的问题
- 处理建议

## 6. 文案要求

帮助体系必须和后台页面保持同一套文案规则：

- 用户主文案统一中文
- 技术值只在辅助区和技术详情里暴露
- 同一个概念在页面和帮助里使用同一个名称

例如：

- 统一用“业务属性”
- 不在帮助里再混写“类别编码”

## 7. v1.0 验收要求

`v1.0` 阶段，在线文档至少必须满足：

1. 主要页面都有帮助说明
2. 主要流程都有操作说明
3. admin/runtime 部署命令文档齐全
4. runtime 对外接口帮助齐全
5. 常见故障排查清单齐全
6. 文案中文化与页面口径一致

## 8. 当前建议

建议把这套文档体系纳入后续重构计划中的独立工作包，而不是继续散落在帮助页、实现说明、运维说明和对外接口文档里。
