## 4. 主数据样本建议

### 4.1 词条样本组

| 样本组 | 说明 | 主要测试点 |
| --- | --- | --- |
| 正常标准词 | 合法、可审核、可发布 | 新建、编辑、审核、进入发布输入池 |
| 单字标准词阻断 | 单字标准词无错误词，或错误词中含单字 | 统一准入阻断 |
| alias 冲突样本 | `canonicalText` 命中其他词条 alias | 冲突 trace 和阻断 |
| `poi_road` 样本 | 路名类 | 业务属性校验、导入和 runtime 纠错 |
| `gov_term` 样本 | 政府部门类 | 业务属性校验、审核、发布 |
| 拼音主读音覆写样本 | 自定义主拼音 | 拼音保存与运行效果 |
| 多音字备用读音样本 | `alternativeReadings` | 拼音候选、运行候选 |

### 4.2 建议固定 canonical

建议优先复用当前仓库中已广泛使用的样本语义：

- `工伤认定`
- `祁顺路`
- `上海市发展和改革委员会`
- `市发改委`

以及一组明确的无效样本：

- 单字标准词无错误词
- 单字标准词带单字错误词
- 栏目词
- 说明词
- alias 冲突词

## 5. 导入样本建议

### 5.1 主模板

当前统一主模板：

- `structured_terms_csv_v2`

建议固定以下样本文件类型：

1. 正常结构化词条导入
2. 含 alias 的结构化导入
3. 含拼音字段的结构化导入
4. 含错误行的阻断导入
5. legacy 兼容导入

### 5.2 其他模板

- `term_aliases_csv_v1`
- `validation_cases_csv_v1`

### 5.3 留档要求

每次导入测试建议留：

- 原始文件
- 预览结果
- 错误行
- 导入结果摘要

## 6. validation 样本建议

建议至少固定：

- 正常命中样本
- 不命中样本
- 发布后风险触发样本
- 常用词误识别样本
- 路名误识别样本
- 政府部门误识别样本

## 7. 运行治理样本建议

建议固定：

- 2~3 个备案节点
- 1 个未注册节点
- 1 个离线节点
- 1 个版本不对齐节点
- 1 个 token / secret 错误节点样本

## 8. 发布演练样本建议

建议准备：

- 1 个可成功 build 的版本输入池
- 1 个可成功 publish 的版本
- 1 个可用于 rollback 的上一版本
- 1 组灰度 `trafficKey`
- 1 组可触发发布后风险样本

## 9. 当前用途

本文档当前作为：

- `JOB-090` 的测试样本与导入资产清单基线
- 后续测试 Codex 准备测试数据时的直接参考
