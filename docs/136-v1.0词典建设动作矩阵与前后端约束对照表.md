# v1.0 词典建设动作矩阵与前后端约束对照表

## 1. 文档目的

本表用于核对“词典建设”域中：

- 页面上是否显示某个动作
- 后端是否真正允许执行该动作
- 当前两者是否一致

适用范围：

- 词典记录
- 批量导入
- 词条审核
- 拼音审核

## 2. 对照原则

- 页面不显示，不代表后端一定拦；需要逐项核后端约束
- 页面显示，后端也必须具备同样的状态约束
- 以“前端显示条件 + 后端真实允许条件”一致为达标

## 4. 本轮已收口的重点问题

### 4.1 词条重复送审

- 旧问题：已通过且无新改动的词条仍显示“提交审核”，且后端也允许重复送审
- 当前处理：前端已隐藏按钮，后端也已拒绝重复送审

### 4.2 导入批次整批审核

- 旧问题：`当前导入批次待审核任务` 实际会混入非待审历史任务，且会受 500 条限制截断
- 当前处理：后端只处理 `pending` 的 `term_review`，且不再被旧分页上限截断

### 4.3 拼音候选重复提审

- 旧问题：前端不显示已通过候选的提交按钮，但后端仍可绕过页面重复提交
- 当前处理：后端已拒绝已通过候选的重复提审

### 4.4 已导入批次取消

- 旧问题：页面只在预览阶段显示“取消批次”，但后端对已导入批次仍可执行取消
- 当前处理：后端已补状态校验，已导入批次不可取消

## 5. 后续复核建议

后续若继续复核“词典建设”域，建议按以下顺序：

1. 先核页面是否显示按钮
2. 再核后端是否允许执行
3. 最后核动作结果提示是否与真实处理结果一致

优先关注高风险动作：

- 提交审核
- 批量提交审核
- 确认导入
- 取消批次
- 审核通过 / 驳回
