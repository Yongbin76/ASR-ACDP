## 1. 文档目的

本文档用于把当前围绕 ACDP `v1.0` 重构已经讨论达成一致的核心原则正式固化，作为后续：

- 方案设计
- joblist 拆解
- 页面重构
- 后端 contract 重构
- 帮助手册编写

的统一基线。

本文档不直接进入编码实现细节，而是明确“后续所有重构必须遵守什么”。

## 2. v1.0 总目标

`v1.0` 的目标定义为：

**ACDP 能被真实业务角色持续使用，并可以完成主数据录入 -> 内容审核 -> 版本构建/灰度/正式发布 -> runtime 节点收敛 -> 风险回看 的最小闭环。**

## 3. 已确认的核心原则

## 3.1 发布硬门槛原则

`v1.0` 当前建议的硬门槛：

- `JOB-006` 必须完成
- `JOB-009` 必须完成
- `JOB-007` 默认不作为 `v1.0` 硬门槛，建议延期到 `v1.1`

说明：

- 没有真实部署验证，不应正式发布
- 没有真实宿主机并发与吞吐验证，不应正式发布

## 3.2 主数据与 release 解耦原则

必须明确：

- 词条中心、导入中心修改的是主数据
- 主数据审核通过后，进入下一次 build 的输入池
- release 是不可变快照
- 页面不再暗示“我在修改某个 release 内容”

## 3.3 审核中心边界原则

审核中心只保留内容审核：

- 词条审核
- 拼音候选审核

而：

- `release publish review` 必须归口到发布中心

## 3.4 发布中心生命周期原则

发布中心必须彻底改成生命周期视图。

必须明确：

- `built`
- `canary`
- `published`
- `发布后风险`

并且：

- `built` 是候选快照状态
- `gate` 只作为 `built/canary` 升级前置条件
- `published` 后的异常不再叫 gate，统一叫“发布后风险”

## 3.5 主数据 gate 与 release 校验分离原则

必须明确：

- 主数据准入与审核发生在 build 前
- `built` 之后，不再被主数据 gate 反向阻断
- 发布域只看该 release 自身快照的版本校验
- `published` 后只看发布后风险

## 3.6 运行节点中心主对象原则

运行节点中心的主对象必须是：

- **已备案节点**

不是：

- 仅当前在线节点
- 仅 `runtime_nodes` 表里的记录

说明：

- 运行节点中心是“备案节点治理视图”
- 未备案节点不应进入主列表

## 3.7 未备案节点处理原则

未备案但有 runtime 上报的节点：

- 不进入运行节点中心主列表
- 视为异常接入事件
- 应进入系统告警或审计视图

## 3.8 实时状态与历史状态分离原则

必须彻底区分：

- 当前实时状态
- 最近一次动作结果
- 历史异常
- 目标状态

不允许继续在一个“状态”里混用这些语义。

## 3.9 用户、角色、权限与治理策略持久化原则

`v1.0` 阶段建议采用：

- 统一配置文件持久化
- 管理页维护
- 后端统一读写接口

而不是：

- 继续散落硬编码
- 同时强行建设完整数据库 IAM 平台

## 3.10 在线帮助手册硬门槛原则

在线文档必须作为 `v1.0` 硬门槛。

至少包括：

- 页面级帮助
- 流程级帮助
- 运维部署帮助
- 故障排查帮助
- runtime 对外接口帮助
- admin / console 接口帮助

## 3.11 全面中文化原则

整个后台页面的用户主文案必须全面中文化。

要求：

- 一级导航
- 二级目录
- 页面标题
- 区块标题
- 按钮
- 提示
- 状态说明

都必须优先中文表达。

技术码只允许出现在：

- 技术详情
- 辅助信息
- 调试视图

## 3.12 导航命名与配置驱动原则

一级、二级目录名称需要进一步收口，并统一改为配置驱动。

建议：

- 一级目录按域命名
- 二级目录按对象或动作命名
- 导航、页面标题、breadcrumb、帮助目录使用同一命名源
- 后续通过统一配置文件维护导航结构

## 3.13 前端重构原则

不建议：

- 继续在现有 `/console` 上做零碎 patch
- 在 `v1.0` 同时做大技术栈迁移

建议：

- 先收口后端 contract
- 再重建一套后台壳层与页面模板
- 在现有技术栈基础上完成 `v1.0` UI 框架统一

## 3.14 帮助体系与接口文档分层原则

必须明确：

- 页面帮助
- 流程帮助
- 运维帮助
- 排障帮助
- runtime 接口帮助
- admin / console 接口帮助

分别成体系，不混写。
