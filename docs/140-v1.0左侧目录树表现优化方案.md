# v1.0 左侧目录树表现优化方案

## 1. 目标

本文档用于收口当前后台左侧目录树的表现问题，并给出后续优化方向。

这里说的“表现效果”不只指颜色和字体，还包括：

- 结构层级是否清晰
- 展开/折叠交互是否稳定
- 激活态是否明确
- 导航、标题、breadcrumb、帮助目录是否一致
- 后续是否方便继续扩展

## 2. 当前实现现状

当前左侧导航实现位于：

- [index.html](../console/client/index.html)
- [app.css](../console/client/app.css)
- [app.js](../console/client/app.js)

现状特点：

1. 采用 `details + summary` 作为一级目录折叠容器
2. 二级目录使用普通链接列表
3. 侧栏整体可收起/展开
4. 一级目录展开状态会写入本地缓存
5. 当前激活项会在左侧高亮

## 3. 当前主要问题

### 3.1 层级感不够强

当前虽然已经有一级和二级，但视觉上仍偏“分组卡片 + 链接列表”，目录树感不够明显。

具体表现：

- 一级目录和二级目录的树状关系不够强
- 二级目录更像按钮列表，不像树节点
- 展开后缺少更明显的层级引导

### 3.2 一级 / 二级信息优先级仍不够稳

当前一级目录采用全大写风格和较强视觉包装，二级目录则全部同一密度展示。

风险：

- 一级目录容易显得“像栏目标题”，但未必像真正可操作的树根
- 二级目录之间的主次关系不明显

### 3.3 收起态过于激进

当前侧栏整体折叠后：

- 品牌文本隐藏
- 一级目录标题隐藏
- 二级目录隐藏

这会带来一个问题：

- 收起后几乎失去导航信息，只剩一个很窄的壳

### 3.4 当前页定位信息还不够完整

目前主要依赖：

- 当前链接高亮

但还缺：

- 当前一级目录的更强激活态
- 当前二级目录所在分组的更明确展开和高亮规则

### 3.5 配置化能力不足

当前导航结构仍主要硬编码在：

- [index.html](../console/client/index.html)

虽然命名规则已有文档，但结构本身还没有完全配置驱动。

风险：

- 后续新增二级页时，容易再次散落修改
- 名称一致性要靠人工维持

### 3.6 左侧背景色过深，与右侧内容区冲突

当前左侧侧栏使用较深的整块深色背景。

问题：

- 视觉重量过大
- 与右侧内容区的浅色卡片体系冲突
- 容易让左侧导航抢掉页面主体注意力

本轮应明确优化：

- 左侧背景要变浅
- 仍保留导航区与内容区的分隔感
- 不能再形成“左深右浅、对撞过强”的割裂感

## 4. 优化目标

本轮优化应同时满足：

1. 一级目录像树根，二级目录像树节点
2. 当前页所在路径一眼可见
3. 收起态仍保留最小可读性
4. 展开/折叠状态稳定
5. 目录命名与页面标题、breadcrumb、帮助目录继续同源
6. 后续新增目录不需要继续手写大块 HTML
7. 左侧背景色与右侧内容区不再产生明显冲突

### 5.1 结构方向

建议把左侧导航更明确地收口为：

- 一级目录：树根
- 二级目录：树节点

具体建议：

- 一级目录保留可折叠能力
- 二级目录在视觉上增加统一缩进和树节点感
- 当前分组始终自动展开

### 5.2 视觉方向

建议增加以下层级规则：

- 一级目录标题：更稳定的分组标题样式，不要只靠全大写制造层级
- 二级目录：缩进、间距、节点符号统一
- 激活项：比 hover 更强
- 激活分组：比普通展开分组更强
- 左侧背景：从当前深色墙面效果收口到更轻的导航底色
- 左右分区：保留分隔，但不再形成过重视觉对撞

### 5.3 收起态方向

建议当前侧栏收起后，不要完全失去分组信息。

可选方向：

1. 保留一级目录图标或缩写
2. hover 时展开浮层
3. 当前激活分组在收起态仍可识别

当前收口建议：

- 收起态统一保留一级目录图标，不再显示残缺文字
- hover / focus 时显示一级目录名称浮层
- 激活分组在收起态继续保持更强图标高亮

### 5.4 交互方向

建议固定：

- 点击一级目录：只负责展开/折叠
- 点击二级目录：负责跳转
- 当前路径所在分组：自动展开，且高于本地缓存优先级

### 5.5 配置化方向

建议把导航结构从 [index.html](../console/client/index.html) 中抽离，至少形成：

- 一级目录 key
- 一级目录 label
- 二级目录 key
- 二级目录 label
- path
- feature / permission
- help slug

## 6. 本轮不建议做的事情

本轮不建议把范围拉大到：

- 全量重写壳层布局
- 全站路由重构
- 重新发明一套前端菜单框架
- 顺手改所有页面标题和 breadcrumb 逻辑

本轮应聚焦：

- 左侧目录树的结构、交互、视觉和配置化

## 7. 收尾标准

只有以下条件同时满足，左侧目录树优化才可视为完成：

- 一级 / 二级层级关系明显
- 当前页定位清晰
- 收起态仍保留最小可辨识能力
- 展开/折叠状态稳定
- 导航名称与标题 / breadcrumb / 帮助目录一致
- 左侧背景色与右侧内容区视觉关系协调
- 结构已向配置驱动迈进一步

## 7.1 封板结论

截至 `2026-04-15`，左侧目录树本轮优化已完成并确认可封板。

封板后的执行口径：

- 不再在 `JOB-037` 内继续追加零碎样式修补
- 不再把左侧导航作为开放式持续优化项反复插单
- 后续若要新增图标体系、拖拽排序、导航配置平台化等扩展能力，应单独新开 job
- 后续页面新增或改名时，只允许按现有导航配置与命名基线做受控同步，不得破坏当前树结构和收起态交互
