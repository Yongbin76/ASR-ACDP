# v1.0 真实环境与工具授权准备清单

## 1. 文档目的

本文档用于把 `JOB-090` 后续进入实际测试执行时，需要你提供或授权的真实环境和工具条件明确下来。

## 2. 真实环境准备

### 2.1 部署环境

推进 `JOB-006` 至少需要：

- `kubeconfig`
- 目标 cluster 可达
- namespace
- 镜像仓库地址
- 镜像拉取凭据
- ingress / service 约束
- deployment 方式说明

### 2.2 压测环境

推进 `JOB-009` 至少需要：

- 真实宿主机
- runtime base URL
- 压测时间窗口
- 是否需要 `ACDP_RUNTIME_TOKEN`
- 是否允许 WebSocket 压测

### 2.3 节点接入环境

如需验证真实节点接入，建议准备：

- 节点 `nodeId`
- `nodeAddress`
- `registration-secret`
- admin 当前使用的 `ACDP_RUNTIME_TOKEN`（如启用）

## 3. 工具授权建议

如果后续允许 Codex 获取或驱动额外测试工具，建议按优先级考虑：

### P0

- 覆盖率工具
- contract snapshot 工具
- 更稳定的 e2e 浏览器驱动
- 性能结果格式化工具

### P1

- Markdown / slug 检查工具
- 证据包自动汇总工具
- K8S 验证辅助脚本

### P2

- 视觉回归工具
- 更完整的日志聚合工具

## 4. 需要你明确授权的事项

后续如进入实际执行窗口，建议明确：

1. 是否允许 Codex 拉取额外测试工具
2. 是否允许 Codex 在目标主机 / 环境中执行更多自动化脚本
3. 是否允许 Codex 访问真实 runtime / admin / cluster
4. 是否允许 Codex 生成并归档真实证据包
