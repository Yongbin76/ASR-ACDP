# Console 内部试用说明

## 1. 文档目的

本文件用于给内部试用人员说明：

- `/console` 当前能做什么
- `/admin` 与 `/console` 当前是什么关系
- 当前哪些能力可试用，哪些能力仍属开发态

## 2. 入口说明

当前后台页面入口收口为：

- `/console`

含义如下：

- `/console`：当前后台页面唯一入口
- `/admin`：兼容跳转入口，访问后应跳转到 `/console`
- `/console/help/*`：控制台内嵌帮助页，可直接在浏览器中查看联调说明和试用说明

## 3. 当前建议试用入口

内部试用优先使用：

- `/console`

## 4. `/console` 当前可试用能力

### 4.1 总览

可试用：

- 基础统计查看
- 当前 stable/canary 查看
- 输入文本并现场演示纠错结果
- 查看命中替换项、候选项和当前命中路由

### 4.2 词典建设

可试用：

- 中文化字段标题与筛选
- 新建词条
- 编辑词条基础信息
- 编辑规则
- 编辑拼音画像
- 提交审核
- 停用词条
- 查看关联样本
- 生成拼音候选
- 提交拼音候选审核
- 已提交/已通过候选的按钮状态约束

### 4.3 批量导入

可试用：

- 查看模板
- 下载模板
- 下载示例
- 下载错误报表
- 创建导入批次
- 查看预览结果
- 确认导入
- 查看导入影响词条和生成的审核任务
- 错误报表在无错误行时的禁用态提示

### 4.4 词典审核

可试用：

- 查看审核任务
- 查看目标摘要与目标对象跳转
- 审核通过
- 审核驳回
- 查看目标快照
- 发布审核在提交人与当前操作人相同时会提前禁用“审核通过”按钮并给出提示

### 4.5 发布中心

可试用：

- 构建 release
- 提交发布审核
- 正式发布
- 回滚
- 查看 gate
- 查看 validation
- 高风险动作二次确认

### 4.6 验证与回流

可试用：

- 新建验证样本
- 停用验证样本
- 查看关联词条

### 4.7 帮助中心

可试用：

- 查看宿主联调说明
- 查看内部试用说明
- 在浏览器内直接阅读，不再跳仓库路径

## 5. 当前仍属开发态/观察态能力

当前仍建议标注为开发态：

- `/console` 页面细节与交互仍在持续优化
- 错误词/别名的更细粒度单条操作还可继续增强
- 拼音候选展示还可继续结构化优化
- 导入中心的错误解释和修复建议还可继续增强
- 宿主机仍需继续积累真实浏览器试用反馈

## 6. 试用建议

内部试用建议优先走以下路径：

1. 新建一个词条
2. 先在总览页“输入与纠错演示”区输入一句业务文本，确认演示输出可用
3. 观察创建成功后的“下一步”提示并提交审核
4. 切换审核员身份完成审批
5. 创建一个导入批次
6. 确认导入
7. 查看导入生成的词条和审核任务
8. 构建并发布一个 release
9. 查看 gate / validation 结果
10. 创建一个验证样本并查看关联词条

## 7. 当前用户与权限体系

当前 `/console` 的权限模型已按四层收口：

- 用户
- 角色
- 权限
- 页面功能

当前试用态不是正式登录系统，而是顶部“操作人 + 角色”的身份模拟。

现阶段请重点理解以下事实：

- 内置用户存在固定可切换角色范围
- 左侧导航由 `pageAccess` 控制
- 页面按钮/动作由 `pageFeatures` 控制
- 无权限时，页面入口会隐藏，动作按钮会禁用或不可见

当前内置试用用户建议如下：

- `console_user`：控制台试用用户，可切管理员/发布员/运营员/审核员/编辑员/只读查看员
- `editor_user`：仅词典编辑员
- `reviewer_user`、`reviewer_user2`：仅词典审核员
- `publisher_user`：仅词典发布员
- `operator_user`：仅词典运营员
- `viewer_user`：仅只读查看员

当前限制条件：

- 这套用户体系用于内部试用与权限边界验证，不等于生产级账号体系或 SSO
- 内置用户不能在界面里切到未分配角色
- 如果手工输入未内置的新操作人，系统会把它当作试用态模拟身份，并只保留当前请求角色，这不等于正式授权
- 发布审核存在职责分离约束：提交人与审核人为同一人时，不允许直接审核通过
- 高风险动作仍受权限和二次确认共同限制，例如确认导入、审核通过、正式发布、回滚、批量停用等
- 运行节点中心当前以查看状态、确认异常和核对版本为主，不等于完整运维控制台

## 8. 反馈建议

试用阶段建议重点反馈：

- 哪个入口最不清楚
- 哪个页面最不好理解
- 哪个操作最不顺手
- 哪个字段说明不够清楚
- 哪个错误提示无法指导修复
- 哪个页面虽然能用，但仍然显得慢或阅读压力大

## 9. 使用边界

当前 `/console` 虽已具备主链路可试用能力，但仍然是：

- 内部试用态
- 持续收口中的正式后台页面入口

现阶段结论应是：

- 可以试用
- 可以联调
- 可以继续收集反馈
- 暂不建议完全替代现有 MVP
