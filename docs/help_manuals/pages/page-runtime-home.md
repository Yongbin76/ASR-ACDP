# 页面手册：运行治理

## 1. 页面目标

`运行治理` 页用于把 runtime 相关能力统一收口到一个入口下，先看备案节点和异常接入，再进入节点备案、运行节点和运行验证。

## 2. 适用角色

- 运维操作人
- 发布人
- 管理员

## 3. 主对象

本页的主对象不是在线 runtime，而是：

- 已备案节点的治理状态
- 未备案 runtime 的异常接入事件

## 4. 页面区块

### 4.1 顶部摘要

用于快速查看：

- 备案节点数
- 未注册节点数
- 当前异常节点数
- 历史异常节点数
- 未备案接入事件数

### 4.2 节点备案

说明节点备案的作用，并进入 `节点备案` 页面。

### 4.3 运行节点

说明运行节点主列表只展示备案节点，并进入 `运行节点` 页面。

### 4.4 运行验证

说明当前验证范围，并进入 `运行验证` 页面。

### 4.5 当前下发模式提示

当前运行治理默认按：

- `admin_http_signed`

理解 runtime 快照安装链路。

如 admin 和 runtime 跨机部署，应先到 `节点备案` 页查看部署说明，确认：

- `ACDP_RUNTIME_CONTROL_ADMIN_BASE_URL`
- `ACDP_RUNTIME_ARTIFACT_BASE_URL`
- `ACDP_RUNTIME_ARTIFACT_SIGNED_URL_SECRET`

## 5. 主要按钮

### 5.1 `进入节点备案`

- 作用：进入节点备案台账维护页
- 前置条件：当前身份具备节点备案查看权限
- 下一步：创建节点备案、查看部署说明或轮换密钥

### 5.2 `进入运行节点`

- 作用：进入备案节点治理主列表
- 前置条件：当前身份具备运行节点查看权限
- 下一步：查看某个节点的注册、实时状态、目标状态、最近动作和历史异常

### 5.3 `进入运行验证`

- 作用：进入 runtime 验证工具
- 前置条件：当前身份具备运行验证权限
- 下一步：验证 `correct` 与 `correct_cand`

## 6. 常见问题

### 6.1 为什么这里会出现“未备案接入事件”

说明有 runtime 正在向 admin 上报，但该 `nodeId` 不在节点备案台账中。它不会进入“运行节点”主列表，应先确认是否补备案或清理异常进程。
