# 页面手册：运行验证

## 1. 页面目标

`运行验证` 页用于直接验证 runtime 对外纠错能力，当前先收口 cluster 当前口径下的：

- `correct`
- `correct_cand`

## 2. 适用角色

- 运维操作人
- 发布人
- 管理员

## 3. 页面区块

### 3.1 `correct`

用于验证主纠错结果。

输出：

- 单条 `correctedText`

### 3.2 `correct_cand`

用于验证候选整句集合。

输出：

- `correctedTexts`
- 第 1 条永远是主结果

### 3.3 后续扩展位

说明当前还未做到：

- 指定节点验证
- 灰度对比验证

## 4. 主要按钮

### 4.1 执行 `correct`

- 作用：验证主纠错结果
- 下一步：确认主结果是否符合当前词典与发布状态

### 4.2 执行 `correct_cand`

- 作用：验证主结果和候选整句集合
- 下一步：确认候选整句是否符合预期
