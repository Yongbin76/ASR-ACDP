# 页面手册：词条审核

## 1. 页面目标

`词条审核` 用于决定词条内容是否能进入下一次 build 的输入池。

## 2. 当前支持的处理方式

- 单条审核通过
- 单条驳回
- 当前页勾选任务的批量审核通过 / 批量驳回
- 非批次场景下，当前筛选结果的批量审核通过 / 批量驳回
- 导入批次上下文下的整批待审核任务批量通过 / 批量驳回

## 3. 批量审核边界

- 批量审核当前只覆盖 `term_review`
- 只在 `词条审核` 视图可用
- 非批次场景下可按当前筛选结果批量处理；当前页面筛选条件主要是任务状态
- 如果页面带有 `importJobId`，可以只处理当前导入批次关联的待审核词条任务
- `当前导入批次待审核任务` 只会命中状态仍为 `pending` 的任务，不会重复处理该批次中已通过或已驳回的历史任务

## 4. 统计口径

- 顶部卡片：显示当前审核工作域的总任务口径
- 如果页面带有 `importJobId`，顶部卡片按当前批次任务口径统计
- 顶部卡片：值为 0 的次级状态卡片默认不展示
- 结果区摘要：显示当前筛选结果口径
- 卡片列表：只显示当前页任务
- 分页：固定放在任务结果区块底部

## 4.1 当前页布局

- `词条审核` 视图下，页面当前按 `审核工作视图 -> 批量审核 -> 筛选条件 -> 结果列表` 的顺序展开
- `筛选条件` 当前为单个状态选择框加两个动作按钮，底部对齐
- `批量审核` 当前为 `批量动作 / 执行范围 / 驳回备注 / 执行按钮` 一行排列，底部对齐

## 5. 审核通过后

- 词条进入可参与下一次 build 的状态

## 6. 审核驳回后

- 返回词条页修正内容

## 7. 推荐使用方式

1. 如果是日常零散任务，直接在当前页勾选后批量处理
2. 如果任务来自 CSV 导入，优先从导入详情进入“本批词条审核”视图
3. 批量通过后，回到版本发布构建新版本
4. 批量驳回后，回到词条页修正内容并重新提交审核
