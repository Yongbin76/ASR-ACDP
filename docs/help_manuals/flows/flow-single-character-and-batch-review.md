# 流程手册：单字标准词与批量审核说明

## 1. 这份说明解决什么问题

这份说明只回答两个高频问题：

- 单字标准词什么时候可以录入
- 导入批次下的批量审核到底会处理哪些任务

## 2. 单字标准词录入规则

当标准词去空后只有 1 个字时，系统不会再直接一刀切拦截。

当前规则是：

- 必须填写至少一个错误词 / 别名
- 错误词 / 别名会按 `|` 拆分
- 拆分后会自动去空、去重
- 只要任意一个错误词 / 别名是单字，就会阻断
- 只有当所有错误词 / 别名都不是单字时，才允许继续录入

示例：

- `标准词=王，错误词=汪`
  结果：阻断
- `标准词=王，错误词=汪|忘了`
  结果：阻断
- `标准词=李，错误词=李某`
  结果：允许
- `标准词=李，错误词为空`
  结果：阻断

## 3. 导入批次批量审核规则

当你从导入详情进入“本批词条审核”后，页面会显示：

- 当前批次关联词条数
- 当前批次审核任务总数
- 当前批次待审核任务数

如果你在批量审核里选择：

- `执行范围 = 当前导入批次待审核任务`

那么系统现在只会处理：

- 当前批次下状态仍为 `pending` 的 `term_review`

不会处理：

- 已通过的历史任务
- 已驳回的历史任务
- 其他类型的审核任务

## 4. 页面结果提示怎么理解

批量审核完成后，页面提示的“命中条数”表示：

- 本次真正进入处理范围的待审核任务数

如果页面提示：

- `当前导入批次已无待审核任务，本次未执行任何批量审核动作`

表示：

- 当前批次已经没有剩余待审核任务
- 不是系统报错

## 5. 推荐操作方式

1. 如果是单字标准词，先准备非单字错误词后再录入。
2. 如果任务来自 CSV 导入，优先从导入详情进入“本批词条审核”。
3. 批量审核前先看页面提示里的“待审核任务数”，再执行整批操作。
