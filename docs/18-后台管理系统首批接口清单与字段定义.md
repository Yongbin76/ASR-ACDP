## 1. 文档目的

本文件定义新后台首批最小可落地接口集，重点回答：

- 第一批新后台接口到底有哪些
- 每个接口的请求字段和响应字段是什么
- 哪些接口是聚合查询接口，哪些是写入接口
- 这些接口如何与现有数据表衔接

本文件基于以下文档继续细化：

- [15-后台管理系统重构方案](./15-后台管理系统重构方案.md)

## 2. 核心约束

### 2.1 不替换现有 MVP 接口

以下现有接口继续保留：

- `/api/admin/*`
- `/api/runtime/*`
- `/api/simulate`
- `/ws/runtime/correct`

本文件中的接口属于新后台独立接口集，建议命名空间：

- `/api/console/*`

### 2.2 先做最小闭环

首批接口只覆盖以下最小闭环：

1. 词条可查可编
2. 导入模板可下载
3. 导入批次可创建、可预览、可确认
4. 审核任务可查可决策
5. 发布与 gate/validation 可查看
6. 样本与回流可查询和基础维护

### 2.3 聚合优先

首批接口优先采用“页面直用”的聚合返回，不要求前端自己拼表。

## 3. 统一返回约定

建议新后台接口统一返回以下结构之一。

### 3.1 列表返回

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "total": 0
}
```

### 3.2 单对象返回

```json
{
  "item": {}
}
```

### 3.3 写操作返回

```json
{
  "ok": true,
  "item": {}
}
```

### 3.4 错误返回

```json
{
  "error": {
    "code": "validation_failed",
    "message": "field x is required",
    "details": []
  }
}
```

## 5. 词条中心接口定义

## 5.1 `GET /api/console/terms`

### 作用

返回词条列表页所需的聚合列表。

### 查询参数

- `query`
- `categoryCode`
- `status`
- `sourceType`
- `riskLevel`
- `sortBy`
- `sortDirection`
- `page`
- `pageSize`

### 响应项字段

每条 `item` 建议包含：

- `termId`
- `categoryCode`
- `canonicalText`
- `status`
- `priority`
- `riskLevel`
- `replaceMode`
- `sourceType`
- `aliasCount`
- `latestReviewStatus`
- `publishedStatus`
- `updatedAt`

### 数据来源

- 主体来自 `terms`
- `aliasCount` 来自 `aliases`
- 审核状态来自 `review_tasks`
- 发布状态来自 `releases/release_terms`

补充说明：

- 当前实现已对词条列表做一轮聚合优化，避免按行重复查询 `review_tasks` 与 `release_terms`
- 当前实现已支持后端真实排序，前端可直接传入 `updated_at`、`priority`、`canonical_text`、`created_at`
- 当前前端列表页已接入统一分页，翻页时保留筛选与排序参数

## 5.2 `GET /api/console/terms/{termId}`

### 作用

返回词条详情页的完整聚合对象。

### 响应结构

```json
{
  "item": {
    "basic": {},
    "aliases": [],
    "rules": {},
    "pinyinProfile": {},
    "reviewSummary": {},
    "releaseSummary": {},
    "validationSummary": {},
    "sourceSummary": {}
  }
}
```

### `basic`

- `termId`
- `categoryCode`
- `canonicalText`
- `status`
- `priority`
- `riskLevel`
- `replaceMode`
- `baseConfidence`
- `sourceType`
- `pinyinRuntimeMode`
- `revision`
- `createdAt`
- `updatedAt`

### `aliases[]`

每条建议包含：

- `aliasText`
- `sourceType`
- `sourceRef`
- `createdAt`
- `updatedAt`

说明：

- 当前底层 `aliases` 表没有完整来源字段
- 首批可先返回基础别名列表
- 后续如补来源映射表，再扩展该结构

### `rules`

- `candidateOnly`
- `minTextLen`
- `maxTextLen`
- `boundaryPolicy`
- `leftContextAllow`
- `rightContextAllow`
- `leftContextBlock`
- `rightContextBlock`
- `regexAllow`
- `regexBlock`

### `pinyinProfile`

- `fullPinyinNoTone`
- `initials`
- `syllables`
- `runtimeMode`
- `polyphoneMode`
- `customFullPinyinNoTone`
- `alternativeReadings`
- `notes`

### `reviewSummary`

- `latestTaskId`
- `latestStatus`
- `pendingCount`
- `approvedCount`
- `lastReviewedBy`

### `releaseSummary`

- `includedReleaseCount`
- `latestReleaseId`
- `latestReleaseVersion`
- `currentStableIncluded`
- `currentCanaryIncluded`

### `validationSummary`

- `relatedCaseCount`
- `failedCaseCount`
- `latestFailedCases`

### `sourceSummary`

- `sourceType`
- `sourceLabel`
- `importJobId`
- `sourceFileName`
- `sourceRowNo`

说明：

- 首批如底层尚无 `importJobId/sourceFileName/sourceRowNo`，可返回 `null`
- 但字段先定义出来，便于后续导入中心接入

## 5.3 `POST /api/console/terms`

### 作用

新建词条。

### 请求体

```json
{
  "categoryCode": "gov_term",
  "canonicalText": "工伤认定",
  "aliases": ["工商认定"],
  "priority": 90,
  "riskLevel": "medium",
  "replaceMode": "replace",
  "baseConfidence": 0.92,
  "sourceType": "manual",
  "pinyinRuntimeMode": "candidate",
  "rules": {},
  "pinyinProfile": {}
}
```

### 字段说明

- `categoryCode`：必填
- `canonicalText`：必填
- `aliases`：选填
- `priority`：选填
- `riskLevel`：选填
- `replaceMode`：选填
- `baseConfidence`：选填
- `sourceType`：选填，默认 `manual`
- `pinyinRuntimeMode`：选填
- `rules`：选填
- `pinyinProfile`：选填

### 响应

返回创建后的聚合词条对象。

## 5.4 `PUT /api/console/terms/{termId}`

### 作用

更新词条主数据、错误词、规则、拼音。

### 请求体

与 `POST /api/console/terms` 相同，但所有字段都可按需部分提交。

### 响应

返回更新后的聚合词条对象。

补充说明：

- 当前控制台已落地 `POST /api/console/terms/{termId}/generate-pinyin-candidates`
- 当前控制台已落地 `POST /api/console/terms/{termId}/pinyin-candidates`
- 这两个接口承接拼音候选生成与提交流程，和 `/api/admin/*` 保持双轨独立

## 5.8 `POST /api/console/terms/batch-submit-review`

### 作用

对多个词条批量提交审核，既支持“当前页勾选 ID”方式，也支持“当前筛选结果”方式。

### 请求体

```json
{
  "termIds": ["term_001", "term_002"],
  "comment": "batch submit review"
}
```

或：

```json
{
  "filters": {
    "query": "工伤",
    "categoryCode": "proper_noun",
    "status": "draft",
    "sourceType": "manual",
    "riskLevel": "medium",
    "sortBy": "updated_at",
    "sortDirection": "desc"
  },
  "comment": "batch submit review"
}
```

### 响应字段

- `total`
- `submittedCount`
- `reusedCount`
- `skippedDisabledCount`
- `items`

补充说明：

- 当前实现支持“当前页勾选”与“当前筛选结果”两种执行方式，但不支持跨页手工累积选择
- 已停用词条会跳过，不会重新拉回审核链路

## 9.4 `POST /api/console/validation-cases/import`

### 作用

直接 JSON 导入样本。

### 请求体

```json
{
  "sourceType": "validation_import",
  "mode": "upsert",
  "items": [
    {
      "caseId": "batch_case_001",
      "description": "工伤认定业务样本",
      "text": "我想了解工商认定的办理流程。",
      "expectedCanonicals": ["工伤认定"],
      "sourceType": "validation_import",
      "notes": "批量导入示例"
    }
  ]
}
```

### 字段说明

- `mode`：支持 `upsert`、`insert_only`
- `sourceType`：批量导入默认来源类型；若单条 `item` 自带 `sourceType`，则优先使用单条值
- `items[]`：样本数组；每条至少需要 `text` 和 `expectedCanonicals`

### 响应字段

- `sourceType`
- `mode`
- `total`
- `createdCount`
- `updatedCount`
- `skippedCount`

补充说明：

- 当前 `/console/validation-cases` 页面仍保留一个 JSON 数组调试导入入口，直接消费本接口
- 正式业务导入建议优先走 `/api/console/import/jobs` + `validation_cases_csv_v1` 模板的文件预览确认链路
