# 后台管理页面模块与操作说明

## 1. 文档目的

本文件用于说明当前 ACDP 原型后台管理页面 `/admin` 的模块划分、模块功能、操作方式和业务作用，便于后续继续优化页面结构、交互和能力。

适用页面：

- `GET /admin`
- `GET /test-client`

## 2. 页面整体结构

当前后台管理页采用单页原型形式，顶部为访问上下文区域，中间为按功能拆分的 sheet/tab，右侧为统一响应结果面板。

当前 sheet 包括：

1. 运行时
2. 词条规则
3. 拼音治理
4. 发布治理
5. 校验样本
6. 审计日志

页面右侧固定有“响应结果”区域，用于显示最近一次接口调用返回值，便于调试和演示。

## 3. 顶部公共模块

### 3.1 Access Context

当前页面最上方提供访问上下文设置。

字段：

- `Operator`
- `Role`

角色选项：

- `dict_admin`
- `dict_editor`
- `dict_reviewer`
- `dict_publisher`
- `dict_operator`
- `dict_viewer`

操作按钮：

- `Current Identity`

如何操作：

1. 在 `Operator` 中填写当前操作人标识
2. 在 `Role` 中选择当前角色
3. 点击 `Current Identity` 查看当前身份和权限集合

作用：

- 用于模拟不同角色的管理行为
- 用于验证 RBAC 权限边界
- 用于演示审核、发布、灰度等动作在不同角色下的差异

## 4. 运行时 Sheet

### 4.1 运行时模拟

用于直接演示“输入 ASR 原始文本，输出修正后的文本”。

字段：

- `测试文本`
- `灰度流量键`

操作按钮：

- `健康检查`
- `当前运行版本`
- `执行模拟`

如何操作：

1. 在 `测试文本` 中输入待测试的 ASR 文本
2. 如需观察灰度分流效果，填写 `灰度流量键`
3. 点击 `执行模拟`
4. 在右侧“响应结果”区域查看：
   - `correctedText`
   - `matches`
   - `candidates`
   - `blocked`

作用：

- 现场演示热词纠错效果
- 验证当前版本是否正确加载
- 验证灰度策略是否生效
- 用于快速回放单条样本

## 5. 词条规则 Sheet

### 5.1 词条编辑

用于新建词条和编辑规则。

字段：

- `类别`
- `优先级`
- `当前目标 termId`
- `标准词`
- `别名列表`
- `规则 JSON`

操作按钮：

- `创建词条`
- `保存规则`
- `刷新词条列表`

如何操作：

1. 选择 `类别`
2. 输入 `标准词`
3. 输入 `别名列表`
4. 需要时调整 `规则 JSON`
5. 点击 `创建词条`
6. 若是已有词条，先在“词条列表”中点 `规则`，系统会自动把 `termId` 填入，再点击 `保存规则`

作用：

- 管理标准词和别名
- 配置命中规则
- 为后续拼音治理、审批和发布提供基础词条

### 5.2 词条列表

用于查看现有词条和执行词条级操作。

每条词条当前支持的动作：

- `详情`
- `送审`
- `直接批准`
- `停用`
- `规则`
- `拼音`

如何操作：

1. 点击 `详情` 查看词条完整信息
2. 点击 `送审` 提交 term review
3. 点击 `直接批准` 把词条状态切到 `approved`
4. 点击 `停用` 停用词条
5. 点击 `规则` 自动切换到当前 sheet，并把规则载入编辑区
6. 点击 `拼音` 自动切换到“拼音治理”sheet，并把拼音画像载入编辑区

作用：

- 作为词条运营主入口
- 支撑词条从草稿到审批再到发布的生命周期管理

## 6. 拼音治理 Sheet

### 6.1 拼音画像编辑

用于维护词条级拼音画像。

字段：

- `拼音运行模式`
- `多音字策略`
- `自定义标准读音`
- `备用读音列表`
- `备注`

操作按钮：

- `保存拼音画像`
- `刷新拼音画像列表`
- `刷新拼音冲突`
- `生成多音字候选`
- `Refresh Compare`

如何操作：

1. 先通过“词条列表”的 `拼音` 按钮载入某个词条
2. 修改拼音画像字段
3. 点击 `保存拼音画像`

作用：

- 人工校正词条读音
- 控制运行时拼音增强行为
- 管理多音字和备用读音

### 6.2 拼音画像列表

用于查看当前已有拼音画像的词条。

操作：

- `载入画像`

作用：

- 快速选择已有画像词条继续编辑

### 6.3 拼音冲突列表与详情

用于查看不同词条共享拼音键的冲突情况。

列表信息包括：

- 冲突类型
- 冲突键
- 严重度
- 分值
- 涉及词条数
- 短词/高重叠提示

操作：

- `查看详情`

详情信息包括：

- 冲突类型
- 严重度
- 原因
- 所有冲突词条列表

作用：

- 发现高风险拼音冲突
- 优先处理短词、高重叠、多词条冲突
- 支撑拼音治理排序

### 6.4 多音字候选

用于为当前词条生成多音字候选，并把候选提交审批。

每个候选当前显示：

- 候选读音
- 来源规则
- 风险等级
- 风险分值
- 冲突提示
- 审批状态

操作：

- `提交候选审批`
- `临时加入备用读音`

如何操作：

1. 先选中一个目标词条
2. 点击 `生成多音字候选`
3. 根据风险和冲突提示决定：
   - 提交审批
   - 或临时加入备用读音后再手工保存

作用：

- 把多音字治理从手工猜测变成半自动流程
- 通过审批流控制风险

### 6.5 拼音 comparison

用于比较默认生成读音与当前人工画像的差异。

列表显示：

- `changed / aligned / default`
- 差异字段数
- 变更字位数

详情显示：

- 默认 full pinyin
- 当前 full pinyin
- alternativeReadings
- 差异字段列表
- 字级差异槽位

作用：

- 发现哪些词条被人工改过
- 解释人工改动具体改了哪个字位

## 7. 发布治理 Sheet

### 7.1 版本管理

用于构建、审批、发布、回滚 release。

字段：

- `构建摘要`
- `灰度百分比`

操作按钮：

- `构建新版本`
- `刷新版本列表`

版本卡片动作：

- `提交发布审批`
- `发布`
- `回滚`
- `挂灰度`

版本卡片还会展示：

- release 状态
- 审批状态
- 是否高风险
- 已审批人数 / 需要审批人数
- gate blocker 数量
- validation case 统计

如何操作：

1. 点击 `构建新版本`
2. 对新版本点击 `提交发布审批`
3. 审核通过后，执行：
   - `挂灰度`
   - 或 `发布`
4. 如有问题，使用 `回滚`

作用：

- 完整管理 release 生命周期
- 展示 release gate 是否阻塞
- 展示是否触发高风险双人审批

### 7.2 灰度策略

用于查看和停用当前灰度策略。

操作：

- `刷新灰度策略`
- `停用`

作用：

- 管理 stable / canary 分流策略
- 回收当前灰度流量

### 7.3 审核任务

用于统一查看 term review、candidate review 和 release review。

操作：

- `刷新审核任务`
- `批准任务`
- `驳回任务`

作用：

- 集中执行审批
- 演示不同任务类型的 review 流程

## 8. 校验样本 Sheet

### 8.1 业务校验样本管理

用于管理 release gate 使用的 validation cases。

字段：

- `caseId`
- `描述`
- `样本文本`
- `期望 canonical 列表`
- `备注`

操作按钮：

- `创建样本`
- `批量导入`
- `扫描 feed 目录`
- `查看 feed 目录`
- `刷新样本列表`

作用：

- 管理业务验证样本
- 支撑 release validation gate
- 作为外部样本回流落地入口

### 8.2 Validation case 列表

每条样本当前显示：

- enabled/disabled 状态
- sourceType
- 描述
- 样本文本
- expected canonicals

操作：

- `停用`

作用：

- 管理哪些样本真正参与 release validation

### 8.3 Feed source 状态

用于查看 file-based connector 的目录状态。

每个 source 当前显示：

- `sourceType`
- pending 文件数
- archived 文件数
- failed 文件数
- inbox 路径
- archive 路径
- error 路径
- pending 文件名
- error 文件名

当前支持的 source：

- `cg3`
- `qa_feedback`
- `online_feedback`

作用：

- 查看是否有待导入样本
- 查看是否存在坏文件
- 排查 feed 导入问题

## 9. 审计日志 Sheet

### 9.1 审计日志

用于回看后台最近的管理动作。

操作：

- `刷新审计日志`

每条日志显示：

- 操作名
- targetType
- operator
- targetId
- createdAt

作用：

- 回溯谁做了什么操作
- 支撑问题排查和演示审计能力

## 10. 响应结果面板

页面右侧固定为“响应结果”面板。

行为：

- 所有按钮点击后的接口返回都展示在此处

作用：

- 调试接口
- 演示接口返回结构
- 辅助定位错误

## 11. 当前页面优化方向建议

如果后续继续优化后台管理页，建议优先考虑：

1. 增加列表筛选和搜索
2. 把审核任务进一步按类型拆分
3. 给 release gate blocker 提供更明显的视觉提示
4. 为 validation feed source 增加错误文件重试/回放操作
5. 给拼音治理增加更强的字级编辑能力
