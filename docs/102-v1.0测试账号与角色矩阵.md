# v1.0 测试账号与角色矩阵

> 口径说明
>
> 本文档中若出现 `内容审核` 旧称，当前统一理解为 `词典审核`。

## 1. 文档目的

本文档用于固定 `JOB-090` 后续测试执行时使用的：

- 测试账号
- 默认角色
- 可切换角色
- 关键验证动作

目标是让后续测试 Codex 不需要再从代码或配置里反查账号和角色。

## 2. 配置来源

- [prototype/config/access_control.json](/Codex/ACDP/prototype/config/access_control.json)

## 3. 测试账号矩阵

| 用户ID | 展示名 | 默认角色 | 可切换角色 | 主要测试用途 |
| --- | --- | --- | --- | --- |
| `console_user` | 控制台试用用户 | `dict_admin` | `dict_admin` / `dict_publisher` / `dict_operator` / `dict_reviewer` / `dict_editor` / `dict_viewer` | 页面入口、导航、帮助、权限切换、中文化总验收 |
| `editor_user` | 词条编辑员 | `dict_editor` | `dict_editor` | 词条新增、导入、提交审核、拼音候选提交 |
| `reviewer_user` | 审核员 | `dict_reviewer` | `dict_reviewer` | 词条审核、拼音审核、发布审核 |
| `reviewer_user2` | 审核员 2 | `dict_reviewer` | `dict_reviewer` | 双审核人 / 高风险发布审核补充验证 |
| `publisher_user` | 发布员 | `dict_publisher` | `dict_publisher` | build、提审、发布 |
| `operator_user` | 运营员 | `dict_operator` | `dict_operator` | 灰度、回滚、运行节点控制 |
| `viewer_user` | 只读查看员 | `dict_viewer` | `dict_viewer` | 只读访问、帮助浏览、无写权限验证 |

## 4. 角色能力矩阵

| 角色 | 关键权限方向 | 建议验证点 |
| --- | --- | --- |
| `dict_viewer` | 只读、模拟、运行查看、帮助查看 | 不能写词典建设数据、不能审核、不能发布 |
| `dict_editor` | 词典建设写入、提交审核、拼音候选提交 | 能新增词条、能提交审核、不能做审核决策 |
| `dict_reviewer` | 内容审核、发布审核决策 | 能审核，不能 build / publish |
| `dict_publisher` | build、提交发布审核、发布 | 能 build / publish，不能审核决策 |
| `dict_operator` | 灰度、回滚、运行节点控制 | 能 rollback、灰度、运行节点控制 |
| `dict_admin` | 全部权限 | 全链路兜底验证 |

## 6. 当前已知注意事项

### 6.1 第二审核员命名差异

配置文件中的第二审核员是：

- `reviewer_user2`

而部分历史测试或脚本中可能出现：

- `reviewer_user_2`

后续进入实际执行前，建议统一一次命名，避免报告和脚本混用两个写法。

### 6.2 header 模拟与配置用户的区别

当前部分测试通过 `x-role` / `x-operator` 直接模拟身份。

若后续测试目标要求“严格按配置用户矩阵验证”，则应优先使用本文件中的用户 ID，而不是随意自定义 `x-operator`。
