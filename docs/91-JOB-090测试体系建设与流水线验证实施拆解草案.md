## 2. JOB-090 目标

`JOB-090` 的目标是：

**在不立即开启大规模测试执行的前提下，先把 `v1.0` 测试体系、测试资源、测试数据、测试证据和执行流水线设计完整，为后续 Codex 主导测试与修正做准备。**

## 3. 当前边界

当前 `JOB-090` 不要求立刻完成全部真实测试。

当前阶段先完成：

- 测试目标收口

- 数据准备要求收口

## 6. 执行顺序建议

建议按 7 个测试阶段推进：

### T01 测试基线与一致性

- 真源与派生文档一致性
- 帮助目录与 slug 一致性
- 配置文件与说明一致性

### T02 域级单元与接口回归

- 主数据域
- 内容审核域
- 发布域
- 运行治理域
- 系统配置域
- 帮助域

### T03 工作流与页面回归

- 词条到发布
- 导入到发布
- 节点备案到运行验证
- 发布到回滚

### T04 部署与运行验证

- admin / runtime 启动
- 多 runtime 收敛
- `file / admin_http_signed / minio` 三种运行快照下发模式验证
- 节点状态与页面一致性

### T05 性能与容量验证

- HTTP 纠错
- WebSocket 纠错
- 并发与吞吐

### T06 发布演练

- build
- review
- canary
- publish
- rollback

### T07 证据包与 go/no-go

- 结果汇总
- 证据包归档
- go / no-go 草稿
