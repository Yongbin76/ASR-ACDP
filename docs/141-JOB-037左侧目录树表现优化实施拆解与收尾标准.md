# JOB-037 左侧目录树表现优化实施拆解与收尾标准

## 1. 目标

本文档用于把 [140-v1.0左侧目录树表现优化方案](./140-v1.0左侧目录树表现优化方案.md) 细化为可直接编码的实施任务。

目标是：

- 把当前后台左侧导航从“分组链接列表”进一步收口为“更清晰的目录树”
- 优化层级感、交互稳定性、收起态可读性和当前页定位
- 同时为后续配置驱动打基础

## 2. 范围

### 2.1 本批次要做

- 左侧目录树的结构表现优化
- 一级 / 二级层级视觉优化
- 当前激活态和分组激活态优化
- 收起态体验优化
- 导航结构配置化第一步

### 2.2 本批次不做

- 不重构整个后台壳层
- 不改业务页面内部布局
- 不在本批次顺手重命名所有页面
- 不引入新的前端框架

## 3. 命名基线

本批次目录名调整统一以 [140-v1.0左侧目录树表现优化方案](./140-v1.0左侧目录树表现优化方案.md) 中“命名冻结表”为准。

必须执行的命名调整：

- `工作台 / 工作台` -> `工作台 / 总览`
- `运行治理 / 运行治理` -> `运行治理 / 总览`
- 一级 `系统配置` -> `系统管理`
- 二级：
  - `系统配置` -> `总览`
  - `用户` -> `用户管理`
  - `角色` -> `角色管理`
  - `权限` -> `权限管理`

## 4. 代码边界

主要文件：

- `console/client/index.html`
- `console/client/app.css`
- `console/client/app.js`

如需配置化：

- 可新增 `prototype/config/console_navigation.json`
- 或新增更轻量的导航配置源

## 5. 任务拆解

### T01 结构梳理

- [x] 明确一级目录和二级目录的数据结构
- [x] 让当前路径所在分组具备更明确的展开规则
- [x] 清理当前硬编码树结构中不必要的重复
- [x] 按命名冻结表同步一级 / 二级目录名称

### T02 视觉层级优化

- [x] 强化一级目录的树根感
- [x] 强化二级目录的树节点感
- [x] 收口一级 / 二级字体、缩进、间距
- [x] 强化当前激活项与激活分组
- [x] 优化左侧背景色，降低与右侧内容区的视觉冲突

### T03 收起态优化

- [x] 重新定义收起态下保留哪些信息
- [x] 让收起态不至于完全失去导航意义
- [x] 保证收起 / 展开切换后布局稳定
- [x] 收起态只保留图标，不再出现残留文字或边框干扰
- [x] 收起态 hover / focus 可显示一级目录名称提示

### T04 交互稳定性

- [x] 固定一级点击为展开/折叠语义
- [x] 固定二级点击为跳转语义
- [x] 优化本地缓存与当前路径自动展开的优先级

### T05 配置化第一步

- [x] 提取当前左侧导航数据源
- [x] 让结构、label、path、help slug 不再只散落在 HTML 中
- [x] 为后续权限控制和新增目录预留字段

### T06 文档与回归

- [x] 更新帮助或 UI 规范文档
- [x] 补前端渲染与导航交互相关回归
- [x] 同步真源与索引

## 6. 完成标准

只有以下条件同时满足，`JOB-037` 才可关闭：

- 左侧目录树层级明显优于当前版本
- 当前页所在分组与当前项一眼可见
- 收起态仍保留最小可识别性
- 收起态只显示图标，且 hover / focus 可恢复一级目录可读性
- 展开 / 折叠行为稳定
- 左侧背景色不再显得过深、过重
- 导航结构不再完全依赖硬编码 HTML
- 文档、真源、回归全部同步
