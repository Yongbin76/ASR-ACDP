# 拆分部署资产使用说明

## 1. 目的

本文件用于说明 runtime/admin 拆分后的基础容器与 K8S 资产如何使用。

若需要完整的安装、配置、启动、检测、停止手册，请直接参考：

- [37-Runtime-Admin服务运维手册](./37-runtime-admin服务运维手册.md)

说明：

- 当前保留现有单体 `Dockerfile` 与 `k8s/deployment.yaml`
- 新增拆分资产仅用于 phase C 验证和后续独立部署演进

## 2. 新增容器资产

- `Dockerfile.runtime`
- `Dockerfile.admin`

建议构建命令：

```bash
cd /Codex/ACDP
docker build -f Dockerfile.runtime -t acdp-runtime:latest .
docker build -f Dockerfile.admin -t acdp-admin:latest .
```

默认端口：

- runtime: `8787`
- admin: `8788`

## 3. 新增 K8S 资产

- `k8s/runtime-deployment.yaml`
- `k8s/runtime-service.yaml`
- `k8s/admin-deployment.yaml`
- `k8s/admin-service.yaml`
- `k8s/artifact-store-secret.example.yaml`

依赖仍使用现有：

- `k8s/namespace.yaml`
- `k8s/pvc.yaml`

建议应用顺序：

```bash
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/pvc.yaml
kubectl apply -f k8s/artifact-store-secret.example.yaml
kubectl apply -f k8s/runtime-deployment.yaml
kubectl apply -f k8s/runtime-service.yaml
kubectl apply -f k8s/admin-deployment.yaml
kubectl apply -f k8s/admin-service.yaml
```

## 4. 当前约束

当前拆分部署仍有以下约束：

- runtime/admin 仍共享同一个 `prototype/workspace` PVC
- admin 容器启动时仍会执行 `npm run setup:prototype`
- runtime 容器当前默认只执行 `npm run start:runtime`
- 当前仍属于 phase C 基础版资产，不代表生产级最终形态

新增说明：

- `runtime-deployment.yaml` / `admin-deployment.yaml` 已预留 artifact store 和 runtime token 的环境变量注入口
- 当前建议通过 `k8s/artifact-store-secret.example.yaml` 派生真实 Secret，再应用 deployment

## 5. 推荐验证

容器或 K8S 启动后，建议分别执行：

```bash
npm run smoke:runtime
npm run smoke:admin
```

并继续保留：

```bash
npm run verify:host:console
```
