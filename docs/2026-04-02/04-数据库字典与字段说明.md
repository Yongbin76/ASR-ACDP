# ACDP 数据库字典与字段说明

## 1. 文档目的

本文档用于基于当前代码实现，整理 ACDP 字典系统涉及的全部实际 SQLite 数据库、表结构与字段说明。

本文件以以下代码为准：

- `prototype/src/lib/platform-db.js`
- `prototype/src/lib/runtime-stats.js`

适用范围：

- 控制面主库：`prototype/workspace/platform.db`
- runtime 本地统计库：`prototype/workspace/runtime_state/runtime_stats.db`

说明：

- 文档中的“类型”以当前 SQLite DDL 声明为准。
- 文档中的“说明”以当前代码实际用途为准，不回写旧设计文档中的理想态表述。
- `import_templates` 虽然在 `platform.db` 中建表，但当前模板元数据的实际读取入口仍是 `prototype/config/import_templates/index.json`，不是这张表。

## 2. 数据库总览

| 数据库文件 | 用途 | 当前来源代码 |
|---|---|---|
| `prototype/workspace/platform.db` | 控制面主库，承载词条、别名、规则、拼音画像、导入、审核、发布、验证样本、运行节点、统计汇总、审计 | `prototype/src/lib/platform-db.js` |
| `prototype/workspace/runtime_state/runtime_stats.db` | runtime 节点本地统计缓冲库，承载事件、游标状态、峰值并发 | `prototype/src/lib/runtime-stats.js` |

## 3. `platform.db` 数据字典

### 3.1 词典主数据

#### `terms`

用途：词条主表，保存标准词的核心业务属性。

| 字段 | 类型 | 说明 |
|---|---|---|
| `term_id` | `TEXT` | 词条主键，系统内部唯一 ID。 |
| `category_code` | `TEXT` | 词条类别编码，例如 `poi_road`、`gov_term`、`proper_noun`。 |
| `canonical_text` | `TEXT` | 标准词文本。 |
| `status` | `TEXT` | 词条当前状态，如草稿、待审、已通过、停用等。 |
| `priority` | `INTEGER` | 词条优先级，影响排序和运行时命中优先级。 |
| `risk_level` | `TEXT` | 风险等级，用于审核、发布和高风险动作控制。 |
| `replace_mode` | `TEXT` | 运行时处理模式，如直接替换、候选提示、拦截等。 |
| `base_confidence` | `REAL` | 基础置信度，作为运行时纠错决策的基础参数。 |
| `source_type` | `TEXT` | 词条来源标记，如手工维护、导入、seed、外部反馈等。 |
| `pinyin_runtime_mode` | `TEXT` | 拼音相关运行模式。 |
| `revision` | `INTEGER` | 词条版本号，每次更新递增。 |
| `created_at` | `TEXT` | 创建时间，ISO 时间字符串。 |
| `updated_at` | `TEXT` | 最后更新时间，ISO 时间字符串。 |

#### `aliases`

用途：词条别名/错误词表。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `INTEGER` | 自增主键。 |
| `term_id` | `TEXT` | 关联的词条 ID，外键指向 `terms.term_id`。 |
| `alias_text` | `TEXT` | 错误词、别名或同义表达。 |

#### `term_sources`

用途：记录词条的来源追踪信息，用于把正式词条和导入批次、源文件、源行号关联起来。

| 字段 | 类型 | 说明 |
|---|---|---|
| `term_id` | `TEXT` | 词条 ID，同时作为主键。 |
| `source_type` | `TEXT` | 来源类型。 |
| `import_job_id` | `TEXT` | 若来自导入批次，记录对应导入批次 ID。 |
| `source_file_name` | `TEXT` | 来源文件名。 |
| `source_row_no` | `INTEGER` | 来源文件中的行号。 |
| `source_ref` | `TEXT` | 来源引用标识，通常用于拼出稳定的批次内来源定位。 |
| `created_at` | `TEXT` | 创建时间。 |
| `updated_at` | `TEXT` | 更新时间。 |

#### `alias_sources`

用途：记录别名的来源追踪信息。

| 字段 | 类型 | 说明 |
|---|---|---|
| `term_id` | `TEXT` | 所属词条 ID。 |
| `alias_text` | `TEXT` | 别名文本。 |
| `source_type` | `TEXT` | 来源类型。 |
| `import_job_id` | `TEXT` | 若来自导入批次，记录对应导入批次 ID。 |

## 4. `runtime_stats.db` 数据字典

### 4.1 说明

`runtime_stats.db` 是 runtime 节点本地统计缓冲库，不是控制面主库。

用途：

- 本地记录每次纠错调用形成的统计事件
- 记录本地上传游标状态
- 记录节点本地峰值并发

当前代码位置：

- `prototype/src/lib/runtime-stats.js`

### 4.2 表结构

#### `runtime_stats_events`

用途：runtime 本地事件明细表。

| 字段 | 类型 | 说明 |
|---|---|---|
| `event_id` | `INTEGER` | 自增事件 ID。 |
| `hour_key` | `TEXT` | 小时分桶键。 |
| `channel` | `TEXT` | 请求通道，如 `http`、`ws`。 |
| `hits_json` | `TEXT` | 当前请求命中的标准词列表 JSON。 |
| `created_at` | `TEXT` | 事件写入时间。 |

#### `runtime_stats_state`

用途：runtime 本地状态键值表。

| 字段 | 类型 | 说明 |
|---|---|---|
| `state_key` | `TEXT` | 状态键名。 |
| `state_value` | `TEXT` | 状态值。 |
| `updated_at` | `TEXT` | 更新时间。 |

当前主要用途包括：

- `last_uploaded_event_id`
- `last_uploaded_peak`

#### `runtime_stats_peak`

用途：runtime 本地峰值并发记录表。

| 字段 | 类型 | 说明 |
|---|---|---|
| `stat_key` | `TEXT` | 统计主键，当前通常固定为 `global`。 |
| `peak_concurrency` | `INTEGER` | 节点本地峰值并发。 |
| `peak_at` | `TEXT` | 峰值出现时间。 |
| `updated_at` | `TEXT` | 更新时间。 |

## 5. 当前数据库层面的关键边界

### 5.1 词典主数据模型是统一的

当前没有“路名专属表”“政府部门专属表”“结构化词条专属表”。

这三类词条最终都统一进入：

- `terms`
- `aliases`
- `term_rules`
- `pinyin_profiles`
- `term_sources`
- `alias_sources`

也就是说，导入模板是导入层差异，不是底层存储层差异。

### 5.2 导入模板和导入批次是两层概念

- `import_jobs` / `import_job_rows` / `import_job_results` 负责“本次导入过程”
- `import_templates` 负责“模板元信息”

但当前模板元数据实际真源仍是：

- `prototype/config/import_templates/index.json`

因此当前 `import_templates` 更接近“预留/兼容性 schema”，不是当前控制台真正读取的模板来源。

### 5.3 runtime 统计是双层存储

- `runtime_stats.db`：节点本地缓冲
- `platform.db` 中的 `runtime_node_*` / `runtime_*`：控制面汇总后的统计结果

因此这两套表不能混为一个数据库理解。
