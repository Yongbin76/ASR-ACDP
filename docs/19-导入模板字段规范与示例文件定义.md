# 导入模板字段规范与示例文件定义

## 1. 文档目的

本文档用于定义当前 ACDP 导入中心实际支持的模板资产、字段规范、示例文件和下载口径。

当前基线已经调整为：

- 词条导入主路径统一为一套结构化词条导入合同
- 路名 / 政府部门 / 常用词差异通过业务属性承接
- 旧版路名模板、政府部门模板、结构化词条 v1 模板继续保留兼容，但不再作为主推荐路径

本文档与以下文档配套使用：

- [17-后台管理系统数据与接口改造方案](./17-后台管理系统数据与接口改造方案.md)
- [18-后台管理系统首批接口清单与字段定义](./18-后台管理系统首批接口清单与字段定义.md)
- [48-导入中心统一结构化词条导入重构方案](./48-导入中心统一结构化词条导入重构方案.md)
- [49-统一词条准入规则与跨中心收口方案](./49-统一词条准入规则与跨中心收口方案.md)

说明：

- 本文档主要定义模板合同、字段与下载资产
- “哪些词条允许进入字典、哪些词条必须阻断”相关统一准入规则，后续以 `2026-04-07` 文档为实现基线

## 2. 基本原则

### 2.1 上传前必须可下载模板

所有导入类型在上传前必须提供：

- 模板下载
- 字段说明
- 示例文件下载
- 常见错误说明

### 2.2 模板必须有版本

每个模板都必须有：

- `templateCode`
- `templateVersion`

### 2.3 模板说明必须与最终落库模型一致

模板中出现的字段，应当满足以下要求之一：

1. 会真实进入当前数据库词典建设模型
2. 仅作为 legacy 兼容输入存在，并在文档中明确标注为兼容字段

### 2.4 当前主路径统一采用 CSV

当前正式主路径统一采用：

- `.csv`
- `UTF-8`

说明：

- `.txt` 当前只保留给 `raw_roads_text_v1` 兼容模板
- `json` 不属于当前导入模板下载资产

## 4. 主路径模板：`structured_terms_csv_v2`

### 4.1 使用场景

适用于以下所有词条导入主路径：

- 路名
- 政府部门
- 常用词

当前不再为上述三类分别维护三套主模板。

### 4.2 页面侧业务属性

创建导入批次时，页面会先让用户选择一次业务属性：

- `路名` -> `poi_road`
- `政府部门` -> `gov_term`
- `常用词` -> `proper_noun`

若文件行中未填写 `categoryCode`，系统将沿用批次级默认业务属性。

### 4.3 字段定义

| 字段名 | 中文名 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| `canonicalText` | 标准词 | 是 | 标准词文本 | `工伤认定` |
| `aliases` | 错误词列表 | 否 | 多个值用 `|` 分隔 | `工商认定|工伤认丁` |
| `categoryCode` | 类别编码 | 否 | 可为空；为空时沿用批次级业务属性 | `gov_term` |
| `priority` | 优先级 | 否 | 默认 `80` | `92` |
| `riskLevel` | 风险等级 | 否 | `low/medium/high` | `medium` |
| `replaceMode` | 替换模式 | 否 | `replace/candidate/block` | `replace` |
| `baseConfidence` | 基础置信度 | 否 | `0~1` | `0.94` |
| `pinyinRuntimeMode` | 拼音运行模式 | 否 | `off/candidate/replace` | `candidate` |
| `customFullPinyinNoTone` | 自定义全拼 | 否 | 人工指定全拼，无声调 | `gong shang ren ding` |
| `alternativeReadings` | 备用读音 | 否 | 多个值用 `|` 分隔 | `gong shang ren ding` |
| `sourceType` | 来源类型 | 否 | 默认 `import_csv` | `import_csv` |
| `remark` | 备注 | 否 | 附加说明 | `示例` |

### 4.4 示例文件

```csv
canonicalText,aliases,categoryCode,priority,riskLevel,replaceMode,baseConfidence,pinyinRuntimeMode,customFullPinyinNoTone,alternativeReadings,sourceType,remark
工伤认定,工商认定,gov_term,92,medium,replace,0.94,candidate,,,import_csv,政务词条示例
祁顺路,,poi_road,80,medium,replace,0.90,candidate,,,import_csv,路名示例
婚姻介绍所,婚姻检查所|婚因介绍所,proper_noun,85,medium,replace,0.93,candidate,,,import_csv,常用词示例
```

### 4.5 导入结果

确认导入后，系统将形成：

- `terms`
- `aliases`
- `term_sources`
- `alias_sources`
- 如提供拼音画像字段，则同步更新 `pinyin_profiles`

说明：

- 不再为政府部门保留主路径专属字段
- 不再为路名保留主路径专属模板

## 5. 错误词补录模板：`term_aliases_csv_v1`

### 5.1 使用场景

适用于已有标准词条，希望单独批量补充错误词/别名。

### 5.2 字段定义

| 字段名 | 中文名 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| `categoryCode` | 类别编码 | 是 | 用于辅助定位词条 | `gov_term` |
| `canonicalText` | 标准词 | 是 | 用于定位词条 | `工伤认定` |
| `aliasText` | 错误词/别名 | 是 | 需要新增的别名 | `工商认定` |
| `sourceType` | 来源类型 | 否 | 默认 `import_alias` | `qa_feedback` |
| `remark` | 备注 | 否 | 说明 | `QA 汇总` |

### 5.3 示例文件

```csv
categoryCode,canonicalText,aliasText,sourceType,remark
gov_term,工伤认定,工商认定,qa_feedback,示例
proper_noun,婚姻介绍所,婚姻检查所,online_feedback,示例
```

### 5.4 导入结果

确认导入后，系统将：

- 先按 `categoryCode + canonicalText` 定位 `terms`
- 再把 `aliasText` 写入 `aliases`

## 6. 验证样本模板：`validation_cases_csv_v1`

### 6.1 使用场景

适用于批量导入 QA、线上反馈、业务回放样本。

### 6.2 字段定义

| 字段名 | 中文名 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| `caseId` | 样本 ID | 否 | 建议提供稳定唯一值 | `qa-feedback-001` |
| `description` | 样本说明 | 否 | 样本描述 | `工伤认定误识别样本` |
| `text` | 原始文本 | 是 | 业务文本 | `我想了解工商认定的办理材料。` |
| `expectedCanonicals` | 期望标准词 | 是 | 多个值用 `|` 分隔 | `工伤认定` |
| `sourceType` | 来源类型 | 否 | 如 `qa_feedback` | `qa_feedback` |
| `notes` | 备注 | 否 | 附加说明 | `reported by qa replay` |

### 6.3 示例文件

```csv
caseId,description,text,expectedCanonicals,sourceType,notes
qa-feedback-001,工伤认定误识别样本,我想了解工商认定的办理材料。,工伤认定,qa_feedback,示例
online-feedback-001,婚姻介绍所误识别样本,附近有婚姻检查所吗？,婚姻介绍所,online_feedback,示例
```

### 6.4 导入结果

导入后写入：

- `validation_cases`

不写入：

- `terms`
- `aliases`

## 7. 兼容模板说明

### 7.1 `structured_terms_csv_v1`

旧版结构化词条模板仍可继续导入，但新导入应优先使用：

- `structured_terms_csv_v2`

### 7.2 `raw_roads_text_v1`

旧版逐行路名文本模板仍保留兼容，系统会将其标准化后再进入词条导入预览。

新导入不再推荐使用该模板作为主路径。

### 7.3 `gov_departments_csv_v1`

旧版政府部门模板仍保留兼容，但其中以下字段不再被视为词条系统主合同字段：

- `district`
- `address`
- `level`
- `category`
- `remark`

当前真正持续进入词条系统核心模型的只有：

- `standardName`
- `shortName`

它们会被映射为：

- `canonicalText`
- `aliases`

## 8. 字段级通用规则

### 8.1 文本清洗

所有文本字段建议在导入前统一执行：

- 去前后空格
- 去 BOM
- 连续空白收敛

### 8.2 多值字段

首批约定统一使用：

- `|` 作为单元格内多值分隔符

适用于：

- `aliases`
- `alternativeReadings`
- `expectedCanonicals`

### 8.3 必填校验

若缺必填字段：

- 进入导入错误行
- 不进入正式词典数据

### 8.4 枚举校验

以下字段要做值域校验：

- `categoryCode`
- `riskLevel`
- `replaceMode`
- `pinyinRuntimeMode`
