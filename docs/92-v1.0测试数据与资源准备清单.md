# v1.0 测试数据与资源准备清单

> 口径说明
>
> 本文档中若出现 `主数据` 旧称，当前统一理解为 `词典建设 + 验证与回流` 的测试数据准备。

## 1. 文档目的

本文档用于把 `v1.0` 测试执行前需要准备的数据、账号、密钥、环境、工具和证据目录明确下来。

目标是让后续其他 Codex、测试执行人或运维在进入测试阶段时，不需要再临时猜：

- 要准备什么数据
- 要向谁要什么权限
- 哪些资源是本地就能做的
- 哪些资源属于外部条件

## 2. 总体原则

### 2.1 数据可重复

- 尽量使用固定样本和固定命名
- 同一批测试尽量复用同一组基准数据

### 2.2 数据可隔离

- 测试用数据不要污染正式业务主数据
- 测试用节点、测试用 release、测试用样本建议统一前缀

### 2.3 数据可留档

- 关键输入、关键输出、关键截图、关键报告都要能归档到证据包

## 3. 基础账号与权限准备

至少准备以下账号或身份模拟对象：

- 编辑
- 审核
- 发布
- 运维
- 管理员
- 只读查看

建议准备内容：

- 用户名
- 角色
- 可访问页面
- 关键限制说明

建议验证范围：

- 用户 / 角色 / 权限配置是否一致
- 治理策略是否对不同角色产生预期限制

## 4. 主数据测试准备

## 4.1 词条样本

建议准备：

- 正常标准词
- 单字词阻断样本
- alias 冲突样本
- `poi_road` 样本
- `gov_term` 样本
- 拼音主读音覆写样本
- 多音字备用读音样本

建议覆盖：

- 新建
- 编辑
- 审核
- 进入发布输入池

## 4.2 导入样本

建议至少准备：

- 结构化词条导入样本
- 错误词批量补录样本
- validation case 导入样本
- 含错误行的阻断样本
- legacy 兼容样本

建议留档：

- 原始导入文件
- 导入预览截图
- 导入结果摘要

## 4.3 样本与回流

建议准备：

- 正常 validation case
- 期望命中样本
- 期望不命中样本
- 发布后风险触发样本

## 5. 发布测试准备

建议准备：

- 至少 1 个可构建 release 的输入池
- 至少 1 个可用于回滚的已发布 release
- 至少 1 组用于灰度验证的 trafficKey

建议覆盖：

- build
- release review
- canary
- publish
- rollback
- 发布后风险查看

## 6. 运行治理测试准备

## 6.1 节点备案样本

建议准备：

- 2~3 个 runtime 节点备案
- 每个节点的：
  - `nodeId`
  - `nodeName`
  - `env`
  - `nodeAddress`
  - `registration-secret`

## 6.2 节点异常样本

建议准备：

- 地址不匹配
- `registration-secret` 不匹配
- token 缺失
- 已备案但未注册
- 已注册但离线
- 已注册但版本不对齐

## 6.3 运行验证样本

建议准备：

- `correct`
- `correct_cand`
- `ws correct`
- `ws correct_cand`

的固定输入文本和预期行为清单。

## 7. 真实环境资源准备

## 7.1 部署资源

如需推进 `JOB-006`，至少需要：

- `kubeconfig`
- 目标 namespace
- 镜像仓库地址
- 镜像拉取凭据
- 目标 ingress / service 约束
- 目标 deployment 方式说明

## 7.2 压测资源

如需推进 `JOB-009`，至少需要：

- 真实宿主机
- 可访问的 runtime base URL
- 可用压测时间窗口
- 是否需要 bearer token
- 是否允许长连接压测

## 7.3 外部接口资源

如需推进外部系统联调或真实节点接入，可能还需要：

- `ACDP_RUNTIME_TOKEN`
- `registration-secret`
- 外部 endpoint
- auth 凭据
- ack / replay 测试条件

## 8. 工具准备建议

建议分类准备：

### 8.1 当前仓库已有

- `pm:check`
- `smoke:console`
- `test:console`
- `test:unit`
- `verify:multi-runtime`
- `test:concurrency`

### 8.2 资源授权后可补

- 更强的 e2e 浏览器驱动
- 性能测试结果格式化工具
- 证据包自动汇总工具

## 9. 证据目录准备

测试执行时建议统一归档到：

- `prototype/workspace/host_verification/`

正式发布准备阶段建议使用：

- [85-R7证据包目录与go-no-go模板](./85-R7证据包目录与go-no-go模板.md)

## 10. 建议的准备优先级

### P0

- 账号与角色矩阵
- 词条 / 导入 / validation 测试样本
- 节点备案样本
- 真实部署与压测资源说明

### P1

- 证据目录准备
- 帮助文档与接口文档校验清单
- 发布与回滚演练输入

### P2

- 扩展工具
- 更复杂的压力模型
- 更复杂的异常注入数据
