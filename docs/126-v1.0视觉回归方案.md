# v1.0 视觉回归方案

## 1. 文档目的

本文档用于把 `JOB-090` 中原本待选的“视觉回归方案”收口为一套可执行的最小基线。

目标不是在 `v1.0` 阶段一次性上完整像素级平台，而是先做到：

- 截图命名固定
- 基线 manifest 固定
- 自动比对入口固定
- 报告结构固定

## 2. 当前策略

当前视觉回归采用：

1. 真实宿主机截图作为候选输入
2. 基线截图目录 + manifest 作为基准
3. 以文件哈希和文件大小做最小可执行比对

当前不做：

- 像素级 diff 热力图
- 浏览器云端截图服务
- 复杂视觉阈值算法

## 5. 自动检查入口

当前已提供 CLI：

- [prepare-visual-baseline.js](/Codex/ACDP/prototype/src/cli/prepare-visual-baseline.js)
- [check-visual-regression.js](/Codex/ACDP/prototype/src/cli/check-visual-regression.js)

### 5.1 首次生成基线 manifest

```bash
cd /Codex/ACDP
npm run prepare:visual-baseline -- \
  --screenshot-dir ./prototype/workspace/host_verification/<report>/screenshots \
  --output-manifest ./prototype/tests/fixtures/visual_regression/console/baseline.manifest.json
```

说明：

- 当前会自动按固定截图文件名清单寻找已存在截图
- 如果只想基于特定文件列表生成，可后续扩展 `--files-json`

### 5.2 比对候选截图

对应 npm 入口：

```bash
cd /Codex/ACDP
npm run check:visual-regression -- \
  --baseline-manifest ./prototype/tests/fixtures/visual_regression/console/baseline.manifest.json \
  --candidate-dir ./prototype/workspace/host_verification/<report>/screenshots \
  --write-actual-manifest ./prototype/workspace/host_verification/<report>/screenshots.actual.manifest.json
```

## 6. 输出结果

当前输出 JSON 包括：

- `ok`
- `candidateDir`
- `comparedCount`
- `missing`
- `changed`
- `unchanged`
- 可选 `actualManifestPath`
