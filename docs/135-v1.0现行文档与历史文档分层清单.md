# v1.0 现行文档与历史文档分层清单

> 口径说明
>
> 本文档中会显式提到 `主数据 / 内容审核` 等旧称，是为了做现行/历史文档分层说明。
>
> 这些旧称本身不再代表当前现行页面和接口名称。

## 1. 文档目的

本文档用于把当前 `docs/` 目录中的文档按“现行编码基线”和“历史讨论基线”分层，避免后续 Codex 在编码时误把旧口径文档当成当前真源。
