## 1. 目标

把当前仍处于“可复用但需重构”的 `/console` 页面收口到更接近 `v1.0` 成品的职责边界。

## 2. 当前重点页面

需要重点收口的页面：

- `/console/reviews`
- `/console/releases`
- `/console/releases/:releaseId`
- `/console/help`
- `/console/help/:slug`
- `/console/system`

## 3. 主要问题

### 3.1 内容审核页

当前问题：

- 仍是统一“内容审核”页
- 未显性拆到：
  - 词条审核
  - 拼音审核

### 3.2 版本发布页

当前问题：

- `/console/releases` 仍然更像“发布中心总页”
- 发布审核 / 灰度 / 发布后风险 / 回滚记录仍主要作为区块存在
- 还不够像“二级工作页体系”

### 3.3 帮助页

当前问题：

- 帮助首页仍偏索引页
- 单篇帮助仍偏通用壳
- 页面 / 流程 / 部署 / 接口 / 排障的分组还不够成品化

### 3.4 系统配置首页

当前问题：

- `/console/system` 还是一个泛入口
- 用户 / 角色 / 权限 / 治理策略虽已存在，但首页层级和指引还不够稳定

## 4. 文件范围

- `console/client/app.js`
- `console/client/index.html`
- `console/client/app.css`
- 相关帮助文档与说明文档
