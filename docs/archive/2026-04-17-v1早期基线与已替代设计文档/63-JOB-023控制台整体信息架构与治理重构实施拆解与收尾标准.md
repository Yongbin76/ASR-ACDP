## 2. JOB-023 范围定义

### 2.1 本批目标

围绕当前 `/console` 做一次系统级重构，统一以下能力：

1. 信息架构
2. 主数据管理归口
3. 内容审核边界
4. 版本发布生命周期
5. 系统管理归口
6. RBAC 与治理策略配置化
7. 中文化与术语统一
8. 配置驱动与编码规范

### 2.2 本批不做

本批不包含：

- 替换 `/admin`
- 多 admin 高可用
- 多环境 release deployment matrix 的完整实现
- 灰度算法重写
- 复杂审批编排引擎

## 3. 交付原则

### 3.1 不再做局部页面 patch

本批是系统级重构，不允许继续以“哪个页面乱了就补哪个页面”为主思路。

必须先收口：

- 领域边界
- 一级导航
- 配置模型
- 状态语义

再进入页面实现。

### 3.2 配置驱动，不写散落硬编码

要求：

- 业务属性
- 用户
- 角色
- 权限
- 治理策略
- 中文术语

后续都不得继续散落硬编码在前后端逻辑中。

### 3.3 函数注释作为硬约束

后续所有新增或重写函数必须带注释，至少包括：

- 作用
- 输入
- 输出

### 3.4 中文化与术语一致性作为验收项

本批不能把中文化当成附属优化。

它是关单标准的一部分。

## 5. 收尾要求

`JOB-023` 收尾时必须明确满足以下要求：

1. 一级导航已按新信息架构重组
2. 主数据管理、内容审核、版本发布、样本与验证、系统管理边界清晰
3. 词条中心与导入中心已归口到主数据管理
4. 审核中心只保留内容审核主线
5. 发布中心已清晰表达 build / canary / published / 发布后风险
6. 节点备案与运行节点已归口到系统管理
7. 用户、角色、权限、治理策略已可配置，不再依赖散落硬编码
8. 页面主文案中文化
9. 关键术语统一
10. 新增函数有规范注释
11. 配置项改动通过配置种子或配置文件承接，不散落硬编码

## 6. 关单标准

`JOB-023` 只有在以下条件全部满足时才能关闭：

1. 新信息架构已落地
2. 业务属性已配置化，并被词条中心与导入中心统一消费
3. release publish review 已迁入发布中心内部流程
4. 系统管理已承接节点备案、运行节点、用户、角色、权限、治理策略
5. 中文化与术语统一已完成首轮收口
6. 关键页面和主链路测试通过
7. `smoke:console`、`test:console`、`test:unit` 通过
8. 文档与单一真源已同步
