## 2. R1 目标

`R1` 的目标是：

**以“已备案节点”为主对象，重建运行治理域。**

这意味着：

- 运行节点中心主列表只展示已备案节点
- 未备案 runtime 只作为异常接入处理
- 实时状态、目标状态、最近动作结果、历史异常彻底分层
- 节点部署说明、注册说明、排查说明形成一致链路

## 6. 收尾标准

`R1` 只有在以下条件全部满足时才能关单：

1. 运行节点中心主列表只展示已备案节点
2. 已备案未注册节点在主列表中可见且状态明确
3. 未备案 runtime 不再进入主列表
4. 节点详情页不再混读实时状态和历史状态
5. 节点部署说明、token、secret、指纹说明清晰
6. 运行治理相关回归通过
