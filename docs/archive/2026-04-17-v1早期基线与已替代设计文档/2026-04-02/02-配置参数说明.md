# 配置参数说明

## 1. 说明

当前 ACDP 代码直接使用 `JSON.parse()` 读取以下配置文件：

- `prototype/config/app.config.json`
- `prototype/config/artifact_store.config.json`
- `prototype/config/validation_feed_connectors.config.json`

因此这两个 JSON 文件保持无注释格式，以保证运行时兼容性。参数说明统一记录在本文档中。

## 2. `app.config.json`

### 2.1 `project`

- `name`
  - 项目名称标识。
- `encoding`
  - 项目默认文本编码，当前用于原型数据文件读写约定。

### 2.2 `server`

- `host`
  - 单进程原型默认监听地址。
- `port`
  - 单进程模式默认端口。
- `runtimePort`
  - runtime split 进程默认端口。
- `adminPort`
  - admin split 进程默认端口。
- `urlBaseForParsing`
  - 服务端在需要把相对 URL 转成可解析 URL 时使用的基地址。

### 2.3 `auth`

- `runtimeBearerToken`
  - runtime 接口 Bearer Token 的静态值；生产环境通常不直接写入仓库。
- `runtimeBearerTokenEnv`
  - Bearer Token 的环境变量名，优先级高于 `runtimeBearerToken`。
- `websocketMaxConnections`
  - 运行时 WebSocket 最大并发连接数。
- `websocketIdleTimeoutMs`
  - WebSocket 空闲超时时间。
- `websocketMaxMessageBytes`
  - 单条 WebSocket 消息大小上限。
- `websocketCallerIdHeader`
  - WebSocket callerId 读取用的 header 名。
- `websocketCallerSecretHeader`
  - WebSocket caller secret 读取用的 header 名。
- `websocketCallerIdQueryKey`
  - WebSocket callerId 读取用的 query 参数名。
- `websocketCallerSecretQueryKey`
  - WebSocket caller secret 读取用的 query 参数名。
- `websocketCallerIpHeader`
  - 解析 caller 源地址时优先读取的代理 header 名。
- `websocketRejectUnknownCallers`
  - 启用后，未注册 caller 不能再回退到 legacy caller 路径；若 caller 注册表非空，未提供 `callerId` 也会被拒绝。
- `websocketDefaultMaxConnectionsPerCaller`
  - 未单独配置时，单个 caller 的默认并发连接上限；`0` 表示只受全局连接上限约束。
- `websocketDefaultMaxRequestsPerMinute`
  - 未单独配置时，单个 caller 的默认每分钟消息上限；`0` 表示不启用 caller 级速率限制。
- `websocketBlacklistIps`
  - 直接拒绝的 caller IP 列表，当前按精确文本匹配。
- `websocketCallers`
  - WebSocket caller 注册表数组，用于声明 caller 身份、secret、状态和配额。
  - 单项字段包括：
    - `callerId`
    - `displayName`
    - `secret`
    - `secretEnv`
    - `enabled`
    - `blacklisted`
    - `blacklistReason`
    - `maxConnections`
    - `maxRequestsPerMinute`

### 2.4 `paths`

- `prototypeDir`
  - 原型根目录相对路径。
- `publicDir`
  - `/admin` MVP 静态页面目录。
- `consoleClientDir`
  - `/console` 前端静态资源目录。
- `workspaceDir`
  - 运行时和控制面共享工作目录。
- `hostVerificationDir`
  - 宿主验证报告输出目录。
- `catalogDir`
  - seed catalog 存放目录。
- `releasesDir`
  - release 版本目录。
- `latestReleaseDir`
  - 当前共享 `latest` 目录。
- `databaseFile`
  - 控制面 SQLite 文件路径。
- `validationFeedInboxDir`
  - Validation feed 待导入目录。
- `validationFeedArchiveDir`
  - Validation feed 成功归档目录。
- `validationFeedErrorDir`
  - Validation feed 错误文件目录。
- `validationFeedReceiptDir`
  - Validation feed connector receipt 目录，用于记录远端 delivery 的导入/ack 结果。
- `demoTermsConfig`
  - demo 词条配置文件路径。
- `seedCatalogFile`
  - 清洗后 seed 词条输出路径。
- `rawRoadsFile`
  - 原始路名源文件路径。
- `rawGovernmentFile`
  - 原始政务部门源文件路径。
- `cleanedDir`
  - 清洗结果输出目录。

## 3. `artifact_store.config.json`

### 3.1 `artifactStore`

- `provider`
  - 制品仓提供方，当前实现以 `minio` 和 `file` 场景为主。
- `endpoint`
  - 对象存储访问地址。
- `endpointEnv`
  - `endpoint` 的环境变量名。
- `publicBaseUrl`
  - 控制面对外生成下载地址时使用的公开基地址。
- `publicBaseUrlEnv`
  - `publicBaseUrl` 的环境变量名。
- `bucket`
  - 发布产物桶名。
- `bucketEnv`
  - `bucket` 的环境变量名。
- `region`
  - 对象存储区域；MinIO 本地场景可为空。
- `regionEnv`
  - `region` 的环境变量名。
- `accessStyle`
  - 访问方式，当前支持 `path` 和 `virtual_host`。
- `useSsl`
  - 是否使用 HTTPS/SSL。
- `presignExpiresSeconds`
  - 预签名下载链接有效期。
- `rootUser`
  - 可选 root 用户，主要用于本地 MinIO 初始化或桶确保。
- `rootUserEnv`
  - `rootUser` 的环境变量名。
- `rootPassword`
  - root 用户密码。
- `rootPasswordEnv`
  - `rootPassword` 的环境变量名。
- `accessKey`
  - 实际对象存储访问账号。
- `accessKeyEnv`
  - `accessKey` 的环境变量名。
- `secretKey`
  - 实际对象存储访问密钥。
- `secretKeyEnv`
  - `secretKey` 的环境变量名。
- `serverDataDir`
  - 本地 MinIO helper 使用的数据目录。
- `apiPort`
  - 本地 MinIO API 端口。
- `consolePort`
  - 本地 MinIO Console 端口。

### 3.2 `runtimeControl`

- `adminBaseUrl`
  - runtime 主动访问 admin 控制面的基地址。
- `adminBaseUrlEnv`
  - `adminBaseUrl` 的环境变量名。
- `nodeId`
  - runtime 节点唯一标识。
- `nodeIdEnv`
  - `nodeId` 的环境变量名。
- `nodeName`
  - runtime 节点展示名称。
- `nodeNameEnv`
  - `nodeName` 的环境变量名。
- `nodeEnv`
  - runtime 节点所属环境标签，如 `local`、`docker`、`k8s`。
- `nodeEnvEnv`
  - `nodeEnv` 的环境变量名。
- `nodeAddress`
  - runtime 节点上报给控制面的访问地址。
- `nodeAddressEnv`
  - `nodeAddress` 的环境变量名。
- `heartbeatIntervalSeconds`
  - runtime 向控制面发送心跳的周期。
- `syncIntervalSeconds`
  - runtime 拉取控制态/版本的周期。
- `nodeOfflineThresholdSeconds`
  - 控制面判定节点离线的超时阈值。
- `downloadTimeoutMs`
  - runtime 下载制品的超时时间。
- `statsFlushIntervalSeconds`
  - runtime 本地统计上传周期。
- `statsFlushMaxBatchSize`
  - 单次统计上传最大批大小。
- `statsRetentionHours`
  - runtime 本地统计保留时长。

### 3.3 `validation_feed_connectors.config.json`

#### `sources[]`

- `sourceType`
  - feed 来源类型，如 `cg3` / `qa_feedback` / `online_feedback`。
- `enabled`
  - 是否启用当前 connector。
- `description`
  - connector 人类可读说明。
- `transportType`
  - connector 方式，当前支持：
    - `file_inbox`
    - `http_pull_json`
- `endpoint`
  - `http_pull_json` 拉取地址。
- `endpointEnv`
  - `endpoint` 的环境变量名。
- `httpMethod`
  - 拉取时使用的 HTTP 方法。
- `timeoutMs`
  - 远端拉取和 ack 的超时时间。
- `authType`
  - 认证方式，当前支持：
    - `none`
    - `bearer`
    - `header`
    - `query`
- `authHeaderName`
  - header 认证时使用的 header 名；bearer 模式默认常用 `Authorization`。
- `authToken`
  - bearer 认证 token。
- `authTokenEnv`
  - `authToken` 的环境变量名。
- `authQueryKey`
  - query 认证时使用的参数名。
- `authValue`
  - 自定义 header/query 认证值。
- `authValueEnv`
  - `authValue` 的环境变量名。
- `cursorQueryKey`
  - 远端增量拉取时使用的 cursor query 参数名。
- `cursorResponseField`
  - 远端返回 payload 中用于提取下一个 cursor 的字段路径，支持点路径如 `meta.nextCursor`。
- `initialCursor`
  - 首次拉取时使用的初始 cursor。
- `initialCursorEnv`
  - `initialCursor` 的环境变量名。
- `includeCursorInAck`
  - 是否在 ack body 中回传 `pulledCursor / nextCursor`。
- `ackType`
  - 导入完成后的确认方式，当前支持：
    - `none`
    - `http_post`
- `ackEndpoint`
  - ack 回调地址。
- `ackEndpointEnv`
  - `ackEndpoint` 的环境变量名。
- `ackMethod`
  - ack 请求使用的方法，当前默认 `POST`。
- `retryMaxAttempts`
  - 单个 delivery 的预期最大重试次数，用于 connector 运行说明与 receipt 元数据。
- `replayFromErrorDir`
  - 是否允许从 error 目录重放失败 payload。

## 4. 环境变量映射

当前代码会优先读取以下环境变量：

- `ACDP_RUNTIME_TOKEN`
  - runtime/admin 共用的 Bearer Token
- `ACDP_ARTIFACT_ENDPOINT`
  - 对象存储访问地址
- `ACDP_ARTIFACT_PUBLIC_BASE_URL`
  - 对外下载基地址
- `ACDP_ARTIFACT_BUCKET`
  - 产物桶名
- `ACDP_ARTIFACT_REGION`
  - 区域
- `ACDP_ARTIFACT_ROOT_USER`
  - root 用户
- `ACDP_ARTIFACT_ROOT_PASSWORD`
  - root 密码
- `ACDP_ARTIFACT_ACCESS_KEY`
  - access key
- `ACDP_ARTIFACT_SECRET_KEY`
  - secret key
- `ACDP_RUNTIME_CONTROL_ADMIN_BASE_URL`
  - runtime 访问 admin 控制面的基地址
- `ACDP_RUNTIME_CONTROL_NODE_ID`
  - runtime 节点 ID
- `ACDP_RUNTIME_CONTROL_NODE_NAME`
  - runtime 节点展示名
- `ACDP_RUNTIME_CONTROL_NODE_ENV`
  - runtime 节点环境标签
- `ACDP_RUNTIME_CONTROL_NODE_ADDRESS`
  - runtime 节点上报地址
- `ACDP_RUNTIME_PORT`
  - runtime split 进程端口
- `ACDP_ADMIN_PORT`
  - admin split 进程端口
- `ACDP_RUNTIME_HOST`
  - runtime split 进程监听主机
- `ACDP_ADMIN_HOST`
  - admin split 进程监听主机
- `ACDP_VALIDATION_FEED_CG3_ENDPOINT`
  - CG3 远端 feed 拉取地址
- `ACDP_VALIDATION_FEED_CG3_TOKEN`
  - CG3 远端 bearer token
- `ACDP_VALIDATION_FEED_CG3_ACK_ENDPOINT`
  - CG3 导入成功后的 ack 地址
- `ACDP_VALIDATION_FEED_CG3_INITIAL_CURSOR`
  - CG3 首次拉取时的初始 cursor
- `ACDP_VALIDATION_FEED_QA_ENDPOINT`
  - QA feedback 远端 feed 拉取地址
- `ACDP_VALIDATION_FEED_QA_TOKEN`
  - QA feedback 远端 bearer token
- `ACDP_VALIDATION_FEED_QA_ACK_ENDPOINT`
  - QA feedback 导入成功后的 ack 地址
- `ACDP_VALIDATION_FEED_QA_INITIAL_CURSOR`
  - QA feedback 首次拉取时的初始 cursor
- `ACDP_VALIDATION_FEED_ONLINE_ENDPOINT`
  - online feedback 远端 feed 拉取地址
- `ACDP_VALIDATION_FEED_ONLINE_TOKEN`
  - online feedback 远端 bearer token
- `ACDP_VALIDATION_FEED_ONLINE_ACK_ENDPOINT`
  - online feedback 导入成功后的 ack 地址
- `ACDP_VALIDATION_FEED_ONLINE_INITIAL_CURSOR`
  - online feedback 首次拉取时的初始 cursor

## 5. K8S 资产中的配置项

### 5.1 `artifact-store-secret.example.yaml`

- `endpoint`
  - 集群内访问对象存储的地址。
- `public-base-url`
  - 控制面下发给 runtime 或控制台用户的对外下载地址。
- `bucket`
  - 发布产物桶。
- `region`
  - 对象存储区域。
- `root-user` / `root-password`
  - 可选 root 账号。
- `access-key` / `secret-key`
  - 实际读写对象存储的凭据。
- `runtime-token`
  - runtime 与 admin 控制面之间共用的 Bearer Token。

### 5.2 `runtime-deployment.yaml`

关键环境变量用途：

- `ACDP_RUNTIME_PORT`
  - runtime Pod 对外监听端口。
- `ACDP_RUNTIME_CONTROL_ADMIN_BASE_URL`
  - runtime 读取控制态时访问 admin Service 的地址。
- `ACDP_RUNTIME_CONTROL_NODE_ID`
  - 当前 Pod 对外登记的节点 ID。
- `ACDP_RUNTIME_CONTROL_NODE_NAME`
  - 控制台展示名称。
- `ACDP_RUNTIME_CONTROL_NODE_ENV`
  - 节点环境标签。
- `ACDP_RUNTIME_CONTROL_NODE_ADDRESS`
  - runtime 节点对控制面上报的访问地址。

### 5.3 `admin-deployment.yaml`

关键环境变量用途：

- `ACDP_ADMIN_PORT`
  - admin Pod 监听端口。
- `ACDP_RUNTIME_TOKEN`
  - admin 校验 runtime 节点接口调用时使用。
- `ACDP_ARTIFACT_*`
  - 控制面构建制品、同步制品、生成下载链接时使用。

## 6. 当前配置原则

- 仓库内 JSON 适合作为默认开发配置，不适合作为生产密钥载体。
- 宿主机与 K8S 场景优先使用 `*Env` + Secret 注入。
- `artifact_store.config.json` 仍是制品仓连接项的唯一配置入口，不在代码里散落写死。
