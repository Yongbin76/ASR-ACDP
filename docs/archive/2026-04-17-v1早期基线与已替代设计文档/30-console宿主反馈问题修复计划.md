## 3. 问题分组

## 3.1 P0 可用性问题

### 问题 A：表单缺少显式标题

表现：

- 词条中心新建词条页大量输入框只靠 placeholder 提示
- 用户不知道字段含义

影响：

- 用户无法快速理解页面
- 录入错误率高

修复目标：

- 所有核心表单字段都必须有清晰 label
- 不再依赖 placeholder 充当字段标题

### 问题 B：创建后的下一步不清楚

表现：

- 词条创建后不知道如何提交审核
- 导入确认后不知道去哪里看结果和审核任务

影响：

- 用户无法形成闭环操作

修复目标：

- 成功动作后给出明确“下一步”入口

### 问题 C：审核中心看不懂

表现：

- 目标摘要显示异常
- 操作列视觉上像“没有按钮”

影响：

- 审核人员无法快速判断任务内容

修复目标：

- 修复目标摘要
- 所有状态都保留清晰详情入口
- 待处理任务明确给出操作按钮

## 3.2 P1 下载与帮助问题

### 问题 D：模板/示例下载体验异常

表现：

- 点击后无明确下载结果

影响：

- 导入中心最关键入口不可用或不可理解

修复目标：

- 模板与示例必须以附件下载方式返回
- 文件名必须清晰

### 问题 E：帮助入口不是产品级页面

表现：

- “联调说明”“试用说明”当前更像跳仓库路径

影响：

- 浏览器中阅读和试用体验不完整

修复目标：

- 改为 `/console/help/...` 内部帮助页

## 3.3 P1 布局与信息架构问题

### 问题 F：词条中心布局不合理

表现：

- 左侧新建表单过大
- 右侧列表阅读体验差

修复目标：

- 让列表成为主视图
- 让新建区更紧凑

### 问题 G：审核中心布局不合理

表现：

- 列宽分配不均
- 目标摘要与操作区展示效果差

修复目标：

- 重排列表列结构
- 优化状态、摘要、操作的布局

### 问题 H：首页空间利用率一般

表现：

- 首屏较空

修复目标：

- 保留摘要卡，但提升信息密度

## 3.4 P1 性能问题

### 问题 I：词条中心加载慢

表现：

- 首页加载可达 10 秒

可能原因：

- 聚合查询存在 N+1
- review/release/validation 摘要重复查询

修复目标：

- 优化 `listConsoleTerms()` 等聚合查询

## 4. 修复轮次建议

建议按 4 轮推进。

## 4.1 第一轮：P0 可用性修复

范围：

- 表单 label
- 下一步引导
- 审核中心目标摘要与操作入口修复

交付目标：

- 用户能看懂页面
- 用户知道下一步怎么做

验收标准：

- 核心表单均有标题
- 创建词条、导入确认后均有下一步入口
- 审核中心不再出现摘要异常与“无按钮错觉”

## 4.2 第二轮：下载与帮助修复

范围：

- 模板下载
- 示例下载
- 帮助页产品化

交付目标：

- 用户能在浏览器中完成模板下载和说明阅读

验收标准：

- 模板/示例点击即下载
- 帮助说明通过 `/console/help/...` 可读

## 4.3 第三轮：布局与页面重排

范围：

- 词条中心首页布局重排
- 审核中心首页布局重排
- 首页信息密度微调

交付目标：

- 页面更符合运营控制台阅读习惯

验收标准：

- 词条列表阅读优先级高于创建表单
- 审核任务可一眼扫描
- 首页不再显得明显空散

## 4.4 第四轮：性能优化与宿主复测

范围：

- 聚合查询优化
- 宿主环境复测

交付目标：

- 词条中心、审核中心首屏加载显著改善

验收标准：

- 列表加载时间显著下降
- 宿主环境复测通过

### 第四轮完成情况

- 词条列表审核/发布摘要查询已改为批量聚合

### 收尾型补充修整完成情况

- 词条停用、批量停用、导入确认/取消、审核通过/驳回、正式发布、回滚、样本停用等动作已增加二次确认
- 表单提交中的按钮会锁定，避免重复点击触发重复写入
- 已停用词条、已有待审词条、已提交或已通过的拼音候选、无错误行的导入批次、不可再次提审或不可发布的 release、已停用样本，均已补齐禁用态提示
- 发布审核详情与审核卡片中，若当前操作人与提交人为同一人，会提前禁用“审核通过”并给出切换操作人提示
- `smoke:console` 结果中已补充 `entryIsolation.adminOk / consoleOk / adminIndependentFromConsole`

## 8. 当前结论

当前 `/console` 不是“功能缺失型问题”，而是“试用体验不足型问题”。

因此下一阶段的重点应是：

- 修易用性
- 修可理解性
- 修下载与帮助
- 修布局
- 修性能

而不是继续大规模扩新功能。
