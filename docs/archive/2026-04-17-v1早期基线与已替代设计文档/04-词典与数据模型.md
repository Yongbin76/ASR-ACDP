# 词典与数据模型

## 1. 设计目标

词典模型不能只保存“错误词 -> 标准词”，还必须保存：

- 类别
- 来源
- 风险
- 上下文约束
- 拼音信息
- 版本状态
- 历史统计

## 2. 核心对象

### 2.1 标准词条 `DictTerm`

表示一个可被平台管理的标准概念，例如：

- `祁顺路`
- `中国宝武`
- `工伤认定`

建议字段：

| 字段 | 含义 |
|------|------|
| `term_id` | 词条唯一 ID |
| `category` | 词典类别 |
| `canonical_text` | 标准词 |
| `status` | 草稿/待审/已发布/停用 |
| `priority` | 类别内优先级 |
| `risk_level` | 低/中/高 |
| `replace_mode` | 自动替换/候选/阻断 |
| `base_confidence` | 基础置信度 |
| `source` | 来源 |
| `owner` | 维护责任人 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 2.2 错误变体 `DictVariant`

表示标准词对应的一个错误形式。

例如：

- `旗顺路` -> `祁顺路`
- `中国保姆` -> `中国宝武`

建议字段：

| 字段 | 含义 |
|------|------|
| `variant_id` | 变体唯一 ID |
| `term_id` | 关联标准词 |
| `wrong_text` | 错误形式 |
| `variant_type` | 人工录入/拼音生成/治理回流 |
| `enabled` | 是否启用 |
| `confidence_prior` | 该变体基础可靠性 |
| `dialect_tags` | 口音标签 |
| `evidence_count` | 样本证据数 |
| `last_verified_at` | 最后确认时间 |

### 2.3 规则配置 `DictRule`

用于限制替换风险。

建议字段：

| 字段 | 含义 |
|------|------|
| `rule_id` | 规则 ID |
| `term_id` | 关联词条 |
| `left_context_allow` | 左侧正向上下文 |
| `right_context_allow` | 右侧正向上下文 |
| `left_context_block` | 左侧阻断上下文 |
| `right_context_block` | 右侧阻断上下文 |
| `boundary_policy` | 边界策略 |
| `min_text_len` | 最小生效长度 |
| `max_text_len` | 最大生效长度 |
| `regex_allow` | 正向正则条件 |
| `regex_block` | 阻断正则条件 |
| `candidate_only` | 是否仅候选 |

### 2.4 拼音画像 `PinyinProfile`

用于支撑中文拼音增强。

建议字段：

| 字段 | 含义 |
|------|------|
| `profile_id` | 拼音画像 ID |
| `term_id` | 关联词条 |
| `full_pinyin` | 全拼 |
| `full_pinyin_no_tone` | 去声调全拼 |
| `initials` | 首字母串 |
| `syllables` | 音节数组 |
| `polyphone_strategy` | 多音词处理策略 |
| `fuzzy_groups` | 可接受混淆组 |
| `runtime_mode` | 精确/候选/关闭 |

## 3. 版本对象

### 3.1 发布版本 `DictRelease`

建议字段：

| 字段 | 含义 |
|------|------|
| `release_id` | 版本 ID |
| `version` | 版本号 |
| `status` | 构建中/已发布/已回滚/已废弃 |
| `artifact_uri` | 快照位置 |
| `created_by` | 创建人 |
| `approved_by` | 审核人 |
| `published_at` | 发布时间 |
| `rollback_from` | 若回滚则记录来源版本 |
| `summary` | 变更摘要 |

### 3.2 快照对象 `DictSnapshot`

运行时加载的不可变产物。

建议包含：

- 快照元信息
- 类别清单
- AC 自动机序列化数据
- 拼音索引
- 词条规则映射
- 校验哈希

## 4. 统计对象

### 4.1 词条统计 `TermStats`

建议字段：

| 字段 | 含义 |
|------|------|
| `term_id` | 词条 ID |
| `version` | 版本号 |
| `hit_count` | 命中数 |
| `replace_count` | 替换数 |
| `candidate_count` | 候选数 |
| `block_count` | 阻断数 |
| `manual_correct_count` | 人工确认正确数 |
| `manual_wrong_count` | 人工确认错误数 |
| `historical_precision` | 历史正确率 |

## 5. 审计对象

### 5.1 审计日志 `AuditLog`

建议记录：

- 操作人
- 操作类型
- 目标对象
- 旧值摘要
- 新值摘要
- 时间戳
- 请求 ID

## 7. 运行时映射模型

编译后，运行时不直接使用完整管理模型，而是拆为三类映射：

- `字面匹配索引`
- `拼音匹配索引`
- `term_id -> 规则/元信息`

这样可以降低运行时内存占用并提升加载速度。

## 8. 设计约束

- 管理态与运行态模型必须分离
- 运行态快照必须不可变
- 拼音画像必须作为一级字段存在
- 词条规则必须可编译为轻量运行时结构
