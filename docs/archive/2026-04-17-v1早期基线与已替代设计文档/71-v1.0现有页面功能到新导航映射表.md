## 1. 文档目的

本文档用于把当前 ACDP 已有页面和功能，映射到拟议中的 `v1.0` 一级 / 二级目录结构下，作为后续开发与重构的对照依据。

本表主要回答 3 个问题：

1. 当前已有页面能否挂接到新导航
2. 哪些页面可以直接复用
3. 哪些页面只是“可复用但需重构”，或仍然缺失

## 2. 覆盖状态说明

- `已覆盖`
  现有页面和功能已基本能承接目标二级目录职责
- `可复用但需重构`
  页面存在，但职责、口径、状态展示或帮助体系仍需按新模型收口
- `缺失需新增`
  当前还没有独立页面或独立职责承接，需要新增

## 5. 当前总结

### 5.1 已经具备的

当前系统并不是“功能缺页”，而是：

- 多数核心功能已经存在
- 多数关键页面已经存在
- 大部分一级 / 二级目录都能找到对应页面或区块来承接

### 5.2 仍需重点重构的

后续仍需重点清理的不是“有没有页面”，而是：

1. 页面职责是否清晰
2. 页面数据口径是否统一
3. 状态展示是否分层
4. 页面帮助是否完整
5. 版本发布是否从“一个详情页塞所有事”改成真正生命周期结构
6. 运行节点中心是否从“运行表视图”改成“备案节点治理视图”

## 6. 用法建议

后续开发时，建议把本表作为对照依据：

1. 每次新增或重构页面，先判断其归属到哪个一级 / 二级目录
2. 先判断是 `已覆盖`、`可复用但需重构` 还是 `缺失需新增`
3. 不再脱离本表单独命名新页面或临时创建新导航层级
