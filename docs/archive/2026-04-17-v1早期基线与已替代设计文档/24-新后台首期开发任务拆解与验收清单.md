# 新后台首期开发任务拆解与验收清单

## 1. 文档目的

本文件把前面的设计文档转成首期可执行开发任务和验收标准。

目标：

- 可以直接进入编码
- 可以按阶段验收
- 可以明确什么叫“完成”

## 2. 首期目标

首期目标不是做完整新后台，而是做出独立的新后台最小可用骨架：

1. 新后台入口独立
2. 只读主页面可访问
3. 模板下载可访问
4. 导入批次模型已落库
5. `/api/console/*` 首批读接口可用
6. 不影响现有 `/admin` MVP

## 3. 任务拆解

## 3.1 基础设施任务

### 任务 A1：新增 import job 相关表

内容：

- 在 `platform-db.js` 中补 `import_jobs` 等 DDL

验收：

- 启动时自动建表
- 不影响现有测试

### 任务 A2：新增模板资产目录

内容：

- 建 `prototype/config/import_templates/`
- 放首批模板文件、示例文件、schema 文件

验收：

- 模板资产在仓库内可见
- 文件命名与 `19` 文档一致

## 3.2 后端读接口任务

### 任务 B1：实现 `/api/console/terms`

验收：

- 可返回分页列表
- 可按基础条件筛选

### 任务 B2：实现 `/api/console/terms/{termId}`

验收：

- 可返回聚合词条详情
- 至少包含 basic / aliases / rules / pinyinProfile / reviewSummary / releaseSummary

### 任务 B3：实现模板接口

范围：

- `GET /api/console/import/templates`
- `GET /api/console/import/templates/{templateCode}`
- `GET /api/console/import/templates/{templateCode}/download`

验收：

- 页面可拿到模板元信息
- 模板文件可下载

### 任务 B4：实现导入批次只读接口

范围：

- `GET /api/console/import/jobs`
- `GET /api/console/import/jobs/{jobId}`
- `GET /api/console/import/jobs/{jobId}/rows`

验收：

- 即使首期没有完整上传流程，也能读到测试批次或空列表

### 任务 B5：实现审核中心只读接口

范围：

- `GET /api/console/reviews`
- `GET /api/console/reviews/{taskId}`

### 任务 B6：实现发布中心只读接口

范围：

- `GET /api/console/releases`
- `GET /api/console/releases/{releaseId}`
- `GET /api/console/releases/{releaseId}/gate`
- `GET /api/console/releases/{releaseId}/validation`

### 任务 B7：实现样本中心只读接口

范围：

- `GET /api/console/validation-cases`
- `GET /api/console/validation-cases/{caseId}`

## 3.3 前端页面任务

### 任务 C1：新后台应用壳

内容：

- 独立页面入口
- 左侧导航
- 顶部上下文栏

验收：

- 新入口与 `/admin` 独立
- 导航可进入六大模块

### 任务 C2：总览页

验收：

- 能展示核心摘要卡

### 任务 C3：词条列表页与详情页

验收：

- 列表可查看
- 点击可进详情页

### 任务 C4：导入中心首页

验收：

- 模板卡片展示
- 模板与示例可下载

### 任务 C5：审核中心列表页

### 任务 C6：发布中心列表页与详情页

### 任务 C7：样本中心列表页与详情页

## 3.4 第二批写接口任务

首期骨架完成后，再进入以下任务：

- 新建/编辑词条
- 创建导入批次
- 确认导入
- 审核通过/驳回
- 构建 release
- 提交发布审核

## 4. 验收清单

进入“可编码完成状态”的验收标准如下：

### 4.1 架构验收

- 新后台目录独立
- 新后台入口独立
- 现有 `/admin` 不受影响

### 4.2 数据验收

- 导入批次表存在
- 模板资产存在
- 不破坏现有 `terms/aliases/review/release/validation_cases`

### 4.3 接口验收

- `/api/console/*` 首批读接口可访问
- 返回结构符合 `18` 文档定义

### 4.4 页面验收

- 六大模块入口存在
- 总览、词条、导入、审核、发布、样本页面可访问
- 风格符合 `20` 文档基线

## 5. 首期不做清单

首期明确不做：

- 完整导入写入闭环
- 全量写接口
- 新后台替换现有 `/admin`
- 复杂报表和高级图表
- 多模板在线配置后台

## 6. 进入编码的判断标准

当以下条件满足时，即可进入编码：

1. `15~24` 文档确认
2. 新后台命名空间确认
3. 模板首批格式确认
4. import job DDL 草案确认
5. 页面骨架与风格基线确认

满足以上条件后，可直接进入代码实现。
