# 实施路线

## 1. 建设阶段

建议分四个阶段推进。

当前原型进度可粗略映射为：

- 阶段一：已完成 MVP 闭环
- 阶段二：已完成词级拼音画像、冲突查看、候选生成、候选审批入库、画像对比、字级差异视图和冲突严重度排序
- 阶段二剩余重点：将 file-based feed connector 升级为真实系统接入
- 阶段三、四：尚未开始正式接入

## 2. 阶段一：平台基础版

### 2.1 目标

先把“词典管理 -> 编译 -> 发布 -> 热更新 -> 回滚”的闭环跑通。

### 2.2 范围

- 三类 AC 词典管理
- 基础词条模型
- 基础审核流
- 字面 AC 快照编译
- 版本发布
- 热更新
- 回滚
- 基础命中统计

### 2.3 产物

- 控制面 API
- 词典管理后台原型
- 运行时快照格式
- 首版 AC 编译器

## 3. 阶段二：中文拼音增强一期

### 3.1 目标

把拼音能力纳入平台主流程，但先控制风险。

### 3.2 范围

- 词级拼音画像
- 拼音混淆规则配置
- 自动候选生成
- 候选审核入库
- 拼音检索能力
- 拼音索引快照
- 拼音通道统计

### 3.3 约束

- 默认不扩大高风险自动替换
- 先以候选召回和变体治理为主

### 3.4 当前完成情况

当前原型已完成：

- 词级拼音画像持久化与人工编辑
- 拼音冲突列表与详情查看
- 多音字候选生成
- 候选提交审批队列
- 审批通过后回写备用读音
- 默认画像与当前画像对比
- 字级差异视图
- 基于短词、高重叠和冲突类型的严重度排序

当前未完成：

- 真实外部系统接入

## 4. 阶段三：运行时增强版

### 4.1 目标

把规则、上下文和拼音候选真正接入在线链路。

### 4.2 范围

- 双通道匹配
- 统一打分与冲突消解
- 上下文规则编译
- 增量稳定输出
- 词条历史正确率接入

## 5. 阶段四：治理闭环版

### 5.1 目标

让平台进入可持续迭代状态。

### 5.2 范围

- 对接 CG3 待审队列
- 误伤/漏召回回流
- 自动聚类
- 版本效果对比报告
- 拼音增强专项评测

## 6. MVP 建议

如果需要尽快落地试点，MVP 建议包含：

- 词典中心
- 审核流
- 编译器
- 发布与回滚
- 三类字面 AC 快照
- 拼音画像和拼音候选生成
- 基础统计与审计

暂不强制纳入：

- 拼音自动替换全量放开
- 全量自动聚类
- 复杂前端大屏

## 7. 风险清单

### 7.1 数据风险

- 历史词典质量不稳定
- 同一标准词缺少足够错误变体
- 高风险短词误伤大

### 7.2 中文风险

- 多音字处理不当
- 拼音混淆规则过宽
- 口音规则扩展过度导致候选爆炸

### 7.3 工程风险

- 运行时直接读管理库导致延迟不稳
- 热更新失败影响在线流量
- 缺少版本回滚演练

## 8. 实施建议

- 先做不可变快照，再做复杂控制面
- 先做词级拼音画像，再做运行时拼音通道
- 先做候选，再决定哪些场景可自动替换
- 先做审计和统计，再扩大词典规模
