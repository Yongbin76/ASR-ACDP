# 后台管理系统视觉风格与交互基线

> 历史说明
>
> 本文档形成于旧后台重构阶段，部分区块命名仍保留旧口径。
>
> 当前如遇到 `主数据 / 内容审核 / 样本与回流` 等旧称，请按现行口径理解：
>
> - `词典建设`
> - `词典审核`
> - `验证与回流`

## 1. 文档目的

本文件用于确定新后台的整体视觉风格、布局语言和交互基线，避免后续实现时退化为“功能堆叠型原型页面”。

本文件适用于独立新后台，不适用于当前：

- `/admin`
- `/test-client`

当前 MVP 页面保持原样，不纳入本文件约束。

## 2. 风格目标

新后台的视觉目标不是“炫”，而是：

- 专业
- 稳定
- 高信息密度但不杂乱
- 适合长时间运营使用
- 让用户对导入、审核、发布等高风险动作有清晰心理预期

一句话概括：

**做成“运营控制台”，不是“技术调试页”。**

## 3. 视觉方向

建议整体风格方向为：

- 浅色主界面
- 天蓝色 + 白色为主视觉体系
- 中性色作为辅助层级色
- 强调色谨慎使用
- 信息卡片化但不过度圆润
- 有明确层级感，但不走花哨大屏路线

建议避免：

- 紫色主色调
- 过多高饱和配色
- 过多渐变背景
- 类似 AI Demo 的强动效风格
- 过度拟物或过度柔和的卡片风格

## 4. 色彩系统建议

## 4.1 主背景

- 页面背景：浅天蓝白底
- 容器背景：纯白
- 大区块分层使用极浅蓝灰

建议基准：

- `--bg-page: #eef7ff`
- `--bg-panel: #ffffff`
- `--bg-muted: #e3f0fb`

## 4.2 主色与辅助色

新后台主体颜色明确采用：

- 天蓝色
- 白色

建议主色基准：

- `--brand-strong: #38a9ff`
- `--brand-main: #59b8ff`
- `--brand-soft: #bfe6ff`
- `--brand-pale: #e9f6ff`

使用建议：

- 一级导航当前态
- 关键按钮
- 重点信息卡标题线
- 选中态
- 信息提示态

## 4.3 主文本

- 一级文本：深灰黑
- 二级文本：中灰
- 弱提示文本：浅灰

建议基准：

- `--text-strong: #18202a`
- `--text-main: #2d3846`
- `--text-subtle: #687586`

## 4.4 语义色

建议严格按语义使用颜色：

- 成功：绿色
- 警告：橙色
- 风险：红色
- 信息：蓝色

建议基准：

- `--success: #1f8f5f`
- `--warning: #c27a12`
- `--danger: #c9473f`
- `--info: #2563eb`

说明：

- 语义色不能替代主品牌色
- 页面大部分“正常态强调”应优先使用天蓝主色，不使用风险色

## 4.5 高风险动作强调

以下操作必须有明显风险样式：

- 发布
- 回滚
- 停用词条
- 批量确认导入
- 审核驳回

但强调方式要克制，避免整页红色污染。

## 5. 字体与排版建议

## 5.1 字体策略

优先使用中文可读性更好的现代无衬线字体组合。

建议方向：

- 中文：`PingFang SC` / `Noto Sans SC` / `Microsoft YaHei`
- 英文与数字：与中文统一风格的 sans-serif

避免：

- 默认系统风格过强的“开发工具感”
- 标题和正文风格割裂过大

## 5.2 排版层级

建议控制为 4 层：

1. 页面标题
2. 模块标题
3. 区块标题
4. 正文与表格文本

不要出现过多字号层级。

## 5.3 数字与状态信息

以下信息应使用更强的视觉权重：

- 数量统计
- 状态标签
- 风险等级
- 发布时间
- 导入结果

## 6. 布局基线

## 6.1 页面骨架

建议采用：

- 左侧主导航
- 顶部上下文栏
- 中间主内容区
- 右侧可选详情抽屉或操作面板

避免沿用当前 MVP 的：

- 多 sheet 切换
- 右侧永久响应结果面板

整体视觉层次建议：

- 左侧导航和顶部上下文区域使用更明确的天蓝色体系
- 主内容区以白底为主
- 区块分层用浅蓝背景和细边框，不用厚重阴影

## 6.2 列表页布局

标准列表页建议分成：

1. 页面标题区
2. 操作与筛选区
3. 主数据表格区
4. 分页区

不要在一个页面中堆太多不同性质的模块。

## 6.3 详情页布局

详情页建议采用：

- 上部摘要卡
- 下部分区块内容

区块顺序建议保持稳定，例如：

1. 基本信息
2. 错误词/别名
3. 规则
4. 拼音
5. 来源
6. 审核
7. 发布
8. 验证

## 7. 组件风格基线

## 7.1 表格

新后台会大量使用表格，表格风格必须稳定。

要求：

- 行高适中
- 表头固定清晰
- 支持悬停态
- 支持状态标签内嵌
- 操作列不堆满按钮

建议：

- 一级动作直接展示
- 次级动作收进下拉菜单

## 7.2 表单

导入、词条编辑、审核、发布都会大量使用表单。

要求：

- 标签与输入框对齐稳定
- 必填项明确标识
- 错误提示贴近字段
- 说明文本不要只藏在 tooltip 里

## 7.3 卡片

卡片用于：

- 首页摘要
- 导入模板入口
- 发布状态摘要

要求：

- 有清晰边界
- 阴影很轻
- 圆角克制
- 不做大面积浮层化

## 7.4 标签与徽标

需要统一标签体系，用于表达：

- 词条状态
- 风险等级
- 导入批次状态
- 审核状态
- 发布状态

要求：

- 同一状态在全系统颜色一致
- 不使用同色表示多种语义

## 8. 交互基线

## 8.1 高风险操作必须二次确认

以下操作必须二次确认：

- 批量确认导入
- 发布
- 回滚
- 停用词条
- 审核驳回

确认弹层必须明确说明：

- 影响对象
- 影响范围
- 是否可回退

## 8.2 错误反馈要可执行

错误提示不能只说“失败”。

必须尽量给出：

- 哪个字段有问题
- 哪一行有问题
- 应该怎么修

尤其是导入中心。

## 8.3 跳转链路必须顺滑

系统中以下跳转必须是一跳直达：

- 导入批次 -> 错误行
- 导入批次 -> 导入后词条
- 词条 -> 审核任务
- 词条 -> 验证失败样本
- release gate -> 失败样本或问题词条

## 8.4 操作反馈要明确

写操作后应明确反馈：

- 成功
- 失败
- 部分成功
- 后续动作建议

例如：

- “导入已创建，待确认”
- “审核已通过，可进入发布中心”
- “发布被 gate 阻断，请先处理 3 条失败样本”

## 9. 首页风格建议

首页不是炫技看板，而是决策面板。

应重点展示：

- 当前词条规模
- 当前错误词规模
- 待审核
- 导入异常
- 最近发布
- 最近验证失败

建议风格：

- 摘要卡 + 简洁趋势区
- 不做复杂图表堆叠
- 不做大面积视觉装饰

## 10. 导入中心的特殊风格要求

导入中心是用户焦虑最高的模块之一，因此需要特殊处理。

### 10.1 上传前

上传前区域必须清楚、安静、可预期。

应突出：

- 模板下载
- 字段说明
- 示例文件
- 支持格式

### 10.2 预览页

预览页应重点区分：

- 可直接导入
- 有警告
- 有错误

建议使用三段式或三色状态分区，但不要过度刺眼。

### 10.3 错误报表

错误行列表必须：

- 行号明确
- 字段名明确
- 错误原因明确
- 可导出

## 11. 词条详情页的特殊风格要求

词条详情页应成为“一个词条的一站式工作台”。

因此风格上要：

- 稳定
- 分区明确
- 不频繁跳页

建议：

- 详情页使用固定锚点导航
- 重要区块保持一致顺序
- 编辑状态和查看状态清晰区分

## 12. 发布中心的特殊风格要求

发布中心承担高风险操作，视觉应更“严肃”。

要求：

- 发布按钮与普通按钮区分明显
- gate blocker 必须强提示
- 验证失败样本必须直观可见
- stable / canary 状态清楚区分

## 13. 新后台与 MVP 的风格关系

当前 MVP 页面：

- 保持原样
- 继续用于演示和联调

新后台：

- 单独定义视觉体系
- 不要求与 MVP 页面统一

也就是说：

- MVP 可以继续偏原型和调试风格
- 新后台必须走正式运营控制台风格

## 14. 首批设计资产建议

建议下一阶段补以下设计资产：

1. 色板与语义色定义
2. 字体与字号层级表
3. 按钮、表格、表单、标签组件规范
4. 列表页线框图
5. 词条详情页线框图
6. 导入中心线框图
7. 发布中心线框图

## 15. 一句话风格结论

新后台的风格建议定为：

**以天蓝色+白色为主体，浅色、克制、稳定、可运营的专业控制台风格。**
