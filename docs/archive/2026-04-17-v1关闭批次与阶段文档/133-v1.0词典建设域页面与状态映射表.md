# v1.0 词典建设域页面与状态映射表

## 1. 目的

本文档用于把“词典建设域”中的页面、对象、状态和主动作映射清楚，作为后续 Codex 编码时的对照依据。

## 2. 页面映射

| 一级目录 | 二级目录 | 主对象 | 主键 | 核心状态 | 主动作 | 说明 |
|---|---|---|---|---|---|---|
| 词典建设 | 词典记录 | 词典记录 | `termId` | `draft / pending_review / approved / disabled` | 保存草稿、编辑、提交审核 | 只承载词典内容本体；提交审核后才生成 `taskId` |
| 词典建设 | 批量导入 | 导入批次 | `importJobId` | `preview_ready / imported / cancelled` | 上传、预览、确认导入 | 只承载词典建设域 CSV 录入过程 |
| 词典建设 | 词典审核 | 审核任务 | `taskId` | `pending / approved / rejected` | 单条审核、批量审核 | 唯一审核原子对象；固定 3 个工作视图 |
| 词典建设 | 基础配置 | 业务属性配置 / 来源类型配置 | `code` | `enabled / disabled` | 新增、编辑、启停 | 页面内两个区块或两个 Tab；不允许删除已引用项 |

## 3. 统一审核映射

| 审核场景 | 实际处理对象 | 查询条件 | 是否依赖 `importJobId` |
|---|---|---|---|
| 单条审核 | 单个 `taskId` | `taskId = ?` | 否 |
| 批量记录审核 | 一组勾选 `taskId` | `taskId in (...)` | 否 |
| 批量导入审核 | 某批次下的一组 `taskId` | `importJobId = ?` | 是 |

结论：

- 3 种审核场景都不新增实体
- 差异只在查询上下文，不在审核逻辑

## 4. 状态映射

## 4.1 词典记录

| 状态 | 含义 | 下一步 |
|---|---|---|
| `draft` | 已保存草稿，但尚未生成 `taskId` | 提交审核 |
| `pending_review` | 已生成 `taskId`，待审核 | 审核通过 / 驳回 |
| `approved` | 审核通过，可进入下一次 build 输入池 | build / release |
| `disabled` | 已停用，不再参与后续链路 | 只读查看 |

## 4.2 导入批次

| 状态 | 含义 | 下一步 |
|---|---|---|
| `preview_ready` | 已完成预览，尚未真正入库 | 确认导入 / 取消 |
| `imported` | 已确认导入并完成入库 | 去看本批 `taskId` |
| `cancelled` | 当前批次已取消 | 重新上传 |

## 4.3 导入行

| 状态 | 含义 | 是否生成任务 |
|---|---|---|
| `ready` | 可直接导入 | 是 |
| `warning` | 需人工确认，但可进入审核流 | 是 |
| `error` | 当前行错误 | 否 |
| `imported` | 已导入 | 已生成或已处理 |

## 4.4 审核任务

| 状态 | 含义 | 下一步 |
|---|---|---|
| `pending` | 待审核 | 通过 / 驳回 |
| `approved` | 已通过 | 回词典建设 / 进入 build |
| `rejected` | 已驳回 | 回词典记录修正 |

## 5. 冲突处理映射

| 场景 | 识别位置 | 展示位置 | 处理原则 |
|---|---|---|---|
| 手工创建 | 词典记录编辑页 | 词典记录页、审核详情页 | `blocked` 不生成任务；`warning` 可保存草稿并提交审核 |
| 手工创建同名冲突 | 词典记录编辑页 | 词典记录页、审核详情页 | `exact_match_existing` 统一转为更新现有词条任务 |
| CSV 预览 | 批量导入预览页 | 导入行、审核详情页 | `warning` 可生成任务 |
| 审核查看 | 不重复识别 | 审核详情页 | 只消费录入阶段摘要 |

## 6. 基础配置映射

## 6.1 业务属性配置

用途：

- 决定词典记录的业务归类
- 驱动筛选、展示、冲突校验

建议字段：

- `code`
- `label`
- `description`
- `enabled`
- `sortOrder`

## 6.2 来源类型配置

用途：

- 统一词典建设和验证回流的来源字典
- 通过 `scope` 防止跨域混用

建议字段：

- `code`
- `label`
- `description`
- `enabled`
- `sortOrder`
- `scope`
- `allowedEntryModes`

## 6.3 `scope` 消费规则

| 页面 | 可见来源类型 |
|---|---|
| 词典记录 | `scope = dictionary` 且允许 `manual` |
| 批量导入 | `scope = dictionary` 且允许 `import` |
| 验证样本 | `scope = validation` 且允许 `manual` |
| 样本导入 | `scope = validation` 且允许 `import` |

## 6.4 配置项删除限制

- 已被词典记录、审核任务、导入批次引用的业务属性不得物理删除
- 已被词典记录、审核任务、导入批次、样本引用的来源类型不得物理删除
- 第一阶段统一通过 `enabled / disabled` 管理生命周期
