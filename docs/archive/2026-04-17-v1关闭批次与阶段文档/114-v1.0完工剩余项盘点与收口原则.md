## 1. 文档目的

本文档用于回答一个现实问题：

**为什么 `JOB-024 ~ JOB-029` 已关闭，但当前 `/console` 视觉与页面观感仍然明显不像“v1.0 完工态”？**

答案不是“代码没做”，而是：

- 核心域模型和首轮页面收口已经做了一轮
- 但仍有一批 `/console` 的可见层遗留，没有被重新显性挂单
- 这些遗留如果不继续拆成新 job，会让 `v1.0` 看起来始终像“过渡态后台”

本文档用于把这些剩余项重新盘点出来，并作为后续 `JOB-031 ~ JOB-033` 的总依据。

## 2. 当前未完工内容的类型

当前剩余项主要分成 4 类：

### 2.1 导航与页面命名仍有过渡态残留

当前问题：

- 左侧导航仍混用旧口径：
  - `总览`
  - `主数据管理`
  - `词条中心`
  - `导入中心`
  - `发布中心`
  - `系统管理`
  - `样本与回流中心`
- 顶部品牌仍是：
  - `ACDP Console`
  - `Console`
  - `新后台独立入口`
  - `内测 / 试用态`
- 页面标题和 breadcrumb 中仍存在旧命名、英文 kicker、旧中心式命名

当前表现文件：

- [index.html](/Codex/ACDP/console/client/index.html)
- [app.js](/Codex/ACDP/console/client/app.js)

### 2.2 二级页面职责还没有完全收口成目标态

当前问题：

- `/console/reviews` 仍是一个统一“内容审核”页，没有真正拆到：
  - 词条审核
  - 拼音审核
- `/console/releases` 仍然是“发布中心”总页，尚未完全显性收口为：
  - 版本列表
  - 发布审核
  - 灰度发布
  - 发布后风险
  - 回滚记录
- `/console/help` 仍是“帮助中心 / 帮助说明”的通用壳，不是最终帮助信息架构
- `/console/system` 虽然有系统配置内容，但标题、导航和内部说明还残留旧“系统管理”语义

当前表现文件：

- [app.js](/Codex/ACDP/console/client/app.js)
- [index.html](/Codex/ACDP/console/client/index.html)

### 2.3 帮助入口和页面内引导还不够“成品化”

当前问题：

- 帮助文档本体已经很多，但页面内帮助入口还不完全统一
- 帮助首页仍偏“索引页”，不够像最终成品帮助台
- 页面内“本页作用 / 下一步去哪 / 关键按钮说明”还没有做到一致收口

当前表现文件：

- [console_help.json](/Codex/ACDP/prototype/config/console_help.json)
- [app.js](/Codex/ACDP/console/client/app.js)
- [docs/help_manuals](/Codex/ACDP/docs/help_manuals)

### 2.4 文档目标态与当前实现态仍存在口径差

当前问题：

- [21-后台管理系统信息架构与页面线框说明.md](/Codex/ACDP/docs/21-后台管理系统信息架构与页面线框说明.md) 仍保留旧导航口径
- [70-v1.0导航命名与配置驱动方案.md](/Codex/ACDP/docs/70-v1.0导航命名与配置驱动方案.md) 的目标命名尚未完全落到代码
- [113-v1.0后台UI统一规范入口.md](/Codex/ACDP/docs/113-v1.0后台UI统一规范入口.md) 已有，但当前实现仍不完全符合

## 3. 当前必须继续收口的范围

为了达到“v1.0 完工态”，当前建议继续收口以下内容：

1. 导航命名、页面标题、breadcrumb、品牌、区块标题的最终中文化与统一
2. `/console/reviews`、`/console/releases`、`/console/help`、`/console/system` 的目标态职责收口
3. 帮助入口、页面内引导、帮助首页分组方式的成品化
4. 文档目标态与实现态的一致性收口

## 4. 不建议继续做的事情

当前不建议再做：

- 零碎 patch 某一页某一句文案
- 不挂真源、只靠对话驱动修改
- 只改文档不补页面
- 只改页面不补帮助

## 5. 建议的新收口批次

建议把剩余完工项重新拆成：

- `JOB-031` 控制台导航、标题与中文化最终收口
- `JOB-032` 控制台页面职责与二级工作页最终收口
- `JOB-033` 控制台帮助入口与页面内引导最终收口

## 6. 当前用途

本文档当前作为：

- `v1.0` 完工剩余项的总盘点
- 后续新收口批次的上层依据
