# v1.0 发布域回滚与发布后风险测试矩阵

## 1. 文档目的

本文档用于固定 `JOB-090` 在发布域中需要重点覆盖的回滚与发布后风险测试矩阵。

## 2. 适用范围

覆盖页面 / 链路：

- 版本列表
- 版本详情
- 发布审核
- 灰度发布
- 正式发布
- 回滚记录
- 发布后风险

## 3. 测试矩阵

| 类别 | 场景 | 预期 | 主要验证点 |
| --- | --- | --- | --- |
| 生命周期 | `built -> canary` | 可升级或被前置条件阻断 | 版本校验与审批 |
| 生命周期 | `built -> published` | 可升级或被前置条件阻断 | 生命周期边界 |
| 生命周期 | `canary -> published` | 可升级或被前置条件阻断 | 灰度到正式 |
| 发布审核 | 审批未满足 | 不可发布 | 审批状态与动作禁用 |
| 发布审核 | 审批满足 | 可发布 | 审批状态与动作联动 |
| 回滚 | 存在上一版本 | 可回滚 | 回滚是动作，不是状态 |
| 回滚 | 回滚后查看记录 | 记录存在 | 回滚记录可读 |
| 风险 | 发布后无风险 | 风险区为空或绿色 | 发布后风险视图 |
| 风险 | 发布后出现风险 | 风险区告警 | 风险与 gate 区分 |
| 风险 | 发布后风险不反向改写 release 状态 | `published` 保持不变 | 生命周期稳定 |

## 4. 当前用途

本文档当前作为：

- `JOB-090` 第一轮发布域矩阵基线
- 后续测试 Codex 补发布域场景时的直接基线
