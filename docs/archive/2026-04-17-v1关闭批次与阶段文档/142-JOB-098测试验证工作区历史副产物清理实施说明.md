# JOB-098 测试/验证工作区历史副产物清理实施说明

## 1. 目标

本文档用于固定 ACDP 中 `prototype/workspace-*` 历史测试、验证、调试和回归副产物的清理口径。

这里的目标是：

1. 回收磁盘空间
2. 清理历史测试残留
3. 不触碰主系统默认工作区
4. 不触碰 runtime 实例工作区

## 2. 清理边界

本任务只处理：

- `prototype/workspace-*`

本任务不处理：

- `prototype/workspace`
- `prototype/runtime_instances/*/workspace`
- 配置文件
- 种子文件
- 代码
- 文档

## 3. 与 JOB-099 的关系

`JOB-098` 对应 `JOB-099` 三步流程中的第一步。

关系如下：

1. `JOB-098` 只负责第一步
2. `JOB-098` 可以独立执行
3. `JOB-098` 完成后，只代表历史测试工作区已清空
4. `JOB-098` 不代表主系统已经零数据化

因此：

1. 如果只是为了瘦身和清磁盘，执行 `JOB-098`
2. 如果是为了恢复“零数据底座”，还要继续执行 `JOB-099` 的第二步和第三步

## 4. 推荐执行方式

推荐只匹配一级目录下的 `workspace-*`，不要把 `workspace` 主目录一起带进去。

示例命令：

```bash
find /Codex/ACDP/prototype -maxdepth 1 -mindepth 1 -type d -name 'workspace-*' \
  -exec sh -c 'find "$1" -mindepth 1 -delete && rmdir "$1"' _ {} \;
```

## 5. 本轮执行结果

本轮已确认：

1. `prototype/workspace-*` 总量约 `63G`
2. `prototype/workspace` 约 `1.2G`
3. `prototype/runtime_instances` 约 `4.6M`

本轮执行后：

1. `prototype/workspace-*` 已清空
2. `prototype/` 总量回落到约 `1.2G`
3. 主工作区 `prototype/workspace` 保持不动
4. `prototype/runtime_instances` 保持不动

## 6. 完成标准

以下条件同时满足时，`JOB-098` 可视为完成：

1. `prototype/workspace-*` 已清空
2. `prototype/workspace` 保持存在
3. `prototype/runtime_instances/*/workspace` 保持不动
4. 已记录清理前后的体积变化

## 7. 注意事项

不要执行以下误删口径：

```bash
rm -rf /Codex/ACDP/prototype/workspace
rm -rf /Codex/ACDP/prototype/runtime_instances
```

因为：

1. 第一条会直接删除主系统当前数据底座
2. 第二条会把 runtime 实例工作区一起删掉

## 8. 后续动作

如果后续再次跑大规模测试、回归、验证或调试，`prototype/workspace-*` 仍可能重新长出来。

此时可重复执行 `JOB-098`，不需要重新定义更大的清库任务。
