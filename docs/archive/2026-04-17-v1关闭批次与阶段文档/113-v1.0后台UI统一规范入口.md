# v1.0 后台 UI 统一规范入口

## 1. 文档目的

本文档用于作为 ACDP 当前后台 UI 规范的统一入口，解决后续开发、测试或交接时的一个问题：

**“现在 UI 设计规范到底应该看哪几份文档？”**

当前 UI 规范并不集中在一份老文档里，而是分散在：

- 视觉与交互基线
- 信息架构与页面骨架
- 导航命名与配置驱动
- `v1.0` 重构原则

本文档的作用就是把这些文档的职责一次性说明清楚。

## 2. 当前 UI 规范的主文档

当前 UI 规范主要基于以下 4 份文档：

1. [20-后台管理系统视觉风格与交互基线](./20-后台管理系统视觉风格与交互基线.md)
2. [21-后台管理系统信息架构与页面线框说明](./21-后台管理系统信息架构与页面线框说明.md)
3. [70-v1.0导航命名与配置驱动方案](./70-v1.0导航命名与配置驱动方案.md)
4. [72-v1.0重构原则定稿版](./72-v1.0重构原则定稿版.md)

如果当前是在做导航、标题、breadcrumb 和中文化的最终收口，还应直接参考：

5. [119-JOB-031文案替换映射表](./119-JOB-031文案替换映射表.md)

## 3. 每份文档分别解决什么问题

## 3.1 视觉与交互

主文档：

- [20-后台管理系统视觉风格与交互基线](./20-后台管理系统视觉风格与交互基线.md)

主要解决：

- 整体视觉风格
- 色彩体系
- 表格、表单、卡片、标签风格
- 高风险动作交互方式
- 工作台、批量导入、词条详情、版本列表等特殊页面的交互要求

如果你的问题是：

- 颜色怎么用
- 卡片和表格怎么画
- 按钮怎么分主次
- 高风险动作怎么确认

优先看这份。

## 3.2 页面结构与线框

主文档：

- [21-后台管理系统信息架构与页面线框说明](./21-后台管理系统信息架构与页面线框说明.md)

主要解决：

- 页面骨架
- 列表页 / 详情页 / 工作流页怎么排
- 区块顺序怎么定
- 页面之间怎么跳

如果你的问题是：

- 这个页面该有哪些区块
- 详情页顺序是什么
- 列表页和详情页怎么切
- 哪些页面应该一跳直达

优先看这份。

## 3.3 导航命名与配置驱动

主文档：

- [70-v1.0导航命名与配置驱动方案](./70-v1.0导航命名与配置驱动方案.md)

主要解决：

- 一级 / 二级目录怎么命名
- 为什么要配置驱动
- 导航、页面标题、breadcrumb、帮助目录怎么同源

如果你的问题是：

- 一级目录叫什么
- 二级目录叫什么
- 后续怎么避免散落硬编码导航名称

优先看这份。

## 3.4 重构边界与中文化原则

主文档：

- [72-v1.0重构原则定稿版](./72-v1.0重构原则定稿版.md)

主要解决：

- 页面职责边界
- 词典建设 / 词典审核 / 发布 / 运行治理 / 系统配置怎么分域
- 中文化要求
- 状态机分层
- 后续编码顺序

如果你的问题是：

- 这个页面到底属于哪个域
- 当前状态应该怎么表达
- 为什么不能把某些内容混在一起
- 中文主文案必须做到什么程度

优先看这份。
