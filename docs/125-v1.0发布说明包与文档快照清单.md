# v1.0 发布说明包与文档快照清单

## 1. 文档目的

本文档用于固定：

- 正式发布时应随证据包归档的文档清单
- 发布说明包应该带哪些文档快照
- 哪些文档只保留在仓库，不进入正式交付包

## 2. 发布说明包建议组成

建议目录：

```text
release_bundle_templates/release_notes_bundle/
  release-notes.md
  deployment-manual.md
  rollback-manual.md
  go-no-go.md
  help-index.md
  api-index.md
```

## 5. 不进入正式发布说明包的文档

以下内容保留在仓库即可，不进入正式交付包：

- `SESSION_HANDOFF.md`
- `NEXT_STEPS.md`
- `docs/38-项目JobList与状态清单.md`
- 各类过程性讨论文档
- 各类测试批次原始 report

原因：

- 它们属于过程管理资料，不属于正式对外交付物

## 6. 与证据包的关系

建议关系：

- 发布说明包放入：
  - `docs/`
- 真实运行与部署证据放入：
  - `deployment/`
  - `concurrency/`
  - `console/`
  - `release/`

这样能保持：

- 文档包
- 技术证据包
- 运行制品包

三者分离。
