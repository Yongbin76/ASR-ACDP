## 2. `/console/reviews`

### 2.1 最终定位

- 一级目录：`内容审核`
- 二级目录：通过**显性视图切换**表达：
  - `词条审核`
  - `拼音审核`

### 2.2 不建议

- 不建议新增 `/console/reviews/terms`、`/console/reviews/pinyin` 独立路由
- `v1.0` 阶段更稳的做法是：
  - 保留 `/console/reviews`
  - 通过 `view=terms|pinyin` 做页内显性切换 “词条审核 / 拼音审核”

### 2.3 页面区块归属

1. 页头  
2. 视图切换区  
3. 筛选区  
4. 任务摘要条  
5. 当前视图任务表 / 卡片流  
6. 分页区  

## 3. `/console/releases`

### 3.1 最终定位

- 一级目录：`版本发布`
- 二级目录：当前阶段保持单一路由，但按区块显性收口为：
  - `版本列表`
  - `发布审核`
  - `灰度发布`
  - `发布后风险`
  - `回滚记录`

当前实现约束：

- 保留 `/console/releases`
- 通过 `view=list|review|canary|risk|rollback` 显性切换工作视图

### 3.2 页面区块归属

1. 页头  
2. 构建版本入口  
3. 列表筛选区  
4. 版本列表表格  
5. 发布审核入口说明区  
6. 灰度发布摘要区  
7. 发布后风险摘要区  
8. 回滚记录入口区  

### 3.3 `release detail`

当前详情页继续保留，但必须固定为：

1. 版本摘要卡  
2. 当前动作区  
3. 版本状态 / 审批状态 / 流量状态  
4. 版本校验  
5. 发布后风险  
6. 节点收敛与验证证据  
7. 变更词条  
8. 回滚相关信息  

## 4. `/console/help`

### 4.1 最终定位

- 一级目录：`帮助`
- 首页按 5 大类固定分组：
  - 页面手册
  - 流程手册
  - 部署手册
  - 接口手册
  - 故障排查

当前实现约束：

- 保留 `/console/help`
- 通过 `pageGroup=pages|flows|ops|apis|troubleshooting` 显性切换帮助分组

### 4.2 单篇帮助页

单篇帮助页固定结构：

1. breadcrumb  
2. 文档标题区  
3. 正文区  
4. 文档信息侧栏  
5. 返回帮助入口  

## 5. `/console/system`

### 5.1 最终定位

- 一级目录：`系统配置`

### 5.2 页面区块归属

1. 页头  
2. 系统配置摘要卡  
3. 用户入口卡  
4. 角色入口卡  
5. 权限入口卡  
6. 治理策略入口卡  
7. 边界说明区
