# v1.0 发布导向重构 joblist 基线

> 历史说明
>
> 本文档保留重构早期的 joblist 草案，不再作为当前真源。
>
> 当前以 `docs/38` 与 `project_management/source_of_truth.json` 为准。

## 1. 文档目的

本文档用于基于 `v1.0` 发布目标，把完整重构主线拆成可执行的工作包基线。

说明：

- 文件名保留“草案”历史编号，但当前内容已作为真源同步后的主线基线
- 后续应以真源中的 `JOB-024 ~ JOB-030` 为准推进

## 2. 基线原则

本基线遵循：

- 先后端 contract，后前端页面
- 先运行治理与发布域，后主数据与系统配置
- 帮助体系与中文化不是收尾点缀，而是并行主线

## 3. 建议主线 joblist

## R1 运行治理域重构

目标：

- 以备案节点为主对象重建运行治理域

范围：

- 节点备案
- 运行节点
- 运行验证
- 节点部署帮助

必须完成：

- 运行节点中心主列表只展示已备案节点
- 未备案节点进入异常接入视图，不再进入主列表
- 节点状态拆成：
  - 注册状态
  - 在线状态
  - 版本对齐状态
  - 最近动作结果
  - 历史异常
- 节点详情页不再混读实时与历史状态

## R2 版本发布域重构

目标：

- 重建 release 生命周期视图

范围：

- 版本列表
- 版本详情
- 发布审核
- 灰度发布
- 发布后风险
- 回滚记录

必须完成：

- `built / canary / published / 发布后风险` 语义彻底分层
- `gate` 只作为升级前置条件
- release review 全量归口发布中心
- 回滚逻辑独立表达

## R3 主数据域重构

目标：

- 重建主数据域页面与读模型

范围：

- 词条
- 批量导入
- 业务属性
- 样本与回流

必须完成：

- 词条中心和导入中心彻底与 release 解耦
- 样本与回流归到主数据域
- 业务属性作为统一可配置对象
- 页面帮助与按钮说明同步落地

## R4 系统配置域重构

目标：

- 把系统治理相关能力统一收口

范围：

- 用户
- 角色
- 权限
- 治理策略

必须完成：

- 用户 / 角色 / 权限 / 治理策略管理入口清晰
- 配置文件持久化与管理页行为一致
- 发布治理规则全部从硬编码切到配置

## R5 帮助域与接口文档体系

目标：

- 建立完整在线帮助手册体系

范围：

- 页面手册
- 流程手册
- 部署手册
- 接口手册
- 故障排查

必须完成：

- 每个主要页面有页面帮助
- 每条主要流程有流程说明
- runtime/admin 接口文档独立成体系
- 部署与排障文档可直接被运维使用

## R6 工作台重构

目标：

- 在前 5 个域稳定后，再重做工作台

范围：

- 首页摘要
- 待办
- 风险提醒
- 快速入口

必须完成：

- 工作台只读稳定聚合结果
- 不再承担跨域兜底解释

## R7 真实环境验证与发布准备

目标：

- 为正式 `v1.0` 发布准备 go / no-go 证据

范围：

- 真实部署
- 真实并发与吞吐
- 文档收口
- 发布准备

必须完成：

- `JOB-006`
- `JOB-009`
- 全量回归
- 文档齐套
- go / no-go 清单

## 4. 与当前 blocked job 的关系

- `R7` 直接吸收 `JOB-006` 与 `JOB-009`
