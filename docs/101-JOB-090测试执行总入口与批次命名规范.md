# JOB-090 测试执行总入口与批次命名规范

## 1. 文档目的

本文档用于为后续测试 Codex 提供统一入口：

- 从哪里开始
- 按什么顺序执行
- 每轮测试怎么命名
- 报告输出到哪里

## 2. 总入口

后续测试 Codex 进入实际执行时，建议按以下顺序开始：

1. `cd /Codex/ACDP`
2. `npm run pm:brief`
3. `npm run pm:check`
4. 阅读：
   - [90-v1.0测试目标与Codex主导测试总方案](./90-v1.0测试目标与Codex主导测试总方案.md)

   - [98-v1.0测试结果归档目录与命名规范](./98-v1.0测试结果归档目录与命名规范.md)
   - [99-v1.0测试执行顺序与域级owner建议](./99-v1.0测试执行顺序与域级owner建议.md)

## 3. 批次命名建议

建议格式：

```text
JOB090-BATCH-01-L0-L2
JOB090-BATCH-02-L3
JOB090-BATCH-03-L4
JOB090-BATCH-04-L5
JOB090-BATCH-05-L6
```

其中：

- `BATCH-01`
  基线和接口层
- `BATCH-02`
  工作流层
- `BATCH-03`
  部署与运行层
- `BATCH-04`
  性能层
- `BATCH-05`
  发布演练层

## 4. 每轮执行最低要求

每轮至少要做到：

1. 记录批次名
2. 记录执行命令
3. 记录环境
4. 输出 `summary.json`
5. 输出 `report.md`
6. 如有失败，输出缺陷报告

## 5. 当前用途

本文档当前作为：

- `JOB-090` 后续测试 Codex 的统一执行入口
- 测试批次命名约束
