# ACDP 术语表与状态机定义

- 文档状态：active
- 适用版本：v1.0
- 文档类型：baseline
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 责任对象：术语统一、状态统一、边界统一

## 1. 术语表

| 术语 | 定义 |
|---|---|
| 词典记录 | 标准词及其别名、规则、拼音画像的组合对象 |
| 标准词 | runtime 最终替换目标文本 |
| 错误词 / 别名 | 触发命中的文本或人工补录的错误表达 |
| 业务属性 | 词条所属业务类别配置项 |
| 来源类型 | 词条或验证样本的数据来源配置项 |
| 审核任务 | 系统内统一的审核原子对象，以 `task_id` 为主语 |
| 导入批次 | 一次批量导入的主对象，以 `job_id` 为主语 |
| release | 一次可构建、可审核、可发布的版本快照主对象 |
| 灰度策略 | 控制 canary 流量比例的规则 |
| 备案节点 | 允许接入 admin 的 runtime 台账对象 |
| 运行节点 | 实际已经注册并持续心跳的 runtime 实时对象 |
| runtime control state | admin 提供给 runtime 的目标版本控制状态 |
| stable | 当前稳定 runtime 快照 |
| canary | 当前灰度 runtime 快照 |
| trafficKey | 参与灰度分流的请求键 |
| candidate | 候选纠错结果，不直接替换主输出 |
| correctedTexts | 候选整句输出数组 |

## 2. 词条状态机

| 状态 | 含义 |
|---|---|
| `draft` | 已创建，但未通过审核 |
| `pending_review` | 已提交审核，等待审核结果 |
| `approved` | 已审核通过，可进入发布 |
| `disabled` | 已停用，不再参与正常使用 |

状态迁移：

1. 创建或导入确认后进入 `draft`。
2. 提交审核后进入 `pending_review`。
3. 审核通过后进入 `approved`。
4. 审核驳回后回到 `draft`。
5. 停用后进入 `disabled`。

## 3. 审核任务状态机

| 状态 | 含义 |
|---|---|
| `pending` | 待审核 |
| `approved` | 已通过 |
| `rejected` | 已驳回 |

## 4. 导入批次状态机

| 状态 | 含义 |
|---|---|
| `uploaded` | 文件已上传 |
| `parsed` | 文件已解析 |
| `preview_ready` | 已完成预览，可等待人工确认 |
| `imported` | 已确认导入完成 |
| `cancelled` | 已取消 |

## 5. 导入行状态机

| 状态 | 含义 |
|---|---|
| `ready` | 当前行可通过 |
| `warning` | 当前行可继续，但存在警告 |
| `error` | 当前行阻断导入 |
| `imported` | 当前行已入库 |

## 6. release 生命周期状态机

| 状态 | 含义 |
|---|---|
| `built` | 已构建 |
| `canary` | 灰度中 |
| `published` | 已发布 |

## 7. release 审批状态机

| 状态 | 含义 |
|---|---|
| `not_submitted` | 未提交审核 |
| `pending` | 已提交，待审批 |
| `partially_approved` | 部分审核通过 |
| `approved` | 审核通过 |
| `rejected` | 审核驳回 |

## 8. runtime 节点状态机

### 8.1 备案状态

| 状态 | 含义 |
|---|---|
| `enabled` | 允许接入 |
| `disabled` | 禁止接入 |

### 8.2 运行状态

| 状态 | 含义 |
|---|---|
| `online` | 当前在线 |
| `offline` | 当前离线 |
| `not_registered` | 已备案但未成功注册 |

### 8.3 收敛状态

常见收敛语义包括：

1. `aligned`
2. `pending`
3. `pending_install`
4. `failed`

## 9. 文档工作区状态机

| 状态 | 含义 |
|---|---|
| `draft` | 草稿文档或草稿工作区 |
| `active` | 当前正式依据 |
| `frozen` | 已封板，不再做增量更新 |
| `archived` | 已归档，不再作为现行依据 |

## 10. Job 状态机

当前建议统一为：

| 状态 | 含义 |
|---|---|
| `pending` | 已识别，未启动 |
| `in_progress` | 已启动，正在执行 |
| `blocked` | 被外部条件或前置依赖阻塞 |
| `done` | 已完成并完成收尾 |

说明：

当前仓库里的历史真源状态定义仍存在旧口径，本工作区后续应以本定义为目标口径统一。

## 11. 当前状态管理原则

1. 生命周期状态、审批状态、流量状态必须分层。
2. 实时状态与历史状态必须分开。
3. 文档状态、工作区状态与 job 状态必须分开。
4. 审核对象统一用 `task_id` 表达，不再让页面本身承担审核主语。
