# ACDP 系统目录结构与代码入口说明

- 文档状态：active
- 适用版本：v1.0
- 文档类型：baseline
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 责任对象：仓库目录、启动入口、代码归属

## 1. 仓库根目录结构

| 目录 | 作用 |
|---|---|
| `console/` | `/console` 前端静态资源 |
| `prototype/` | 服务端代码、配置、测试和工作区 |
| `data_sources/` | 原始与清洗数据源 |
| `docs/` | 历史文档与当前正式文档工作区 |
| `k8s/` | 基础 K8S 资产模板 |
| `project_management/` | 单一真源与项目管理脚本 |
| `release_bundle_templates/` | 发布包模板目录 |

## 2. 前端目录

### 2.1 `console/client`

文件：

1. `index.html`
2. `app.js`
3. `app.css`

职责：

1. 加载 `/console` 主页面。
2. 根据路由渲染工作台、词典建设、发布、运行治理等页面。
3. 调用 `/api/console/*` 聚合接口。

## 3. 服务端目录

### 3.1 入口文件

| 文件 | 作用 |
|---|---|
| `prototype/src/server.js` | 单进程模式装配器，同时包含 runtime WebSocket 处理与主业务装配 |
| `prototype/src/admin-server.js` | admin 独立进程入口 |
| `prototype/src/runtime-server.js` | runtime 独立进程入口 |

### 3.2 HTTP surface

| 文件 | 作用 |
|---|---|
| `prototype/src/http/admin-surface.js` | admin、console、runtime-control 相关 HTTP 接口 |
| `prototype/src/http/runtime-surface.js` | runtime 对外 HTTP / WebSocket 接口 |

### 3.3 核心业务库

| 文件 | 作用 |
|---|---|
| `prototype/src/lib/config.js` | 统一读取应用与配置文件 |
| `prototype/src/lib/platform-db.js` | SQLite 表结构与主要数据读写逻辑 |
| `prototype/src/lib/runtime.js` | runtime 匹配和纠错执行 |
| `prototype/src/lib/runtime-candidates.js` | 候选整句构造 |
| `prototype/src/lib/runtime-artifacts.js` | runtime 本地版本安装、切换和回滚 |
| `prototype/src/lib/runtime-control-client.js` | runtime 到 admin 的控制请求客户端 |
| `prototype/src/lib/runtime-stats.js` | runtime 本地统计与回传 |
| `prototype/src/lib/artifact-store.js` | 快照存储与下载地址处理 |
| `prototype/src/lib/release-gates.js` | 发布门禁与验证汇总 |
| `prototype/src/lib/console-service.js` | `/console` 聚合服务 |
| `prototype/src/lib/access-control.js` | 用户、角色、权限配置 |
| `prototype/src/lib/business-properties.js` | 业务属性配置 |
| `prototype/src/lib/source-types.js` | 来源类型配置 |
| `prototype/src/lib/governance-policies.js` | 治理策略配置 |
| `prototype/src/lib/import-jobs.js` | 导入批次解析、确认、入库 |
| `prototype/src/lib/term-admission.js` | 词条准入规则 |
| `prototype/src/lib/pinyin.js` | 拼音相关能力 |
| `prototype/src/lib/validation-feed-importer.js` | 验证回流导入 |
| `prototype/src/lib/console-help.js` | 帮助目录读取 |
| `prototype/src/lib/console-navigation.js` | 导航目录树读取 |

## 4. CLI 目录

目录：`prototype/src/cli`

当前主要分为 6 类脚本：

1. 初始化类：
   - `prepare-data.js`
   - `bootstrap-db.js`
   - `setup-prototype.js`
2. 构建类：
   - `build-snapshot.js`
   - `prepare-release-bundle.js`
   - `prepare-visual-baseline.js`
3. 启动类：
   - `start-runtime-instance.js`
   - `service-manager.js`
4. 验证类：
   - `verify-runtime-control.js`
   - `verify-multi-runtime.js`
   - `verify-validation-feeds.js`
   - `verify-host-console.js`
5. 检查类：
   - `check-api-contract-snapshots.js`
   - `check-control-managed-config.js`
   - `check-v1-local-readiness.js`
   - `check-target-cluster.js`
6. smoke 与测试类：
   - `console-smoke.js`
   - `admin-smoke.js`
   - `runtime-smoke.js`
   - `console-e2e.js`
   - `concurrency-test.js`

## 5. 配置目录

目录：`prototype/config`

当前主要配置文件如下：

| 文件 | 作用 |
|---|---|
| `app.config.json` | 端口、路径、鉴权等全局配置 |
| `artifact_store.config.json` | 制品仓、runtime control、runtime delivery 配置 |
| `access_control.json` | 权限、角色、用户 |
| `business_properties.json` | 业务属性配置 |
| `source_types.json` | 来源类型配置 |
| `governance_policies.json` | 治理策略配置 |
| `console_navigation.json` | `/console` 左侧目录树配置 |
| `console_help.json` | 帮助目录配置 |
| `validation_feed_connectors.config.json` | 验证回流接入配置 |
| `release_validation_cases.json` | 发布验证样本配置 |
| `import_templates/` | 导入模板与示例文件 |

## 6. 工作区目录

目录：`prototype/workspace`

该目录用于承载本地运行时状态和控制面数据。

主要子目录如下：

| 路径 | 作用 |
|---|---|
| `platform.db` | 控制面主数据库 |
| `catalog/` | 词典 seed 资产 |
| `releases/` | release 目录、manifest 和 snapshot |
| `runtime_artifacts/` | runtime 已安装版本 |
| `runtime_state/` | runtime 当前状态与统计缓冲 |
| `validation_feeds/` | 回流文件 inbox / archive / error / receipts |
| `host_verification/` | 宿主验证和运行验证证据 |
| `import_jobs/` | 导入相关中间文件 |
| `service/` | 服务管理临时状态 |

## 7. 多 runtime 实例目录

目录：`prototype/runtime_instances`

每个实例目录一般包含：

1. 独立 workspace。
2. 独立 runtime_state。
3. 独立 runtime_artifacts。
4. 独立运行时配置快照。

## 8. 测试目录

目录：`prototype/tests`

当前结构如下：

1. `unit/`
   - 单元测试
2. `fixtures/`
   - 测试样本与固定数据
3. `integration.js`
   - 集成测试入口

## 9. 项目管理目录

目录：`project_management`

当前职责：

1. 保存单一真源 `source_of_truth.json`。
2. 通过脚本生成仓库级项目管理派生视图。
3. 执行 `pm:sync`、`pm:check`、`pm:brief`。

## 10. 当前主要启动命令

| 命令 | 作用 |
|---|---|
| `npm run start:prototype` | 单进程启动 |
| `npm run start:admin` | 启动 admin |
| `npm run start:runtime` | 启动 runtime |
| `npm run start:runtime:instance -- ...` | 启动独立 runtime 实例 |
| `npm run smoke:console` | console smoke |
| `npm run test:console` | console e2e |
| `npm run test:unit` | 单元测试 |
| `npm run pm:sync` | 同步项目管理派生文档 |

## 11. 代码归属边界

1. `/console` 前端逻辑只放在 `console/client`。
2. HTTP 路由分为 admin-surface 与 runtime-surface。
3. 数据库结构与业务写入逻辑统一归口 `platform-db.js`。
4. 页面聚合口径统一归口 `console-service.js`。
5. runtime 纠错与候选逻辑统一归口 `runtime.js` 与 `runtime-candidates.js`。
6. 配置型主数据统一归口 `prototype/config` 与对应 lib 文件。
