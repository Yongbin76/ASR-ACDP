# ACDP 前后台功能模块总览

- 文档状态：active
- 适用版本：v1.0
- 文档类型：baseline
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 责任对象：前后台模块职责、页面边界、后端域划分

## 1. 模块总览

ACDP 当前的功能模块分为 4 层：

1. 主后台前端模块。
2. admin 控制面后端模块。
3. runtime 数据面模块。
4. CLI 与部署辅助模块。

## 2. `/console` 主后台模块

当前 `/console` 的正式导航结构如下。

### 2.1 工作台

页面：`/console`

作用：

1. 展示待办、异常、版本摘要和快捷入口。
2. 汇总审核、导入、发布、节点和样本的关键异常。
3. 作为不同角色的统一起点。

### 2.2 词典建设

#### 词典记录

页面：`/console/dictionary/terms`

作用：

1. 手工创建词典记录。
2. 查看、筛选、导出词典记录。
3. 提交审核、停用、查看详情。

#### 批量导入

页面：`/console/dictionary/import-jobs`

作用：

1. 按模板上传导入文件。
2. 预览逐行校验结果。
3. 确认导入或取消导入。
4. 查看导入后形成的审核任务。

#### 词典审核

页面：`/console/dictionary/reviews?targetType=term`

作用：

1. 处理词条审核任务。
2. 处理批量审核。
3. 跟踪导入批次关联的审核任务。

#### 基础配置

页面：`/console/dictionary/config`

作用：

1. 管理业务属性。
2. 管理来源类型。
3. 为词典建设和验证回流提供统一配置项。

### 2.3 验证与回流

#### 验证样本

页面：`/console/validation/cases`

作用：

1. 手工维护验证样本。
2. 导入验证样本。
3. 停用样本。
4. 查看样本关联的词条。

### 2.4 版本发布

#### 版本列表

页面：`/console/releases`

作用：

1. 查看 release 列表。
2. 构建新版本。
3. 查看 release 详情和状态。

#### 发布审核

页面：`/console/releases?view=review`

作用：

1. 查看发布审核视图。
2. 提交审核、审核通过或驳回。
3. 查看审批进度和审批人限制。

#### 灰度发布

页面：`/console/releases?view=canary`

作用：

1. 查看 `built/canary` 状态的版本。
2. 进入 release 详情页执行灰度。

#### 发布后风险

页面：`/console/releases?view=risk`

作用：

1. 查看已发布版本的风险与门禁结果。
2. 追踪 rollout、validation 和节点异常。

#### 回滚记录

页面：`/console/releases?view=rollback`

作用：

1. 查看可回滚版本和回滚入口。
2. 从 release 详情页执行回滚。

### 2.5 运行治理

#### 总览

页面：`/console/runtime`

作用：

1. 汇总节点备案、节点在线状态、当前 stable/canary 版本。
2. 提供节点备案、运行节点、运行验证入口。

#### 节点备案

页面：`/console/runtime-node-registry`

作用：

1. 管理允许接入的 runtime 节点。
2. 维护节点 ID、名称、环境、地址和注册密钥。
3. 提供部署说明和启动命令示例。

#### 运行节点

页面：`/console/runtime-nodes`

作用：

1. 查看已注册 runtime 节点。
2. 观察当前版本对齐情况、最近心跳、最近应用结果和统计摘要。
3. 查看节点详情。

#### 运行验证

页面：`/console/runtime-verify`

作用：

1. 查看当前 stable/canary 版本。
2. 调用 `correct` 验证。
3. 调用 `correct_cand` 验证。

### 2.6 系统管理

#### 总览

页面：`/console/system`

作用：

1. 汇总用户、角色、权限和治理策略入口。

#### 用户管理

页面：`/console/users`

作用：

1. 管理用户及默认角色。

#### 角色管理

页面：`/console/roles`

作用：

1. 管理角色与权限组合。

#### 权限管理

页面：`/console/permissions`

作用：

1. 查看权限定义和角色覆盖情况。

#### 治理策略

页面：`/console/governance-policies`

作用：

1. 管理审批、门禁等系统治理规则。

### 2.7 帮助

页面：`/console/help?pageGroup=...`

作用：

1. 输出页面手册。
2. 输出流程手册。
3. 输出部署手册。
4. 输出接口手册。
5. 输出故障排查手册。

## 3. admin 控制面后端模块

### 3.1 admin-surface

路径：`prototype/src/http/admin-surface.js`

作用：

1. 处理 `/api/admin/*` 兼容接口。
2. 处理 `/api/console/*` 主后台接口。
3. 处理 runtime 节点注册、心跳、控制状态、统计上传和 snapshot 下载。

### 3.2 console-service

路径：`prototype/src/lib/console-service.js`

作用：

1. 对多表、多状态、多域的数据进行聚合。
2. 统一后台展示口径。
3. 为工作台、release 详情、节点详情、导入详情等提供视图对象。

### 3.3 platform-db

路径：`prototype/src/lib/platform-db.js`

作用：

1. 维护 SQLite 表结构。
2. 提供词典、审核、发布、导入、验证、节点、统计的读写逻辑。

### 3.4 业务配置模块

路径：

1. `prototype/src/lib/business-properties.js`
2. `prototype/src/lib/source-types.js`
3. `prototype/src/lib/access-control.js`
4. `prototype/src/lib/governance-policies.js`

作用：

1. 读取和维护配置型主数据。
2. 为后台和治理逻辑提供统一入口。

### 3.5 帮助与导航模块

路径：

1. `prototype/src/lib/console-help.js`
2. `prototype/src/lib/console-navigation.js`

作用：

1. 读取帮助目录和导航配置。
2. 支撑 `/console` 左侧目录树和帮助中心。

## 4. runtime 数据面模块

### 4.1 runtime-surface

路径：`prototype/src/http/runtime-surface.js`

作用：

1. 输出 runtime HTTP 接口。
2. 处理 WebSocket 升级。

### 4.2 runtime 核心引擎

路径：`prototype/src/lib/runtime.js`

作用：

1. 加载 snapshot。
2. 执行字面匹配。
3. 执行拼音召回。
4. 应用规则、合并命中并输出修正结果。

### 4.3 runtime-candidates

路径：`prototype/src/lib/runtime-candidates.js`

作用：

1. 基于一次纠错详情生成候选整句集合。
2. 为 `correct_cand` 和 `ws correct_cand` 提供输出。

### 4.4 runtime-control-client

路径：`prototype/src/lib/runtime-control-client.js`

作用：

1. 向 admin 发起 register、heartbeat、apply result、stats upload 请求。

### 4.5 runtime-artifacts

路径：`prototype/src/lib/runtime-artifacts.js`

作用：

1. 下载 runtime 快照。
2. 安装到本地版本目录。
3. 原子切换当前版本。
4. 回滚到旧版本。

### 4.6 runtime-stats

路径：`prototype/src/lib/runtime-stats.js`

作用：

1. 记录本地请求量、命中量、峰值和延迟样本。
2. 周期性向 admin 上传聚合统计。

## 6. 当前后台边界说明

1. `/console` 是正式后台主入口。
2. `/admin` 是兼容入口，不应继续承接新的主流程设计。
3. runtime 外部合同由 `/api/runtime/*` 和 `/ws/runtime/*` 承担。
4. 管理后台不直接对外替代 runtime 数据面接口。
5. 文档与帮助已被视为系统正式模块，而不是附属材料。
