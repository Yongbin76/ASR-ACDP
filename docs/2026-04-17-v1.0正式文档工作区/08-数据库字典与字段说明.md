# ACDP 数据库字典与字段说明

- 文档状态：active
- 适用版本：v1.0
- 文档类型：baseline
- 所属工作区：2026-04-17-v1.0正式文档工作区

## 1. 数据库概述

当前控制面主数据库为 SQLite，默认文件为：

`prototype/workspace/platform.db`

本文件按表分组说明当前系统的主要字段和用途。

## 2. 词典建设相关表

### 2.1 `terms`

作用：词典标准词主表。

| 字段 | 含义 |
|---|---|
| `term_id` | 主键，词条 ID |
| `category_code` | 类别编码 / 业务属性代码 |
| `canonical_text` | 标准词 |
| `status` | 词条状态，常见值 `draft/pending_review/approved/disabled` |
| `priority` | 优先级 |
| `risk_level` | 风险级别 |
| `replace_mode` | runtime 替换模式 |
| `base_confidence` | 基础置信度 |
| `source_type` | 来源类型 |
| `pinyin_runtime_mode` | 拼音运行模式 |
| `revision` | 词条修订号 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 2.2 `aliases`

作用：词条别名表。

| 字段 | 含义 |
|---|---|
| `id` | 自增主键 |
| `term_id` | 所属词条 |
| `alias_text` | 错误词 / 别名文本 |

约束：

1. `(term_id, alias_text)` 唯一。

### 2.3 `term_sources`

作用：标准词来源追踪表。

| 字段 | 含义 |
|---|---|
| `term_id` | 主键，关联词条 |
| `source_type` | 来源类型 |
| `import_job_id` | 导入批次 ID，可为空 |
| `source_file_name` | 来源文件名 |
| `source_row_no` | 来源行号 |
| `source_ref` | 来源引用说明 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 2.4 `alias_sources`

作用：别名来源追踪表。

| 字段 | 含义 |
|---|---|
| `term_id` | 所属词条 |
| `alias_text` | 别名文本 |
| `source_type` | 来源类型 |
| `import_job_id` | 导入批次 ID |
| `source_file_name` | 来源文件名 |
| `source_row_no` | 来源行号 |
| `source_ref` | 来源引用说明 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

主键：

1. `(term_id, alias_text)`

### 2.5 `term_rules`

作用：词条规则表。

| 字段 | 含义 |
|---|---|
| `term_id` | 主键，关联词条 |
| `candidate_only` | 是否仅候选输出 |
| `min_text_len` | 文本最小长度 |
| `max_text_len` | 文本最大长度 |
| `boundary_policy` | 边界策略 |
| `left_context_allow` | 左侧允许上下文 JSON |
| `right_context_allow` | 右侧允许上下文 JSON |
| `left_context_block` | 左侧阻断上下文 JSON |
| `right_context_block` | 右侧阻断上下文 JSON |
| `regex_allow` | 允许正则 JSON |
| `regex_block` | 阻断正则 JSON |
| `updated_at` | 更新时间 |

### 2.6 `pinyin_profiles`

作用：词条拼音画像表。

| 字段 | 含义 |
|---|---|

## 10. 当前数据库设计要点

1. 词条主对象、审核主对象、导入主对象、release 主对象、runtime 节点主对象彼此独立。
2. `review_tasks` 是审核链统一主语。
3. `runtime_node_registry` 与 `runtime_nodes` 必须分开。
4. 统计表按“节点级”和“全局级”拆分。
5. 当前数据模型围绕 SQLite 设计，强调单机可运行和原型可交接。
