# ACDP 项目总览与版本边界

- 文档状态：active
- 适用版本：v1.0
- 文档类型：baseline
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 责任对象：项目定位、交付范围、阶段验收口径

## 1. 项目定位

ACDP 是一个面向中文 ASR 热词纠错场景的词典平台原型。

当前版本的核心目标不是单纯提供一个纠错接口，而是提供一套完整链路：

1. 词典内容建设。
2. 审核与准入控制。
3. 版本构建、发布、灰度与回滚。
4. runtime 对外纠错服务。
5. runtime 节点备案、注册、心跳、版本收敛与统计回传。
6. 管理后台、帮助体系、测试与发布包准备。

## 2. 当前版本的正式目标

本版本以 `v1.0` 作为阶段性验收目标，强调“可运行、可治理、可交接”，而不是生产级完全体。

当前版本的交付目标如下：

1. `/console` 成为主后台入口，完成词典建设、验证回流、版本发布、运行治理、系统管理和帮助域的工作台化收口。
2. runtime 对外暴露稳定的 HTTP / WebSocket 纠错接口，并支持 `correct_cand` 候选整句输出。
3. 管理端支持词典记录创建、批量导入、审核任务、版本构建、灰度和发布后观察。
4. 支持 `1 admin + N runtime` 的治理模型，具备节点备案、注册和状态追踪能力。
5. 支持 `file`、`minio`、`admin_http_signed` 三类快照下发方式。
6. 建立完整的测试、帮助和发布包准备资料。

## 3. 当前版本的正式交付物

当前版本的正式交付物包括：

1. 运行服务：
   - admin 服务
   - runtime 服务
   - 单进程 prototype 服务
2. 管理后台：
   - `/console` 主后台
   - `/admin` 兼容入口
3. 数据与配置：
   - SQLite 控制库
   - 配置文件
   - import 模板
   - validation feed 配置
4. 运行资产：
   - release snapshot
   - runtime 本地安装目录
   - runtime 统计与状态目录
5. 文档与测试：
   - 本工作区正式文档
   - 帮助页内容
   - CLI 测试脚本
   - Docker 与发布包模板

## 4. 当前版本范围内

### 4.1 业务能力

1. 词典记录的手工创建、编辑、停用、提交审核。
2. 结构化词条 CSV 导入、错误词批量补录、逐行校验与导入确认。
3. 词条审核、拼音候选审核、发布审核。
4. 验证样本的创建、导入、停用和关联词条查看。
5. 版本构建、版本审核、灰度、正式发布、回滚。
6. runtime 节点备案、节点注册、心跳、收敛观察、运行验证。
7. 用户、角色、权限、治理策略配置。

### 4.2 技术能力

1. 基于 SQLite 的控制面数据存储。
2. 基于 snapshot 的 runtime 词典加载与切换。
3. 基于 Aho-Corasick 和拼音索引的纠错执行。
4. runtime 统计缓冲与定时回传。
5. 本地 CLI 验证、回归、构包和 smoke。
6. Docker 打包与基础 K8S 模板。

## 5. 当前版本明确不在范围内

以下内容不属于当前版本正式交付范围：

1. 多副本 admin 的高可用控制面。
2. PostgreSQL、Redis、消息队列等生产级基础设施替换。
3. 分布式 runtime 一致性调度。
4. 生产级 IAM、SSO、多租户权限隔离。
5. 真正的对象存储高可用治理和跨地域复制。
6. 真实目标集群的正式上线证据。
7. 真实宿主机并发吞吐证据。
8. 全量生产数据规模下的容量证明。

## 6. 当前版本的正式后台口径

### 6.1 `/console`

`/console` 是当前版本的正式管理后台入口。

它承接：

1. 业务操作。
2. 发布治理。
3. runtime 节点治理。
4. 配置治理。
5. 帮助与文档入口。

### 6.2 `/admin`

`/admin` 仍保留为兼容入口和 MVP 形态的后台页面，但不作为当前版本的主验收后台入口。

当前正式验收和后续衍生开发应以 `/console` 为准。

## 7. 当前版本的正式服务口径

### 7.1 admin

admin 是控制面服务，负责：

1. 管理后台页面与聚合接口。
2. 数据写入与治理规则。
3. release 构建、门禁检查、目标版本下发。
4. runtime 节点注册与控制。
5. 帮助与配置输出。

### 7.2 runtime

runtime 是数据面服务，负责：

1. 对外提供纠错能力。
2. 加载 stable/canary snapshot。
3. 执行灰度分流。
4. 定期向 admin 心跳和上报统计。
5. 本地安装、切换与回滚 runtime 版本。

## 8. 当前版本的阶段验收判断标准

一个版本可以被视为当前阶段可验收，至少要满足：

1. `/console` 主链路可用。
2. runtime HTTP / WebSocket 主接口可用。
3. 词典建设、审核、发布、节点治理形成闭环。
4. 本工作区文档可支撑开发、测试、运维和验收。
5. 本地回归脚本通过。
6. 外部阻塞项被明确标记，而不是被混入“已完成”。

## 9. 当前版本的主要外部阻塞

当前仍依赖外部条件的事项主要有：

1. 真实目标 K8S 集群验证。
2. 真实宿主机并发与吞吐验证。
3. 真实外部 validation feed 联调。
4. 正式发布前的 go / no-go 证据补齐。

这些事项属于当前版本的发布准备工作，不改变当前版本已经形成的本地可运行能力。
