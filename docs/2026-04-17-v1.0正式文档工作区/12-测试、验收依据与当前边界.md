# ACDP 测试、验收依据与当前边界

- 文档状态：active
- 适用版本：v1.0
- 文档类型：baseline
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 责任对象：测试口径、验收标准、已知边界与外部阻塞

## 1. 验收依据

当前版本的阶段验收应同时检查：

1. 功能链路。
2. 接口合同。
3. 配置与部署能力。
4. 文档完整性。
5. 回归结果。
6. 外部阻塞说明。

## 2. 当前主要测试命令

| 命令 | 作用 |
|---|---|
| `npm run test:unit` | 单元测试 |
| `npm run test:unit:coverage` | 覆盖率版单元测试 |
| `npm run test:console` | console e2e |
| `npm run smoke:console` | console smoke |
| `npm run smoke:admin` | admin smoke |
| `npm run smoke:runtime` | runtime smoke |
| `npm run check:api-contracts` | 接口快照检查 |
| `npm run check:visual-regression` | 视觉回归检查 |
| `npm run check:v1-local-readiness` | 本地 readiness 汇总检查 |
| `npm run verify:runtime-control` | runtime-admin 控制链验证 |
| `npm run verify:multi-runtime` | 多 runtime 验证 |
| `npm run test:concurrency` | 并发验证脚本 |

## 3. 当前可视为本地通过的验收项

当前本地阶段可视为已经具备的能力包括：

1. `/console` 主后台可启动并运行。
2. runtime 主接口可启动并运行。
3. `correct` / `correct_cand` / WebSocket 主链路存在。
4. 词典建设、审核、导入、发布、节点治理页面已闭环。
5. 多 runtime 实例启动与注册能力已在本地验证。
6. `admin_http_signed` 快照下发能力已落地。
7. Docker 镜像构建命令已验证。

## 4. 当前验收时必须明确说明的边界

### 4.1 已完成但仅限本地或原型级

1. SQLite 控制面。
2. 单控制面 admin。
3. 本地 MinIO / file / admin_http_signed 模式验证。
4. 本地多 runtime 实例治理。

### 4.2 仍受外部条件阻塞

1. 真实目标 K8S 集群验证。
2. 真实宿主机并发吞吐采集。
3. 真实外部回流系统联调。
4. 正式发布前 go / no-go 证据补齐。

## 5. 当前阶段的验收判断标准

### 5.1 功能验收

至少满足：

1. 词典记录可创建、审核和停用。
2. 批量导入可预览、确认、形成审核任务。
3. 验证样本可维护和关联词条。
4. release 可构建、审核、灰度、正式发布和回滚。
5. 节点可备案、注册、心跳、收敛可见。
6. runtime 纠错接口可调用。

### 5.2 文档验收

至少满足：

1. 本工作区文档可独立解释当前系统。
2. 基线文档覆盖项目总览、架构、模块、目录、数据模型、接口、配置、测试和边界。
3. 治理文档覆盖 JOB、CHECKLIST、单一真源、归档规则。

### 5.3 项目管理验收

至少满足：

1. 任务状态与文档同步逻辑明确。
2. 关单前必须定义文档同步要求。
3. 后续衍生开发不再脱离新工作区单独散写文档。

## 6. 当前主要风险

1. 若继续混用旧文档和新文档，现行口径会再次漂移。
2. 若未把文档纳入真源与 job 约束，后续文档仍会脱节。
3. 若在没有真实 K8S 和并发证据的前提下直接宣称正式发布，会高估当前成熟度。
4. 若把 `/admin` 与 `/console` 同时当主后台，会再次打乱后台边界。

## 7. 当前版本的建议验收表述

当前版本可以表述为：

1. 已完成本地可运行、可治理、可交接的 v1.0 原型基线。
2. 已具备阶段性项目验收条件。
3. 正式生产级发布仍需要补齐真实部署和真实吞吐证据。
