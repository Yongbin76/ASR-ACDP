## 2. JOB-106 统一口径

### 2.1 保存前系统建议

用户输入标准词、别名、拼音和规则后，页面必须在保存前直接展示系统建议。

系统建议至少包括：

1. `可直接替换`
2. `仅推荐候选`
3. `建议并入已有词条`
4. `建议补录到已有词条`
5. `阻断，不能保存`

### 2.2 页面行为冻结

1. 建议为 `save_replace`
   - 允许保存
   - 页面显示 `建议：直接替换`
2. 建议为 `save_candidate`
   - 允许保存
   - 页面自动锁定为 `推荐候选`
   - 不允许切回 `直接替换`
3. 建议为 `merge_existing`
   - 页面不走普通保存
   - 直接提示目标词条和并入建议
4. 建议为 `append_alias_to_existing`
   - 页面不走普通保存
   - 直接提示目标词条和补录建议
5. 建议为 `skip_blocked`
   - 页面禁止保存
   - 直接展示阻断原因
6. 若词条运行方式为 `candidate`
   - 页面必须显式展示 `仅推荐`
   - 页面不得把其呈现为“可直接替换”

### 2.3 页面必须展示的字段

手工录入页或详情编辑页必须展示：

1. `准入结果`
2. `运行方式`
3. `系统建议`
4. `原因摘要`
5. `审核提示`
6. `目标词条`
7. `运行标识`

### 2.4 reviewHints 呈现口径

1. 页面中的“审核提示”即 `reviewHints`
2. 第一版不要求单独落库
3. 页面读取详情、录入校验结果或审核摘要时，按当前状态动态生成
4. 页面只负责中文展示，不直接暴露内部英文键名

### 2.5 candidate 标识

若当前词条为 `candidate`，页面固定使用中文 badge：

1. `仅推荐`
2. `不可直接替换`

### 2.6 系统建议说明卡片

手工录入页第一版必须提供一张轻量版“系统建议说明”帮助卡片，用于解释 5 类系统建议动作，不展开成长文教程。

卡片至少说明：

1. `可直接替换`
   - 当前词条可按替换词条保存
2. `仅推荐候选`
   - 当前词条只能进入候选推荐链
3. `建议并入已有词条`
   - 当前词条更适合并入现有词条，不建议新建
4. `建议补录到已有词条`
   - 当前词条更适合作为现有词条的错误词或补充信息
5. `阻断，不能保存`
   - 当前词条不适合进入 runtime 词典
