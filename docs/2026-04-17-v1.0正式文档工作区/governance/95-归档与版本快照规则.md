# 归档与版本快照规则

- 文档状态：active
- 适用版本：v1.0
- 文档类型：governance
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 责任对象：工作区冻结、归档、版本快照

## 1. 目标

本文件用于防止后续版本演进时再次出现：

1. 多套现行文档并存。
2. 旧工作区直接被覆盖。
3. 已冻结版本被继续混写。

## 2. 工作区生命周期

工作区状态统一使用：

1. `active`
2. `frozen`
3. `archived`

### 2.1 `active`

含义：

1. 当前现行正式文档工作区。

### 2.2 `frozen`

含义：

1. 已达到阶段封板。
2. 不再做日常增量修改。
3. 仅允许补归档说明或明显错误修订。

### 2.3 `archived`

含义：

1. 已被新工作区替代。
2. 只保留追溯价值。

## 3. 新版本工作区生成规则

当出现以下任一情况时，应建立新的工作区：

1. 版本号升级。
2. 后台信息架构大改。
3. 数据模型或发布机制发生系统级变化。
4. 当前工作区已经封板，但新阶段仍需继续演进。

## 4. 生成新工作区时的步骤

1. 冻结当前工作区。
2. 在新目录下创建新的正式工作区。
3. 复制仍然有效的基线结构。
4. 按新代码事实更新正文。
5. 在旧工作区记录“被哪个新工作区替代”。
6. 将旧工作区状态切到 `archived`。

## 5. 归档规则

### 5.1 文档级归档

当单份文档被替代时：

1. 原文档状态改为 `archived`。
2. 在文档头部注明替代文档。
3. 在归档索引登记。

### 5.2 工作区级归档

当整个工作区被替代时：

1. 工作区冻结。
2. 在 `archive/00-归档索引.md` 登记。
3. 记录被哪个工作区替代。

## 6. 快照要求

每次阶段封板时，应保留：

1. 工作区索引。
2. 基线文档全集。
3. 治理文档全集。
4. 关联的 job / ADR / 模板目录。
