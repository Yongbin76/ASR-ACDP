# CHECKLIST 与收尾标准

- 文档状态：active
- 适用版本：v1.0
- 文档类型：governance
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 责任对象：job 执行、文档同步、关单约束

## 1. 固定节奏

后续每个 coding job 固定按以下节奏推进：

1. 代码实现。
2. `/compact`。
3. 回归测试。
4. 文档更新。
5. 单一真源同步。
6. 关单或进入下一单。

## 2. 文档类 job 的固定节奏

对于纯文档类 job，固定按以下节奏推进：

1. 梳理代码和配置事实。
2. 更新正式文档。
3. 更新 `JOB与文档绑定矩阵`。
4. 更新 `文档变更记录`。
5. 如涉及项目状态，再同步真源。
6. 自检目录闭环。

## 3. 每个 job 的最小 checklist

### 3.1 立项前

- [ ] 已确认 job 目标
- [ ] 已确认影响域
- [ ] 已确认影响文档
- [ ] 已确认是否影响 ADR

### 3.2 开工前

- [ ] 已切换 job 状态
- [ ] 已明确 owner
- [ ] 已明确 checklist
- [ ] 已明确收尾标准

### 3.3 实施中

- [ ] 代码或文档改动围绕单一范围推进
- [ ] 无关范围未被顺手扩张
- [ ] 影响的正式文档同步更新
- [ ] 必要时补充或更新 ADR

### 3.4 收尾前

- [ ] 回归测试已执行
- [ ] 正式文档已同步
- [ ] 变更记录已同步
- [ ] Job 与文档绑定矩阵已同步
- [ ] 单一真源已同步

### 3.5 关单前

- [ ] checklist 全部完成
- [ ] 风险与边界已明确
- [ ] 若有外部阻塞，已显式保留为 blocked，不混入 done
- [ ] 最终状态可被后续 Codex 直接接手

## 4. 文档收尾标准

正式文档更新后，至少满足：

1. 已写明文档状态、版本、类型、更新时间。
2. 已被总索引收录。
3. 已属于清晰的文档类型。
4. 未引用目录外旧文档。
5. 已与受影响的 job 建立关系。

## 5. 代码类 job 的关单标准

一个代码类 job 进入 `done` 前，必须同时满足：

1. 代码功能完成。
2. 回归测试完成。
3. 本工作区正式文档已同步。
4. `文档变更记录` 已同步。
5. `JOB与文档绑定矩阵` 已同步。
6. 真源状态已同步。

## 6. 纯文档类 job 的关单标准

一个纯文档类 job 进入 `done` 前，必须同时满足：

1. 正式文档已形成闭环。
2. 治理规则明确。
3. 后续新增文档规则明确。
4. 与 job / checklist / 真源的关系明确。
5. 目录结构、模板、归档规则已齐备。
