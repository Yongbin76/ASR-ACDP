# 单一真源与文档管理绑定规则

- 文档状态：active
- 适用版本：v1.0
- 文档类型：governance
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 责任对象：单一真源、JOBLIST、文档注册关系

## 1. 目标

当前项目管理真源已经存在，但正式文档工作区尚未完全纳入同一条真源链。

本文件定义新的绑定原则：

1. 单一真源不仅管理 job 状态，还要管理正式文档元数据。
2. 正式文档正文继续保存在 Markdown 文件中。
3. 真源只管理结构化关系，不承载大段正文。

## 2. 当前现实与当前要求

### 2.1 当前现实

当前仓库中的单一真源文件为：

`project_management/source_of_truth.json`

它已经管理：

1. job 摘要字段。
2. 仓库级项目管理派生视图。

### 2.2 当前已完成的首轮扩展

当前真源已经完成首轮扩展，已正式管理：

1. 当前工作区路径。
2. 正式文档注册表。
3. job 与文档的绑定关系。
4. ADR 注册表。

## 3. 当前绑定后的结构

当前单一真源已扩展以下结构化对象。

### 3.1 `docWorkspace`

作用：声明当前正式工作区。

建议字段：

1. `workspaceId`
2. `workspacePath`
3. `versionScope`
4. `status`
5. `activatedAt`
6. `frozenAt`

### 3.2 `docRegistry`

作用：注册本工作区中的正式文档。

建议字段：

1. `docId`
2. `title`
3. `path`
4. `docType`
5. `status`
6. `domain`
7. `versionScope`
8. `owner`
9. `lastUpdatedAt`
10. `relatedJobs`
11. `authoritative`
12. `mustSyncWhen`

### 3.3 `jobDocBindings`

作用：描述 job 对文档的影响关系。

建议字段：

1. `jobId`
2. `docImpact`
3. `docDeliverables`
4. `checklistRef`
5. `adrRefs`
6. `docSyncRequired`
7. `docSyncStatus`

### 3.4 `adrRegistry`

作用：登记本工作区内 ADR。

建议字段：

1. `adrId`
2. `title`
3. `path`
4. `status`
5. `supersedes`
6. `relatedJobs`

### 3.5 `docChangeLog`

作用：登记正式文档变更。

建议字段：

1. `changeId`
2. `date`
3. `jobId`
4. `affectedDocs`
5. `changeType`
6. `summary`

## 4. 真源与 Markdown 的职责分离

### 4.1 真源负责什么

1. 状态。
2. 关系。
3. 绑定。
4. 收尾要求。
5. 校验规则。

### 4.2 Markdown 负责什么

1. 正式正文。
2. 说明表格。
3. 字段说明。
4. 接口说明。
5. 验收口径。

## 5. Job 绑定规则

一个 job 至少要回答 4 个问题：

1. 它影响哪些正式文档。
2. 它需要交付哪些新文档。
3. 它是否影响架构决策。
4. 它在什么条件下才能关单。

因此，后续每个 job 至少要绑定：

1. `docImpact`
2. `docDeliverables`
3. `checklist`
4. `closeCriteria`

## 6. 派生视图规则

后续建议保留两类派生视图：

1. 仓库全局派生视图
   - 项目总表视图
   - 交接视图
   - 下一步视图
2. 工作区派生视图
   - `JOB与文档绑定矩阵`
   - `文档变更记录`
   - `ADR索引`
   - `JOB索引`

这两类视图都应来自同一份真源关系，而不是分别手工维护。

## 8. 绑定后的关单要求

后续 job 关单前必须同时满足：

1. 代码完成。
2. checklist 完成。
3. 文档同步完成。
4. 真源状态同步完成。
5. 回归验证完成。

只做代码、不做文档同步，不允许关单。

本工作区不仅定义了正式规则，也已经完成首轮落地：

1. `source_of_truth.json` 已新增工作区与文档注册元数据。
2. `project_management/lib/project-docs.js` 已支持生成工作区视图并校验一致性。
