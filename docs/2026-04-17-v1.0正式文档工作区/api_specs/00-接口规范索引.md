## 1. 作用

本目录用于把现有接口说明扩展成真正可开发的详细规范。

目标是让其他 Codex 不只知道“接口叫什么”，而且知道：

1. 参数类型和默认值。
2. 鉴权方式。
3. 典型请求与响应。
4. 错误码。
5. 副作用。
6. 代码入口。
7. 测试覆盖点。
