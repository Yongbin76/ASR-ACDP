# 词典建设域详细设计

- 文档状态：active
- 适用版本：v1.0
- 文档类型：design
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-18
- 责任对象：词典记录、基础配置、准入校验

## 1. 设计目标

词典建设域负责回答三个问题：

1. 什么词条可以进入系统。
2. 词条进入系统时需要保存哪些结构化信息。
3. 词条保存后如何进入审核链和发布链。

## 2. 主对象

词典建设域的主对象是：

1. `term`
2. `aliases`
3. `term_rules`
4. `pinyin_profiles`
5. `business property`
6. `source type`

说明：

1. `term` 是主语。
2. `aliases`、`rules`、`pinyin profile` 都从属于 `term`。
3. `business property` 和 `source type` 是配置型主数据，不直接进入 runtime 引擎。

## 3. 页面与路由

### 3.1 页面

1. `/console/dictionary/terms`
2. `/console/dictionary/config`

### 3.2 核心接口

1. `/api/console/dictionary/terms*`
2. `/api/console/dictionary-config/business-attributes*`
3. `/api/console/dictionary-config/source-types*`

## 4. 词典记录的输入模型

一个词典记录在当前系统中至少包含：

1. `categoryCode`
2. `canonicalText`
3. `aliases`
4. `priority`
5. `riskLevel`
6. `replaceMode`
7. `baseConfidence`
8. `sourceType`
9. `pinyinRuntimeMode`
10. `rules`
11. `pinyinProfile`

## 5. 状态机

### 5.1 词条状态

1. `draft`
2. `pending_review`
3. `approved`
4. `disabled`

### 5.2 状态迁移

1. 创建后进入 `draft`
2. 提交审核后进入 `pending_review`
3. 审核通过后进入 `approved`
4. 审核驳回后回到 `draft`
5. 停用后进入 `disabled`

## 6. 准入设计

### 6.1 统一准入入口

当前统一准入逻辑位于：

1. [`term-admission.js`](/Codex/ACDP/prototype/src/lib/term-admission.js)

### 6.2 当前准入检查重点

1. 标准词不能为空
2. 别名要去空、去重
3. 来源类型和业务属性必须有效
4. 单字错误词阻断规则生效
5. 完全重复或冲突条目要识别出来

### 6.3 准入输出

准入层输出：

1. `level`
2. `issues`
3. `summary`

当前系统中，这些结果会被：

1. 词条创建接口消费
2. 词条更新接口消费
3. 导入预览消费
4. 审核详情消费

## 7. 基础配置设计

### 7.1 业务属性

业务属性用于表达：

1. 词条属于哪类业务主题

当前常见值：

1. `poi_road`
2. `gov_term`
3. `proper_noun`

### 7.2 来源类型

来源类型用于表达：

1. 词条或验证样本来自哪里

当前来源类型要求至少有：

1. `scope`
2. `enabled`
3. `code`
4. `label`

## 8. 主要流程

### 8.1 手工创建词条

1. 页面填写字段
2. 调用 `POST /api/console/dictionary/terms`
3. 进入准入校验
4. 通过后写入 `terms/aliases/rules/pinyin_profiles`
5. 返回 `draft` 状态词条

### 8.2 编辑词条

1. 页面读取详情
2. 修改基础信息或规则 / 拼音画像
3. 调用 `PUT /api/console/dictionary/terms/{termId}`
4. 准入校验再次执行
5. 更新词条并增加 `revision`

### 8.3 提交审核

1. 调用 `POST /submit-review`
2. 再次校验准入摘要
3. 状态切到 `pending_review`
4. 创建或复用 `term_review`

## 9. 关键边界

1. 词典建设只负责“录入和维护”，不直接负责发布。
2. 词条创建成功不等于可发布，必须先通过审核。
3. 基础配置是配置型主数据，不应与 release 生命周期绑定。
