# 版本发布域详细设计

- 文档状态：active
- 适用版本：v1.0
- 文档类型：design
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-18
- 责任对象：release build、发布审核、gray、publish、rollback

## 1. 设计目标

版本发布域负责把“已通过审核的词条集合”转换成“可对 runtime 生效的版本”。

## 2. 主对象

1. `release`
2. `release_terms`
3. `gray_policy`
4. `release_publish_review`

## 3. 状态分层

### 3.1 生命周期状态

1. `built`
2. `canary`
3. `published`

### 3.2 审批状态

1. `not_submitted`
2. `pending`
3. `partially_approved`
4. `approved`
5. `rejected`

### 3.3 流量状态

1. `no_gray`
2. `gray_enabled`
3. `gray_closed`

## 4. 主流程

### 4.1 build

1. 调用 `buildRelease()`
2. 读取 buildable 词条
3. 生成 snapshot / manifest
4. 写入 `releases` 与 `release_terms`

### 4.2 submit review

1. 调用 `submitReleaseReview()`
2. 校验 release 状态必须是 `built/canary`
3. 创建 `release_publish_review`

### 4.3 canary

1. 调用 `createGrayPolicy()`
2. release 可进入灰度
3. runtime 根据灰度策略分流

### 4.4 publish

1. 调用 `ensureReleaseApprovalForExposure()`
2. 校验审批和 gate
3. 激活 release 为 published
4. 刷新 runtime state

### 4.5 rollback

1. 选择目标 release
2. 激活为 published
3. 刷新 runtime state

## 5. 门禁设计

当前 gate 重点关注：

1. release 中是否仍包含非 `approved` 词条
2. 是否存在待审核词条任务
3. 是否存在待审核拼音候选任务
4. validation 样本是否通过

## 7. 修改风险

1. 改 release 状态机会影响页面与控制链。
2. 改审批规则会影响高风险曝光控制。
3. 改 gate 输入会影响发布判断。
