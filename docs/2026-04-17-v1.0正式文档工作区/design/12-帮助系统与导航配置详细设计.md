# 帮助系统与导航配置详细设计

- 文档状态：active
- 适用版本：v1.0
- 文档类型：design
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-18
- 责任对象：`console_help.json`、`console_navigation.json`

## 1. 设计目标

帮助系统和导航配置负责：

1. 让 `/console` 左侧目录树配置化。
2. 让帮助目录和页面入口保持同源。

## 2. 导航配置

配置文件：

1. [`console_navigation.json`](/Codex/ACDP/prototype/config/console_navigation.json)

当前配置要素：

1. 一级分组
2. 二级页面
3. `path`
4. `navKey`
5. `helpSlug`
6. `iconGlyph`
7. `shortLabel`

## 3. 帮助配置

配置文件：

1. [`console_help.json`](/Codex/ACDP/prototype/config/console_help.json)

当前配置要素：

1. `slug`
2. `title`
3. `summary`
4. `kicker`
5. `sourceDocPath`

## 4. 关键边界

1. 页面帮助入口与导航分组都应从配置读取。
2. 帮助正文文件当前主要仍位于 `docs/help_manuals/`。
3. `sourceDocPath` 当前也承接少量在用根目录文档，如 `docs/25`、`docs/26`。

## 5. 关键代码入口

1. [`console-navigation.js`](/Codex/ACDP/prototype/src/lib/console-navigation.js)
2. [`console-help.js`](/Codex/ACDP/prototype/src/lib/console-help.js)
3. [`admin-surface.js`](/Codex/ACDP/prototype/src/http/admin-surface.js)
4. [`app.js`](/Codex/ACDP/console/client/app.js)

## 6. 修改风险

1. 改 `navKey/path/helpSlug` 会同时影响页面导航和帮助跳转。
2. 改 `sourceDocPath` 会影响帮助正文下载。
