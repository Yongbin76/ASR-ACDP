# 验证与回流域详细设计

- 文档状态：active
- 适用版本：v1.0
- 文档类型：design
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-18
- 责任对象：验证样本、外部回流、validation feed

## 1. 设计目标

验证与回流域负责：

1. 维护验证样本。
2. 接入外部反馈。
3. 为 release gate 和发布后风险提供观测输入。

## 2. 主对象

1. `validation_case`
2. validation feed connector
3. feed receipt / archive / error 文件流转

## 3. 样本来源

当前样本来源包括：

1. 手工创建
2. CSV 导入
3. feed 回流导入

## 4. feed 接入设计

当前 feed 支持：

1. `file_inbox`
2. `http_pull_json`

当前示例来源：

1. `cg3`
2. `qa_feedback`
3. `online_feedback`

## 5. 数据流

### 5.1 手工创建 / CSV 导入

1. 录入样本文本和期望标准词
2. 写入 `validation_cases`
3. 供页面和 gate 消费

### 5.2 外部回流

1. 从 inbox 或远端接口获取载荷
2. 解析成统一 validation case 输入
3. 导入成功后归档到 `archive`
4. 失败则流转到 `error`
5. 记录 receipt

## 6. 与词典建设的边界

1. 验证样本不是词典记录。
2. 验证样本不直接生成 release。
3. 它通过 gate 和风险视图间接影响发布判断。

## 8. 修改风险

1. 改 feed 协议会影响外部联调。
2. 改 validation case 输入结构会影响 CSV、手工和 feed 三条路径。
3. 改 gate 对样本的消费方式会影响 release 判断。
