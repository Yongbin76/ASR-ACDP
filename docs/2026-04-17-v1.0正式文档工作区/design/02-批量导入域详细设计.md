# 批量导入域详细设计

- 文档状态：active
- 适用版本：v1.0
- 文档类型：design
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-18
- 责任对象：导入模板、导入批次、逐行预览、确认导入

## 1. 设计目标

批量导入域负责：

1. 接收结构化文件输入。
2. 解析成系统统一的词条或样本输入模型。
3. 在入库前给出逐行预览和问题说明。
4. 在确认后写入主数据并生成审核任务。

所有导入行最终都会被归一化为：

6. `decision`

8. `errorMessage`

## 6. 预览设计

### 6.1 预览阶段的目的

预览阶段不直接写入主数据，而是回答：

1. 哪些行可导入
2. 哪些行仅有 warning
3. 哪些行会阻断
4. 哪些行会更新已有词条

### 6.2 `warning` 与 `error`

1. `warning`
   - 当前行可继续，但有风险提示
2. `error`
   - 当前行阻断入库，但不再默认拖死整批确认

## 7. 确认导入设计

### 7.1 核心约束

当前推荐口径为：

1. 整批阻断只保留给模板级、文件级、系统级错误
2. 行级 `error` 只阻断对应行
3. 批次确认时允许继续导入 `ready` / `warning` 行
4. 行级 `error` 通过 `skipped_blocked` 标记为“阻断未导入”

### 7.2 确认导入后的行为

#### 词条导入

1. 新词条：
   - 创建 `terms`
   - 写别名、来源、规则、拼音画像
2. 已存在词条：
   - 更新词条
   - 合并 alias
3. 行级 `error`：
   - 不入库
   - 不生成审核任务
   - 保留在批次中供后续导出和复查
4. 所有变更词条：
   - 统一进入 `submitTermReview()`
   - 生成 `term_review`

#### 验证样本导入

1. 调用 `importValidationCases()`
2. 不生成词条审核任务

### 7.3 导入结果汇总

确认后会写 `import_job_results`，并返回：

1. 新增词条数
2. 更新词条数
3. 新增别名数
4. 更新别名数
5. 导入成功数
6. warning 导入数
7. 跳过阻断数
8. 错误数
9. 生成的审核任务 ID

## 7.4 页面交互要求

批次详情页必须同步满足以下要求：

1. 主按钮文案改为 `导入可通过记录`
2. 当 `importableRows > 0` 时允许点击
3. 被跳过的阻断行在导入完成后继续可筛选、可下载
4. 页面按钮、卡片、下拉 label 和提示文案全部中文化

## 8. 关键边界

1. 批量导入不直接改变 release。
2. 导入只是词条建设的一种输入方式。
3. 导入确认后仍必须走统一审核主语 `taskId`。

## 10. 修改风险

1. 改模板字段会影响：
   - 模板文件
   - 模板说明
   - 解析逻辑
   - 导入测试
2. 改 `confirmImportJob()` 行为会影响：
   - 主数据入库
   - 审核任务生成
   - 导入详情页
