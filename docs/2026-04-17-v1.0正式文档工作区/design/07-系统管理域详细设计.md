# 系统管理域详细设计

- 文档状态：active
- 适用版本：v1.0
- 文档类型：design
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-18
- 责任对象：用户、角色、权限、治理策略

## 1. 设计目标

系统管理域负责回答：

1. 谁能登录并操作什么页面。
2. 哪些动作由权限控制。
3. 哪些高风险动作由治理策略进一步约束。

## 2. 主对象

1. `permissions`
2. `roles`
3. `users`
4. `PAGE_FEATURE_MATRIX`
5. `governance policies`

## 3. 当前模型

### 3.1 权限

权限是最小授权单位，例如：

1. `term.read`
2. `term.write`
3. `review.decide`
4. `release.publish`
5. `runtime.node.registry.manage`

### 3.2 角色

角色是权限集合，例如：

1. `dict_viewer`
2. `dict_editor`
3. `dict_reviewer`
4. `dict_publisher`
5. `dict_operator`
6. `dict_admin`

### 3.3 用户

用户绑定默认角色和可切换角色集合。

### 3.4 页面能力点

页面能力点把权限映射到具体页面动作。

例如：

1. `terms.create`
2. `reviews.bulk.approve`
3. `releases.publish`

### 3.5 治理策略

治理策略用于补充权限无法表达的限制，例如：

1. 发布审核双人审批
2. 提交人与审核人分离
3. 高风险动作二次确认

## 5. 关键边界

1. 权限只回答“能不能做”。
2. 治理策略回答“在什么条件下允许做”。
3. 页面是否显示入口取决于 `pageAccess / pageFeatures`。

## 6. 修改风险

1. 改角色权限会影响多个页面入口。
2. 改页面能力点会影响前端按钮可见性和后端动作校验。
3. 改治理策略会影响 release 审批与高风险动作。
