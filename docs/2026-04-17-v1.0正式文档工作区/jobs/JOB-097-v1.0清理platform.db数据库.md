# JOB-097 v1.0 清理 platform.db 数据库

- 文档状态：active
- 适用版本：v1.0
- 文档类型：job
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-22
- 责任对象：`prototype/workspace/platform.db` 历史过程数据清理与物理压缩

## 1. 目标

针对 [platform.db](/Codex/ACDP/prototype/workspace/platform.db) 体积过大的问题，按三步收口方式完成数据库清理与优化：

1. `JOB-097A`
   - 清 `audit_logs`
   - 执行 `VACUUM`
2. `JOB-097B`
   - 清历史过程表
   - 执行 `VACUUM`
3. `JOB-097C`
   - 做索引健康检查与优化
   - 必要时执行定向 `REINDEX`

## 2. 边界

本 job 只处理数据库文件瘦身，不处理：

1. 词条主数据重建
2. release / snapshot 文件清理
3. runtime 实例工作区清理
4. 零数据底座初始化

## 3. 执行前提

执行任一子任务前，必须先满足：

1. 停止会写入主库的 admin / prototype / runtime 进程
2. 先备份 `platform.db`
3. 记录清理前基线：
   - 文件大小
   - `PRAGMA page_count`
   - `PRAGMA freelist_count`
   - 目标表行数

## 4. 子任务拆分

### 4.1 JOB-097A

范围：

1. 仅清 `audit_logs`
2. 不碰其他业务表

目标：

1. 优先回收最大头的审计历史空间
2. 在不影响现有词条、导入、审核和发布状态的前提下完成首轮瘦身

### 4.2 JOB-097B

范围：

1. 清历史过程表：
   - `review_tasks`
   - `import_job_rows`
   - `import_jobs`
   - `import_job_files`
   - `import_job_results`
   - `alias_sources`
   - `term_sources`
2. 不碰以下主表：
   - `terms`
   - `aliases`
   - `term_rules`
   - `pinyin_profiles`
   - `releases`
   - `release_terms`
   - `runtime_*`

目标：

1. 清掉导入、审核、来源追溯这些历史过程数据
2. 保留当前主词典与版本发布成果

### 4.3 JOB-097C

范围：

1. `integrity_check`
2. `ANALYZE`
3. `PRAGMA optimize`
4. 关键查询 `EXPLAIN QUERY PLAN`
5. 必要时的定向 `REINDEX`

目标：

1. 做索引健康检查
2. 做统计信息刷新
3. 让关键查询计划可留档
4. 只在必要时重建目标索引

## 5. 统一执行口径

每个子任务都按同一顺序执行：

1. 停服务
2. 备份 `platform.db`
3. 记录清理前基线
4. 执行对应子任务动作
5. 执行（若该子任务需要）：
   - `PRAGMA wal_checkpoint(TRUNCATE);`
   - `VACUUM;`
6. 记录清理后基线
7. 启动服务并验证核心功能
