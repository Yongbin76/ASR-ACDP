# ADR-003 正式文档工作区与单一真源强绑定

- 文档状态：active
- 适用版本：v1.0
- 文档类型：adr
- 所属工作区：2026-04-17-v1.0正式文档工作区
- 最后更新时间：2026-04-17
- 决策状态：accepted

## 1. 决策

正式文档工作区必须与单一真源、JOBLIST 和 CHECKLIST 统一治理，不能作为独立漂浮体系存在。

## 2. 理由

1. 过去的问题不是文档不够，而是文档、job 状态和代码事实彼此脱节。
2. 如果文档不进入真源关系，后续任何新增功能都会重新长出“另一套现行文档”。

## 3. 结果

1. 文档工作区必须有索引、治理规则、job 绑定矩阵、变更记录和归档规则。
2. 后续 job 关单必须包含文档同步。
3. 后续真源应扩展到同时管理文档注册表与 job 绑定关系。
