# 拆分部署首轮验证执行清单

## 1. 目的

本清单用于在具备 `docker` 与 `kubectl` 的宿主机或运维节点上，执行 runtime/admin 拆分部署的首轮验证。

完整运维说明请配合：

- [37-Runtime-Admin服务运维手册](./37-runtime-admin服务运维手册.md)

说明：

- 当前仓库中的拆分部署资产已经准备完成
- 当前主机已经具备 `docker` 与 `kubectl`
- 但真实目标集群的 kube context 仍可能缺失
- 因此本清单仍是下一步实际目标集群验证的直接执行入口

## 2. 前置条件

执行前确认：

1. 宿主机已安装 `docker`
2. 宿主机已安装 `kubectl`
3. `kubectl config current-context` 指向目标集群
4. 目标集群允许拉取或加载本地镜像
5. `acdp` namespace 与 PVC 可创建
6. 已准备 artifact store / runtime token 对应 Secret

建议先执行：

```bash
cd /Codex/ACDP
npm run check:k8s-target
```

用途：

- 检查 kubectl client 是否可用
- 检查当前是否已有 `current-context`
- 检查 `acdp` namespace 是否存在
- 检查 `acdp-artifact-store` 与 `acdp-runtime-auth` Secret 是否存在

## 3. 构建镜像

```bash
cd /Codex/ACDP
docker build -f Dockerfile.runtime -t acdp-runtime:latest .
docker build -f Dockerfile.admin -t acdp-admin:latest .
```

建议记录：

- 镜像构建开始/结束时间
- 镜像 ID
- 构建失败日志

## 4. 应用 K8S 资产

```bash
cd /Codex/ACDP
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/pvc.yaml
kubectl apply -f k8s/artifact-store-secret.example.yaml
kubectl apply -f k8s/runtime-deployment.yaml
kubectl apply -f k8s/runtime-service.yaml
kubectl apply -f k8s/admin-deployment.yaml
kubectl apply -f k8s/admin-service.yaml
```

建议记录：

- `kubectl get secret -n acdp`
- `kubectl get pods -n acdp`
- `kubectl get svc -n acdp`
- `kubectl describe deploy acdp-runtime -n acdp`
- `kubectl describe deploy acdp-admin -n acdp`

## 5. 验证项

### 5.1 Runtime

目标：

- runtime pod 正常启动
- `/health` 正常
- `/api/runtime/current` 正常
- runtime service 不暴露 `/admin`
- runtime 能自动 register 到 admin，并在 configured 模式下从 MinIO 拉取制品

建议命令：

```bash
kubectl get pods -n acdp
kubectl port-forward svc/acdp-runtime 8787:8787 -n acdp
curl http://127.0.0.1:8787/health
curl http://127.0.0.1:8787/api/runtime/current
curl -i http://127.0.0.1:8787/admin
```

### 5.2 Admin

目标：

- admin pod 正常启动
- `/admin` 与 `/console` 正常
- `/api/admin/dashboard` 与 `/api/console/overview` 正常
- admin service 不暴露 `/api/runtime/current`
- admin 能把 release 制品上传到配置指向的 MinIO

建议命令：

```bash
kubectl port-forward svc/acdp-admin 8788:8788 -n acdp
curl http://127.0.0.1:8788/admin
curl http://127.0.0.1:8788/console
curl -H 'x-role: dict_admin' -H 'x-operator: deploy_verify' http://127.0.0.1:8788/api/admin/dashboard
curl -H 'x-role: dict_admin' -H 'x-operator: deploy_verify' http://127.0.0.1:8788/api/console/overview
curl -i http://127.0.0.1:8788/api/runtime/current
```

## 6. 建议留档

建议统一记录到：

- `docs/35-拆分部署资产使用说明.md`
- 宿主/集群执行日志目录
- `docs/27-console联调记录模板.md` 中的扩展备注

建议至少保留：

- `kubectl get pods -n acdp -o wide`
- `kubectl logs deploy/acdp-runtime -n acdp`
- `kubectl logs deploy/acdp-admin -n acdp`
- `curl` 返回截图或文本

## 7. 当前阻塞

当前这一步若仍无法继续，常见原因应优先收敛为：

- `kubectl config current-context` 为空
- 目标 kubeconfig 尚未提供
- `acdp` namespace / Secret 尚未创建

因此后续若要继续“真实拆分部署验证”，建议先跑 `npm run check:k8s-target`，再补齐对应阻塞项。
