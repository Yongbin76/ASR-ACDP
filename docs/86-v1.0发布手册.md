# v1.0 发布手册

## 1. 文档目的

本手册用于指导 `v1.0` 正式发布时的实际执行。

适用对象：

- 发布负责人
- 审核负责人
- 运维负责人

## 2. 发布目标

本手册覆盖的发布目标包括：

- 确认当前候选版本具备正式发布条件
- 完成发布前检查
- 执行正式发布
- 留存发布证据

## 3. 发布前提

执行正式发布前，必须同时满足：

1. 当前 release 已完成构建
2. 当前 release 已完成发布审核
3. 当前 release 版本校验通过
4. `JOB-006` 真实部署验证已完成
5. `JOB-009` 真实宿主机并发与吞吐验证已完成
6. `smoke:console`、`test:console`、`test:unit` 全部通过
7. 发布说明、回滚手册、go / no-go 清单已准备完毕

## 4. 发布前检查

建议按以下顺序执行：

1. 确认当前候选 release 的 `releaseState`、`approval`、`releaseCheck`
2. 确认当前不存在阻断级发布后风险
3. 确认运行节点中心当前状态可解释，且关键节点在线
4. 确认帮助文档、接口文档和关键页面中文化收口完成
5. 生成本次发布证据目录
6. 填写 go / no-go 清单

## 5. 执行步骤

### 5.1 控制台核查

在控制台中完成以下核查：

1. 进入“版本发布 -> 版本列表”
2. 打开目标 release 详情
3. 核对：
   - 版本状态
   - 发布审核
   - 流量状态
   - 版本校验
   - 发布后风险
4. 进入“运行治理 -> 运行节点”
5. 确认关键节点在线且目标版本可收敛

### 5.2 执行发布

操作原则：

1. 先确认审批状态已满足
2. 再由具备发布权限的角色执行正式发布
3. 发布后立即记录发布时间、执行人、目标 releaseId

### 5.3 发布后核查

发布后立即执行：

1. 检查 release 已进入 `published`
2. 检查灰度策略是否符合预期
3. 检查运行节点的目标版本与当前版本
4. 检查关键 runtime 对外接口是否正常
5. 检查工作台和发布后风险页是否出现新的阻断问题

## 6. 部署交付与发布说明包

正式发布前，应至少准备两类附属交付物：

1. 部署交付包
2. 发布说明包

目录与命名建议：

- [124-v1.0镜像标签与部署交付包目录模板](./124-v1.0镜像标签与部署交付包目录模板.md)
- [125-v1.0发布说明包与文档快照清单](./125-v1.0发布说明包与文档快照清单.md)

本地可先生成模板目录：

```bash
cd /Codex/ACDP
npm run prepare:release-bundle -- \
  --version v1.0.0 \
  --release-id rel_demo_001 \
  --image-registry registry.example.com \
  --output-dir ./prototype/workspace/release_bundle_preview
```

## 7. 证据留档

发布时建议至少留档：

- release 列表导出
- release 详情导出
- 发布审核通过证据
- 发布后运行节点状态证据
- 控制台截图
- go / no-go 清单

建议归档到：

- `prototype/workspace/host_verification/<timestamp>_v1_release_readiness/`

## 8. 失败处理

如果发布前任一步骤不满足，应立即停止，并记录：

- 当前阻断项
- 负责人
- 下一步动作

如果发布后出现阻断级问题，应转入：

- [87-v1.0回滚手册](./87-v1.0回滚手册.md)
