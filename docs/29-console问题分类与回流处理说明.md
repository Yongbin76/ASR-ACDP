# Console 问题分类与回流处理说明

## 1. 文档目的

本文件用于给 `/console` 的联调问题和试用反馈建立统一分类和回流处理机制。

## 2. 问题分类建议

### 2.1 环境问题

典型包括：

- Node 版本不符合要求
- 本地端口监听受阻
- 文件权限问题
- 网络访问限制

### 2.2 页面问题

典型包括：

- 页面打不开
- 静态资源加载失败
- 导航异常
- 表单显示问题
- 布局错位

### 2.3 交互问题

典型包括：

- 操作路径不清楚
- 提示不清楚
- 错误反馈不可执行
- 表单过于复杂

### 2.4 API/业务问题

典型包括：

- 导入失败
- 审核失败
- 发布失败
- gate 误判
- validation 结果异常

### 2.5 数据问题

典型包括：

- 词条未写入
- 别名未写入
- 来源映射缺失
- 导入结果与页面展示不一致

## 3. 严重级别建议

### P0

系统不可用，阻塞联调或试用。

### P1

主链路严重受阻，影响试用核心任务。

### P2

功能可绕过，但明显影响体验或结果正确性。

### P3

细节优化项，不阻塞当前试用。

## 4. 回流处理建议

每个问题建议记录：

- 问题编号
- 来源
- 分类
- 严重级别
- 复现步骤
- 当前状态
- 责任人
- 预计处理时间

## 5. 当前状态建议

建议状态：

- `new`
- `triaged`
- `in_progress`
- `fixed`
- `verified`
- `deferred`

## 6. 回流节奏建议

### 日常试用阶段

- P0/P1：当天处理
- P2：纳入本轮修复
- P3：纳入后续优化池

### 阶段性回顾

每轮试用结束后，建议按以下输出：

1. 问题总量
2. 各等级分布
3. 各分类分布
4. 已修复/未修复情况
5. 下轮优先级
