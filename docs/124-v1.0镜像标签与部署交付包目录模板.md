# v1.0 镜像标签与部署交付包目录模板

## 1. 文档目的

本文档用于固定：

- `v1.0` 正式发布时的镜像 tag 命名规则
- 部署交付包目录结构
- 运维交接时必须包含的文件清单

目标是让后续执行 `JOB-100` 或 `JOB-030` 时，不需要再自己决定：

- 镜像叫什么
- 部署包长什么样
- 交给运维的最终文件有哪些

## 2. 镜像 tag 命名规则

建议采用 3 层 tag：

### 2.1 固定版本 tag

- `acdp-admin:v1.0.0`
- `acdp-runtime:v1.0.0`

用途：

- 正式发布
- 正式回滚
- 最终留档

### 2.2 release 绑定 tag

- `acdp-admin:release-<releaseId>`
- `acdp-runtime:release-<releaseId>`

示例：

- `acdp-admin:release-rel_1775721135265`
- `acdp-runtime:release-rel_1775721135265`

用途：

- 与控制面 release 记录一一对应
- 便于证据包和镜像版本回溯

### 2.3 环境推进 tag

- `acdp-admin:rc`
- `acdp-runtime:rc`
- `acdp-admin:latest`
- `acdp-runtime:latest`

用途：

- `rc`：候选验证环境
- `latest`：仅限内部非正式追踪，不作为正式回滚依据

## 3. tag 使用原则

1. 正式发布必须同时产出：
   - 固定版本 tag
   - release 绑定 tag
2. 回滚时优先使用：
   - release 绑定 tag
   - 或固定版本 tag
3. 不允许只依赖：
   - `latest`
   - 无 release 绑定信息的临时 tag

## 4. 部署交付包目录模板

建议目录：

```text
release_bundle_templates/deployment_bundle/
  README.md
  images/
    image-tags.json
  k8s/
    namespace.yaml
    admin-deployment.yaml
    admin-service.yaml
    runtime-deployment.yaml
    runtime-service.yaml
    ingress.yaml
  env/
    admin.env.example
    runtime.env.example
  secrets/
    secrets-template.md
  docs/
    deployment-manual.md
    rollback-manual.md
    release-notes.md
```

## 5. 每个目录必须包含什么

### 5.1 `images/`

必须包含：

- `image-tags.json`

建议字段：

```json
{
  "version": "v1.0.0",
  "releaseId": "rel_xxx",
  "adminImage": "registry.example.com/acdp-admin:v1.0.0",
  "runtimeImage": "registry.example.com/acdp-runtime:v1.0.0",
  "adminReleaseTag": "registry.example.com/acdp-admin:release-rel_xxx",
  "runtimeReleaseTag": "registry.example.com/acdp-runtime:release-rel_xxx"
}
```

### 5.2 `k8s/`

必须包含：

- namespace
- admin deployment/service
- runtime deployment/service

如有对外入口，还应包含：

- ingress

### 5.3 `env/`

必须包含：

- `admin.env.example`
- `runtime.env.example`

说明：

- 只放样例，不放真实密钥

### 5.4 `secrets/`

必须包含：

- `secrets-template.md`

应说明：

- artifact store 凭据
- runtime token
- registration secret 注入方式

### 5.5 `docs/`

必须包含：

- `deployment-manual.md`
- `rollback-manual.md`
- `release-notes.md`
